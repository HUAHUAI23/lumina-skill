# task.video

## 适用场景

- 用户希望把当前轮需求整理成可直接下发的视频生成任务。
- 任务只依赖当前消息文本、当前消息上传素材和已注入的项目风格，不读取项目维护的人物、场景或道具资产。
- 适合单段视频、短剧片段、产品广告、口播、动作展示、氛围镜头、参考图生视频和首帧图生视频。

## 负责什么

- 将当前轮的视频需求整理成清晰、可生成的视频任务描述。
- 判断当前轮是纯文本生成、首帧图生视频、首尾帧、参考图生视频、参考视频风格/运镜复刻、产品展示、短剧对白、音乐卡点还是视频编辑类需求。
- 只基于当前轮上传素材判断参考素材是否参与，以及每个素材承担什么职责。
- 压实主体、场景、动作、运镜、节奏、光影、声音、台词和稳定性约束。
- 对剧本、对白、戏文或分场说明进行镜头化改写，而不是照搬原文。

## 不负责什么

- 不读取项目维护的人物、场景、道具或历史资产。
- 不做资产绑定。
- 不把 story-to-video 多阶段流程规划塞进当前轮视频任务。
- 不承诺自动完成长视频延长、跨段拼接或已有项目视频续写；当前任务只整理本轮可下发的视频生成需求。
- 不输出 SD 标签串、权重括号、命令行参数或平台私有语法。

## 判断原则

- Seedance 2.0 类视频模型更适合自然语言、清晰时序、明确镜头和具体动作；不要只堆风格词。
- 参考素材只来自本轮上传；每个参考素材都要说明职责，例如首帧、尾帧、人物外观、产品外观、场景、构图、姿态、动作、运镜、节奏、音色或声音氛围。
- 对 4-8 秒短视频，聚焦 1-2 个核心动作或产品展示点；对 9-12 秒短场景，拆成 2-3 个时间段；对 13-15 秒完整叙事，优先拆成 3-4 个时间段。
- 单段视频以 4-15 秒为主要控制范围。用户要求更长时，当前链路可拆成多个独立片段任务，但不要伪装成自动视频延长链路。
- 一个镜头里尽量只指定一种主运镜方式，例如固定镜头、缓慢推近、平稳横移、跟拍、俯拍、过肩或特写切换；不要同时要求推、拉、摇、移。
- 动作优先写连续、可衔接的小动作，减少狂奔、大跳、剧烈翻滚等容易导致画面漂移的动作，除非用户明确要求动作大片。
- 情绪要外化为可见信号，例如停顿、眼神、嘴角、眼眶、肩膀、手指、呼吸和步伐，不只写抽象情绪。

## 核心写法

- 固定参数单独收束：时长、比例、画面大小、画风、节奏、情绪、光线和声音方向。
- 正文按“时间 + 镜头运镜 + 精准主体 + 动作细节 + 场景环境 + 光影色调 + 声音设计 + 稳定性约束”组织。
- 如果用户给原始剧本或对白，先提取可拍内容，再把台词转成身体动作、视线、停顿、表情和镜头关系。
- 台词要标注说话人和语气，放在对应画面段落里；不要让整段视频变成念白。
- 声音设计要和画面同步，分别考虑环境音、动作音效、对白/旁白和 BGM 情绪强度。
- 结尾补稳定性约束：人物身份一致、面部稳定、动作自然、光影连续、材质不闪变、画面无卡顿无水印。

## 输出模式

- 直投平台模式：用户要“可直接复制到 Seedance / 火山引擎 / 平台使用”的最终提示词时，可以把固定参数、公共参考素材、时间轴正文、声音设计和约束条件合并成一份完整 prompt。
- 管线结构化模式：用户要进入 Lumina 工作流、story_to_video、任务 JSON 或编译器链路时，固定参数、visualPrompt、dialogues、audioDirectives 和 negativeConstraints 必须分字段；visualPrompt 不写“台词：”“声音：”“BGM：”栏目。
- 默认按直投平台模式整理当前轮单次视频任务；若上游已明确字段名或编译器会追加台词/声音，改用管线结构化模式。
- 两种模式都要保留同一套镜头逻辑：参考素材职责清楚、时间段动作连续、台词有画面锚点、声音有画面来源、结尾约束短而有效。

## 参考素材写法

- 面向火山引擎 / Seedance 平台直用的提示词，统一使用官方引用命名：`@图片1`、`@图片2`、`@视频1`、`@音频1`。不要混用 `@图1`、`@img1`、`参考图1`、`视频一` 等非官方写法。
- 如果运行时只提供“当前上传图片编号”，输出时仍要把它们转成平台可读的 `@图片N` / `@视频N` / `@音频N`，并逐一说明职责。
- 纯文本生成：直接写主体、动作序列、环境光影、镜头语言、声音和风格。
- 首帧图生视频：明确当前上传图作为起始画面，视频从这张图自然动起来，后续动作和镜头延展要与首帧构图一致。
- 首尾帧：明确起始画面和结束画面，中间动作要自然过渡，不突然换身份、换空间或换画风。
- 多参考图：逐张说明职责，例如 `@图片1` 锁人物身份，`@图片2` 锁场景空间，`@图片3` 锁产品材质，`@图片4` 锁姿态或特写；不要把参考图当逐帧动作。
- 图片生成提示词和视频提示词要分层：图片生成阶段可以完整描述“生成什么图”；视频阶段只保留短职责绑定，例如“参考 `@图片1` 的无人街道空间，参考 `@图片2` 的人物身份，参考 `@图片3` 的道具形态”。不要把每张图片的完整生成说明塞进视频正文。
- 多图职责绑定可以写“参考素材只锁定身份、场景、道具和关键证据，不代表画面出现顺序”；避免写“按 `@图片1` 至 `@图片4` 的顺序读取参考”这类容易被误解成逐帧播放的表达。
- 参考视频：只在用户确实上传视频或明确要求时使用，写清参考的是运镜、动作、节奏、特效、剪辑还是音色，不要模糊写“参考视频整体感觉”；例如“参考 `@视频1` 的手持跟拍节奏”。
- 音频参考：写清参考的是 BGM 节奏、音色、环境声、旁白质感还是卡点节拍；例如“BGM 气质参考 `@音频1`，但动作音效按画面重新生成”。
- 素材多时，先列“公共参考素材”职责，再写时间轴正文，避免正文里反复解释同一张参考图。

## 场景类型策略

- 短剧/对白：画面和台词绑定，台词由动作和情绪触发；优先中景、近景、过肩、轻推镜等稳定镜头。
- 仙侠/奇幻：写清法阵、能量、粒子、武器、衣袍和光效的运动逻辑；特效服务动作，不遮挡人物脸。
- 产品/广告：写清产品外观、材质、展示角度、主光、反光、操作动作、卖点和声音节奏；避免人物或背景抢主体。
- 真人口播/带货：优先近景或中景，明确人物手持产品、视线、表情、中文口播、背景和稳定面部约束。
- MV/音乐卡点：明确节拍、切换点、动作卡点、光影变化和参考音频/视频节奏；不要只写“跟音乐卡点”。
- 一镜到底：明确全程不切镜头，写清空间路径、遮挡转场、主体移动和镜头跟随关系。

## 时长与节奏

- 4-8 秒：适合一个动作、一个产品展示、一个氛围镜头或一个短转场，避免塞入复杂剧情。
- 9-12 秒：适合一个完整短场景，可拆成开始、推进、落点。
- 13-15 秒：适合完整微叙事，建议拆成 3-4 个时间段，每段只承担一个主信息点。
- 多段任务之间要有清楚的衔接画面，但当前任务不负责自动把上一段视频上传为下一段延长输入。
- 节奏可以写 slow、medium、fast，但必须落到镜头长度、动作密度、切换频率和声音强度上。
- 动作密度按时长收敛：6 秒内最多 2-3 个主镜头重心，15 秒内最多 3-4 个主信息点。动作戏允许奔跑、追逐、撞击，但不要同时堆多次切镜、复杂走位、多个命中结果和额外反应。

## 画面与声音

- 光影要服务情绪和空间连续性，例如居家常光、清晨暖光、夜晚冷光、雷光、金光、霓虹反射、侧逆光或柔光补光。
- 动态模糊、景深和特效要克制，保证主体清晰、动作可读、面部稳定。
- 环境音写真实空间质感，例如脚步、衣料、呼吸、风声、水声、室内底噪、雷声、金属碰撞。
- BGM 写情绪和强度，例如悬念轻喜感 low、史诗温柔 high、紧张鼓点 mid；不要只写“好听的音乐”。
- 需要无字幕时要明确无字幕、无文字、无 LOGO、无水印；需要台词时也不默认生成屏幕字幕。
- 声音去重后再输出：同类环境声、动作声和 BGM 只保留一次；声音设计要短写成“环境音 + 动作音 + BGM 转折”，不要把同一音效换词重复。

## 压缩与反例过滤

- 最终 prompt 删除导演表格字段：叙事目的、关键确认、黄金三秒、镜头意图、情绪解释、观众会理解什么。
- 最终 prompt 删除过细摄影参数：焦段、光圈、镜头型号、f 值、毫米数；除非用户明确要仿摄影方案。
- 保留可生成信息：时间、景别/运镜、主体、可见动作、场景、光线、台词位置、声音事件和稳定性约束。
- 如果原文像 shot list 或导演案，先提炼成 Seedance 时间分镜，不原样保留栏目堆叠。

## 质量标准

- 视频目标明确，能直接下发生成。
- 主体、动作、场景、运镜、声音、台词和时长不互相冲突。
- 参考素材职责清楚，没有把人物图当场景图、把场景图当逐帧动作或把风格图当身份图。
- 每个时间段都有明确镜头重心，动作之间有自然过渡。
- 人物身份、服装、面部、空间、材质、光影和声音连续稳定。

## 失败回退

- 素材不足时，优先整理纯文本可执行版本，再指出缺口。
- 用户只给抽象创意时，先补主体、场景、动作、运镜、光影、声音和时长。
- 用户要求过长或过复杂时，拆成多个独立短视频任务，并说明每段的画面衔接点。
- 不偷偷引入项目资产、历史视频或流程层上下文。
