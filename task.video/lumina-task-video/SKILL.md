# task.video

## 适用场景

- 用户希望把当前轮需求整理成可直接下发的视频生成任务。
- 任务只依赖当前消息文本、当前消息上传素材和已注入的项目风格，不读取项目维护的人物、场景或道具资产。
- 适合单段视频、短剧片段、产品广告、口播、动作展示、氛围镜头、参考图生视频和首帧图生视频。

## 负责什么

- 将当前轮的视频需求整理成清晰、可生成的视频任务描述。
- 判断当前轮是纯文本生成、首帧图生视频、首尾帧、参考图生视频、参考视频风格/运镜复刻、产品展示、短剧对白、音乐卡点还是视频编辑类需求。
- 只基于当前轮上传素材判断参考素材是否参与，以及每个素材承担什么职责。
- 压实主体、场景、动作、运镜、节奏、光影、声音、台词和稳定性约束。
- 对剧本、对白、戏文或分场说明进行镜头化改写，而不是照搬原文。

## 不负责什么

- 不读取项目维护的人物、场景、道具或历史资产。
- 不做资产绑定。
- 不把 story-to-video 多阶段流程规划塞进当前轮视频任务。
- 不承诺自动完成长视频延长、跨段拼接或已有项目视频续写；当前任务只整理本轮可下发的视频生成需求。
- 不输出 SD 标签串、权重括号、命令行参数或平台私有语法。

## 判断原则

- Seedance 2.0 类视频模型更适合自然语言、清晰时序、明确镜头和具体动作；不要只堆风格词。
- 参考素材只来自本轮上传；每个参考素材都要说明职责，例如首帧、尾帧、人物外观、产品外观、场景、构图、姿态、动作、运镜、节奏、音色或声音氛围。
- 对 4-8 秒短视频，聚焦 1-2 个核心动作或产品展示点；对 9-12 秒短场景，拆成 2-3 个时间段；对 13-15 秒完整叙事，优先拆成 3-4 个时间段。
- 单段视频以 4-15 秒为主要控制范围。用户要求更长时，当前链路可拆成多个独立片段任务，但不要伪装成自动视频延长链路。
- 一个镜头里尽量只指定一种主运镜方式，例如固定镜头、缓慢推近、平稳横移、跟拍、俯拍、过肩或特写切换；不要同时要求推、拉、摇、移。
- 动作优先写连续、可衔接的小动作，减少狂奔、大跳、剧烈翻滚等容易导致画面漂移的动作，除非用户明确要求动作大片。
- 对进出门、靠近/远离镜头、推撞、追逐、摔倒、绕行等高风险运动，优先把文学动作改写为空间锚点、屏幕方向、纵深变化和物理反馈。
- 情绪要外化为可见信号，例如停顿、眼神、嘴角、眼眶、肩膀、手指、呼吸和步伐，不只写抽象情绪。

## 核心写法

- 固定参数单独收束：时长、比例、画面大小、画风、节奏、情绪、光线和声音方向。
- 正文按自然镜头节拍组织：每段先写可拍的镜头句，再在高风险动作里补空间状态、运动方向和稳定性约束。
- 如果用户给原始剧本或对白，先提取可拍内容，再把台词转成身体动作、视线、停顿、表情和镜头关系。
- 台词要标注说话人和语气，放在对应画面段落里；不要让整段视频变成念白。
- 声音设计要和画面同步，分别考虑环境音、动作音效、对白/旁白和 BGM 情绪强度。
- 结尾补稳定性约束：人物身份一致、面部稳定、动作自然、光影连续、材质不闪变、画面无卡顿无水印。

## 空间轨迹写法

- 先给不动锚点，再写主体运动：门框、墙面、桌沿、床边、车门、台阶、灯光边界或画面中心物体都可以作为锚点。
- 多参考图时，明确哪张图锁空间锚点，例如“`@图片1` 锁无人走廊和门框，`@图片2` 锁人物身份”；图片编号只代表职责，不代表时间顺序。
- 用屏幕坐标和纵深替代模糊动词：从画面左侧进入、推向画面右侧墙面、向画面深处退后、沿纵深靠近镜头、被门框遮挡。
- 向内移动要有体型变小、被前景或门框遮挡、走向深处光区等可见证据；向外靠近要有体型变大、遮挡背景、脚步和呼吸更近等可见证据。
- 光影可作为空间边界：冷色阴影是起点，暖光亮区是终点，人物跨越明暗交界线后不要再反向漂回。
- 动作戏按“预备 -> 爆发 -> 阻尼”写：重心偏移、脚尖转向或肩膀下沉作为起势；主位移只保留一个；结尾用发丝、衣袖、纸张、道具晃动或碰撞反弹证明惯性。
- 同一个 3 秒片段不要同时写“转身 + 走远 + 回头 + 说话”；拆成先原地转身或停顿，再沿 X/Y/Z 方向移动。
- 穿过门、墙、桌子、身体等不透明物体的动作要改成推开、绕过、撞上、停住或被遮挡，并补短约束。
- 高风险动作的反向约束要短而具体，例如“禁止向镜头前方反向移动”“禁止穿过不透明门板”“保持门框和墙面固定不漂移”。

## 自然节拍分镜（Atomic Beat，建议）

直投平台模式即使没有结构化 schema 约束，也建议按"自然节拍"组织 Seedance 2.0 / 多模态视频模型的 prompt，避免单段散文导致的 attention 失焦：

- 单镜头可以按 `SHOT N (起–止s)` 组织，每个 SHOT 先写一句自然镜头句，再用 `镜头：...` 补自然镜头提示。
- 如果进入 Lumina 管线结构化模式，景别/运镜应写进 `cameraHint` 的自然短语，不要把它们当固定枚举字段。
- 时间切片服从镜头节奏：≤6s 通常 1-3 段；7-10s 通常 3-4 段；11-15s 通常 4-5 段。
- 每个 SHOT 只引用本段真正有帮助的 `@图片N` 锚点；特写优先人物/道具，大全景优先场景，需要光线或材质时可以例外。
- 风格形容词放在固定参数段；SHOT 内保留主体、动作、视线、口型、情绪落点和必要空间信息。
- 不为了满足格式而硬拆动作；只有多个动作互相抢权重时才拆成连续两段 SHOT。

直投示例（8s 大门戏）：

```text
画风：国风3D二次元都市超自然惊悚 | 时长：8s | 比例：16:9 | 画面大小：720p

锚点：
- @图片1 = 场景:幸福小区大门
- @图片2 = 人物:小美 身份/服饰
- @图片3 = 道具:门禁红灯

总览：从冷色门外推近到唇部特写，等待→试探→诱导。

SHOT 1 (0.0–2.0s)
镜头：全景，缓慢推向铁门。
参考：@图片1 + @图片3。冷蓝夜色下的小区门口空旷安静，半掩铁门和闪烁门禁成为画面中心。

SHOT 2 (2.0–5.5s)
镜头：中近景固定。
参考：@图片2 + @图片1（主：@图片2）。小美停在铁门外，双手抱胸微微前倾，先看保安室玻璃，再把视线勾向玻璃后的人影。
台词1：小美（甜腻）：「小哥哥，我是三栋404的业主，忘带门禁卡了。」

SHOT 3 (5.5–8.0s)
镜头：大特写，缓慢推到唇边和指尖。
参考：@图片2。镜头推到唇边和指尖，小美食指轻点红唇，笑意加深后留下半拍停顿。
台词2：小美（妖媚）：「你想做什么，都可以。」

声音：环境=远风+门禁电流；动作=高跟鞋+玻璃后呼吸；BGM=轻喜悬念 low
约束：人物身份/服装一致 / 面部稳定 / 明暗交界线不漂移 / 无字幕水印
```

## HDVP 矢量化写法（高风险镜头建议）

针对位移、碰撞、穿行、跨光影区等容易"空间倒置/穿模/方向反转"的高风险镜头，直投 prompt 也建议采用 HDVP 三层写法：

- **空间锚点**：以 `@图片N` 中的门框、墙面、桌沿、明暗交界线作为不动点，画面内其它主体围绕该锚点运动。
- **运动矢量（SDV）**：用 `<起点位置> → <X+/X-/Y+/Y-/Z+/Z-> → <终点位置>` 替代"进门/退后/躲开"等模糊动词。
  - X+ = 向画面右；X- = 向画面左
  - Y+ = 上升；Y- = 下沉
  - Z+ = 向画面深处（远离镜头）；Z- = 靠近镜头
- **物理反馈**：碰撞、撞墙、扑倒、压制等动作才补可见反馈（发丝/衣料/纸张/反弹/光影变化），证明运动质感。
- **光影深度墙**：跨越冷暖区时显式声明"冷色起点 → 矢量 → 暖光终点；明暗交界线不漂移"，并在约束里加"明暗交界线稳定"。
- **主图降噪**：多 ref 时声明"主参考 @图片N"，让模型知道脸/衣服 100% 继承自该图，其它图只作辅助。
- **反向约束**：高风险位移补"禁止位移方向反转 / 禁止穿过不透明物体"等短禁令。

HDVP 句式示例（压墙戏 2.0–4.5s 段）：

```text
SHOT 2 (2.0–4.5s)
镜头：中景轻手持。
空间：以@图片1门框为原点；姜夜起点画面左侧冷色阴影 → X+ → 终点抵@图片1右侧墙面，不可穿透。
姜夜从阴影切入扣住咽喉，手臂把小美推向右侧墙面。
约束：禁止位移方向反转；明暗交界线不漂移。
```

## 输出模式

- 直投平台模式：用户要“可直接复制到 Seedance / 火山引擎 / 平台使用”的最终提示词时，可以把固定参数、公共参考素材、时间轴正文、声音设计和约束条件合并成一份完整 prompt。
- 管线结构化模式：用户要进入 Lumina 工作流、story_to_video、任务 JSON 或编译器链路时，使用 `shotSynopsis + microShots + audioDirectives + constraintHints`；microShots 内以 `visualBeat` 为核心，按需补 `cameraHint/spatialHint/emotionHint/physicalHint/beatRole`，不重复写“台词：”“声音：”“BGM：”栏目。
- 默认按直投平台模式整理当前轮单次视频任务；若上游已明确字段名或编译器会追加台词/声音，改用管线结构化模式。
- 两种模式都要保留同一套镜头逻辑：参考素材职责清楚、时间段动作连续、台词有画面锚点、声音有画面来源、结尾约束短而有效。
- 直投平台模式里也不要重复同一信息：公共参考素材只列一次，地点/情绪/光线只保留在固定参数或首段正文中一处，长画面概述只做短引子，不重复时间轴里的每个动作。

## 参考素材写法

- 面向火山引擎 / Seedance 平台直用的提示词，统一使用官方引用命名：`@图片1`、`@图片2`、`@视频1`、`@音频1`。不要混用 `@图1`、`@img1`、`参考图1`、`视频一` 等非官方写法。
- 如果运行时只提供“当前上传图片编号”，输出时仍要把它们转成平台可读的 `@图片N` / `@视频N` / `@音频N`，并逐一说明职责。
- 纯文本生成：直接写主体、动作序列、环境光影、镜头语言、声音和风格。
- 首帧图生视频：明确当前上传图作为起始画面，视频从这张图自然动起来，后续动作和镜头延展要与首帧构图一致。
- 首尾帧：明确起始画面和结束画面，中间动作要自然过渡，不突然换身份、换空间或换画风。
- 多参考图：逐张说明职责，例如 `@图片1` 锁人物身份，`@图片2` 锁场景空间，`@图片3` 锁产品材质，`@图片4` 锁姿态或特写；不要把参考图当逐帧动作。
- 如果参考图承担空间锚点，要写清锚点不可漂移，例如门框、墙角、桌沿、产品台面、车门边缘、明暗分界线保持固定。
- 图片生成提示词和视频提示词要分层：图片生成阶段可以完整描述“生成什么图”；视频阶段只保留短职责绑定，例如“参考 `@图片1` 的无人街道空间，参考 `@图片2` 的人物身份，参考 `@图片3` 的道具形态”。不要把每张图片的完整生成说明塞进视频正文。
- 多图职责绑定可以写“参考素材只锁定身份、场景、道具和关键证据，不代表画面出现顺序”；避免写“按 `@图片1` 至 `@图片4` 的顺序读取参考”这类容易被误解成逐帧播放的表达。
- 如果原文同时有“`@图片N：生成/锁定...`”和“参考: 图片N=...”，最终直投 prompt 只保留一个“公共参考素材”段；图片生成说明留给素材阶段，不和视频正文重复输出。
- 素材数量要审计职责重叠：身份图 + 当前状态图可以共存；当前状态图已清楚包含手持物时，额外道具图只在该道具是关键笑点、产品或容易变形时保留。接近 6 张以上时说明为什么每张不可合并。
- 参考视频：只在用户确实上传视频或明确要求时使用，写清参考的是运镜、动作、节奏、特效、剪辑还是音色，不要模糊写“参考视频整体感觉”；例如“参考 `@视频1` 的手持跟拍节奏”。
- 音频参考：写清参考的是 BGM 节奏、音色、环境声、旁白质感还是卡点节拍；例如“BGM 气质参考 `@音频1`，但动作音效按画面重新生成”。
- 素材多时，先列“公共参考素材”职责，再写时间轴正文，避免正文里反复解释同一张参考图。

## 场景类型策略

- 短剧/对白：画面和台词绑定，台词由动作和情绪触发；优先中景、近景、过肩、轻推镜等稳定镜头。
- 仙侠/奇幻：写清法阵、能量、粒子、武器、衣袍和光效的运动逻辑；特效服务动作，不遮挡人物脸。
- 产品/广告：写清产品外观、材质、展示角度、主光、反光、操作动作、卖点和声音节奏；避免人物或背景抢主体。
- 真人口播/带货：优先近景或中景，明确人物手持产品、视线、表情、中文口播、背景和稳定面部约束。
- MV/音乐卡点：明确节拍、切换点、动作卡点、光影变化和参考音频/视频节奏；不要只写“跟音乐卡点”。
- 一镜到底：明确全程不切镜头，写清空间路径、遮挡转场、主体移动和镜头跟随关系。

## 时长与节奏

- 4-8 秒：适合一个动作、一个产品展示、一个氛围镜头或一个短转场，避免塞入复杂剧情。
- 9-12 秒：适合一个完整短场景，可拆成开始、推进、落点。
- 13-15 秒：适合完整微叙事，建议拆成 3-4 个时间段，每段只承担一个主信息点。
- 多段任务之间要有清楚的衔接画面，但当前任务不负责自动把上一段视频上传为下一段延长输入。
- 节奏可以写 slow、medium、fast，但必须落到镜头长度、动作密度、切换频率和声音强度上。
- 动作密度按时长收敛：6 秒内最多 2-3 个主镜头重心，15 秒内最多 3-4 个主信息点。动作戏允许奔跑、追逐、撞击，但不要同时堆多次切镜、复杂走位、多个命中结果和额外反应。

## 画面与声音

- 光影要服务情绪和空间连续性，例如居家常光、清晨暖光、夜晚冷光、雷光、金光、霓虹反射、侧逆光或柔光补光。
- 当光影承担空间边界时，写清冷暖区、明暗线或投影边缘的起点和终点职责，避免空间在运动中折叠。
- 动态模糊、景深和特效要克制，保证主体清晰、动作可读、面部稳定。
- 环境音写真实空间质感，例如脚步、衣料、呼吸、风声、水声、室内底噪、雷声、金属碰撞。
- BGM 写情绪和强度，例如悬念轻喜感 low、史诗温柔 high、紧张鼓点 mid；不要只写“好听的音乐”。
- 需要无字幕时要明确无字幕、无文字、无 LOGO、无水印；需要台词时也不默认生成屏幕字幕。
- 声音去重后再输出：同类环境声、动作声和 BGM 只保留一次；声音设计要短写成“环境音 + 动作音 + BGM 转折”，不要把同一音效换词重复。
- 画外音如果有明确台词，才进入台词栏并用引号写原文；如果只是“楼上传来冷女声、内容下一镜揭示”，不要写成引号台词，改写成声音事件和人物反应。
- postlap 或下一镜承接声音要写清“内容不清晰 / 具体台词留到下一镜”，并在当前画面安排听者停顿、抬眼、僵住或回头，不让模型念出说明文字。

## 压缩与反例过滤

- 最终 prompt 删除导演表格字段：叙事目的、关键确认、黄金三秒、镜头意图、情绪解释、观众会理解什么。
- 最终 prompt 删除过细摄影参数：焦段、光圈、镜头型号、f 值、毫米数；除非用户明确要仿摄影方案。
- 保留可生成信息：时间、自然镜头提示、主体、可见动作、场景、光线、台词位置、声音事件和稳定性约束。
- 保留空间轨迹信息：静态锚点、起点、终点、屏幕方向、纵深变化、遮挡/碰撞反馈；删除抽象心理解释和无法拍出的运动意图。
- 如果原文像 shot list 或导演案，先提炼成 Seedance 时间分镜，不原样保留栏目堆叠。

## 成品 prompt 清理顺序

1. 先判定输出模式：直投平台还是管线结构化。
2. 合并参考素材：只保留一个公共参考素材段，删除“按图片顺序读取参考”。
3. 合并画面正文：短引子 + 时间轴；若长引子和时间轴重复，保留时间轴。
4. 校验台词：只有角色真实说出口的内容进入引号；描述性画外声改成声音事件。
5. 声音去重：环境音、动作音、BGM 各保留一次，重复词删掉。
6. 压动作密度：按时长删掉次要动作或拆镜，保证每段一个主镜头重心。
7. 对高风险运动补空间审计：锚点是否固定，方向是否唯一，是否有纵深证据，是否避免穿模和反向移动。
8. 最后补短约束：身份、面部、动作、材质、无字幕/文字/LOGO/水印。

## 质量标准

- 视频目标明确，能直接下发生成。
- 主体、动作、场景、运镜、声音、台词和时长不互相冲突。
- 参考素材职责清楚，没有把人物图当场景图、把场景图当逐帧动作或把风格图当身份图。
- 每个时间段都有明确镜头重心，动作之间有自然过渡。
- 人物身份、服装、面部、空间、材质、光影和声音连续稳定。
- 高风险运动有明确锚点、起点、终点、方向、纵深变化和惯性/碰撞反馈，不出现空间倒置、穿模或运动逻辑反向。

## 失败回退

- 素材不足时，优先整理纯文本可执行版本，再指出缺口。
- 用户只给抽象创意时，先补主体、场景、动作、运镜、光影、声音和时长。
- 用户要求过长或过复杂时，拆成多个独立短视频任务，并说明每段的画面衔接点。
- 运动方向含混时，优先补静态锚点和屏幕坐标；仍然冲突时，删掉次要动作或拆成两个时间段。
- 出现穿模风险时，把“穿过”改成“推开、绕过、撞上、停住、被遮挡”，并补不透明边界约束。
- 不偷偷引入项目资产、历史视频或流程层上下文。
