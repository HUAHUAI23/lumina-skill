## 前置声明
本文件是 `screenplay` skill 的完整执行规范。  
收到任务后，必须先完成模式路由与输入补齐，再按主流程执行（STEP 1 -> STEP 9），扩展步骤按触发条件执行。  
禁止跳步；禁止先出成稿再补分析。

---

## 0. 路由与输入契约（先执行）

### 模式路由
- `MODE_NOVEL_TO_SCRIPT`：小说/故事改编拍摄剧本
- `MODE_AD_SCRIPT`：广告创意脚本（含 CTA）
- `MODE_MOTION_COMIC`：动态漫脚本（含分层、运镜、音效）
- `MODE_IMAGE_TO_SCREENPLAY`：根据图片/画面意向生成剧本
- `MODE_SCRIPT_POLISH`：已有剧本改写优化

### 快速触发（首轮默认包）
- 若用户仅说“开始 / 需要 / 先来一版 / 给我首轮”，直接触发 `MODE_FIRST_ROUND_SCREENPLAY_BUNDLE`。
- 该模式不追问，先给“首轮默认四套剧本包”，并显式标注假设。

### 输入契约（缺失即补问，或做假设并标注）
1. 题材类型（爱情/悬疑/都市/玄幻/历史/商业）
2. 目标平台与画幅（竖屏短视频/横屏长视频）
3. 单集时长（60s/3min/5min 等）
4. 角色清单（姓名、外貌、身份、性格、关系）
5. 任务目标（还原原著 / 强节奏改编 / 投放转化）
6. 输出偏好（纯拍摄剧本 / 动态漫标准格式 / 广告脚本）
7. 审核与尺度边界（品牌合规/平台审查/年龄限制）
8. 参考素材（原文、图片、现有脚本、分镜意向）

### 提示词工程优化挂件（内置）
1. 分解生成：先结构后文案，避免一次性大段直出
2. 角色分工：`Planner -> Writer -> Editor -> QA`
3. Rubric 评分：每轮对结构、节奏、可拍性、格式合规打分
4. 迭代修订：未达标则仅重写低分段落，不全量重写

### 提示词封装规范（用于稳定输出）
1. 指令前置：角色、任务、输出格式、硬约束必须放在输入最前。
2. 上下文分隔：原文素材与执行指令使用清晰分隔（如 `###` 或结构化标签）。
3. 样例锚定：至少给 1 个最小格式样例，降低格式漂移。
4. 长文顺序：长原文在前、问题与指令在后，减少长上下文丢失。
5. 参数建议：结构化写作优先低随机度（温度低），创意发散再提高随机度。

## STEP 0.5｜首轮默认四套剧本包（仅快速触发时执行）

**目标**：在缺少细节时先给可拍可改的剧本草案，提高首轮效率。

**固定输出四套：**
1. `MODE_NOVEL_TO_SCRIPT`：1 幕 2 场的小说改编样例（严格 `△` 与对话格式）
2. `MODE_AD_SCRIPT`：15s 广告剧本样例（含 A/B Hook + CTA）
3. `MODE_MOTION_COMIC`：1 集 3 场动态漫样例（含音效/运镜/分层）
4. `MODE_SCRIPT_POLISH`：同段落“原稿 -> 优化稿”对照样例

**默认假设（需标注）：**
- 题材：都市情绪 + 轻悬念
- 画幅：竖屏 9:16（广告/动态漫）与横屏 16:9（拍摄剧本可切换）
- 受众：18-35 岁短视频用户
- 平台：抖音/视频号优先

**首轮包交付要求：**
1. 每套都要附“可替换变量清单”（题材、人设、平台、时长、风格）。
2. 每套都要附“可拍清单”（场景、关键道具、核心音效）。
3. 广告与动态漫样例必须包含 A/B 钩子。

**质量检查点：**
> 四套是否都可直接复用，且都遵守对应格式硬约束？

## STEP 1｜原文解构

**目标**：读懂素材，提取可视化信息与冲突主线。

**执行方法：**
1. 列出关键事件序列（只保留有因果关系节点）。
2. 给每个事件标注情绪温度：`冷静 / 紧张 / 爆发 / 余震`。
3. 标记“视觉盲区”（内心活动、时间跳跃、背景说明）。
4. 对每个盲区给出转化方式：`OS / 回忆 / 道具特写 / 对话隐喻`。
5. 标记天然高光帧（封面级画面、冲突峰值瞬间）。

**质量检查点：**
> 是否能用一句话说清“本段最激动人心的瞬间”？

---

## STEP 2｜情节结构规划

**目标**：搭建三幕结构与情绪曲线。

**执行方法：**
1. 按结构位规划剧情：
   - 第一幕（铺垫）：角色、关系、冲突前景
   - 诱发事件：打破平衡，进入主冲突
   - 第二幕（对抗）：障碍升级，信息揭露
   - 中点反转：认知变化或立场逆转
   - 高潮：最大冲突 + 最大情绪
   - 第三幕（决议）：余震处理 + 悬念收口
2. 若为连载，结尾必须植入 Cliffhanger。
3. 输出情绪曲线（低 -> 高 -> 回落 -> 再上冲）。

**广告分支（MODE_AD_SCRIPT）补充：**
1. 结构优先采用 `Hook -> Problem -> Solution -> Proof -> CTA`。
2. 3 秒内必须出现钩子（冲突画面或反常识台词）。
3. CTA 必须可执行（如“点击领取/立即咨询/下滑查看”）。

**质量检查点：**
> 情绪曲线是否存在明显起伏，而非一条水平线？

---

## STEP 2.5｜节奏模型选择（SCU / N1 / N2）

**目标**：把节奏从“感觉”变为“可执行模板”。

### SCU（爽点循环单元）
1. 压力：压迫/冲突建立
2. 引燃：主角或关键线索触发
3. 爆发：反转/解决/打脸兑现
4. 发酵：后果展示与情绪回味

### N1 高频模型（转化与追更优先）
- 0-8 秒：黄金钩子
- 8-40 秒：SCU 循环 I
- 40-75 秒：SCU 循环 II
- 75-90 秒：留扣（悬念或 CTA 前截断）

### N2 过山车型（传播与情绪深度优先）
- 0-15 秒：快速抬升
- 15-45 秒：危机深潜
- 45-80 秒：绝地反弹
- 80-90 秒：悬念坠崖

**选择建议：**
- `MODE_AD_SCRIPT` 默认优先 N1。
- `MODE_NOVEL_TO_SCRIPT` 与 `MODE_MOTION_COMIC` 可按章节目标切换 N1/N2。

**质量检查点：**
> 是否至少完成 1 个完整 SCU，且结尾仍保留下一步驱动力（悬念或 CTA）？

---

## STEP 3｜场景拆分与场景卡

**目标**：把结构转为可执行场景列表（不是最终剧本正文）。

**执行方法：**
1. 按连续时空拆场：地点或时间变更即换场。
2. 场景数量参考：
   - 60 秒：8-12 场
   - 3 分钟：20-30 场
   - 5 分钟：35-50 场
3. 每场先输出“场景卡”：

```text
场景 [编号]
目的：推进情节 / 塑造人物 / 营造氛围
情绪温度：冷静 / 紧张 / 爆发 / 余震
核心视觉：本场最重要的可视化瞬间
出场角色：角色名单
承接关系：上一场结束于__，本场开始于__
```

**质量检查点：**
> 相邻场景是否自然衔接？同温度连续场是否需要过渡镜？

---

## STEP 4｜视觉语言设计

**目标**：为每场确定景别、运镜、分层需求。

**景别选择逻辑：**
- 全景（LS）：空间建立、开场说明
- 中景（MS）：人物互动、常规叙事
- 近景（MCU）：情绪推进
- 特写（CU）：线索与细节
- 大特写（ECU）：极端情绪节点

**运镜选择逻辑：**
- 推镜：压迫感、揭露真相
- 拉镜：孤立感、格局放大
- 横移：跟随、巡视
- 上摇：威压登场
- 下摇：由人到物的线索强调
- 固定：沉默对峙与震惊瞬间

**分层标注（动态漫必须）：**
- 头发/衣物飘动
- 眼球/眼睑眨动
- 口型层（若配音）
- 武器道具独立层
- 粒子层（雨雪烟尘火星）

**质量检查点：**
> 该场核心帧能否在 0.5 秒传达情绪？

---

## STEP 5｜对话精炼

**目标**：台词短、准、可演，角色声音可区分。

**执行方法：**
1. 砍字原则：每句先写完整，再删减约 30% 冗余。
2. 潜台词原则：明说信息最少，情绪与动机靠语境补足。
3. 角色一致性：每个角色维持稳定句式和语气。
4. 旁白限制：仅用于不可视化关键信息，且每处不超过 20 字。

**角色台词参考：**
- 强势角色：短句、命令式、少疑问
- 弱势角色：停顿、省略、句未尽
- 智慧角色：反问、话中有话
- 爆发时刻：单字、断句、中断

**质量检查点：**
> 去掉任一句台词会不会完全不影响理解？若是，删掉。

---

## STEP 6｜节奏与钩子设计

**目标**：保证开头抓人、过程有梯度、结尾可续看。

**开篇黄金 3 秒（至少选一种）：**
- A：视觉冲击开场（高冲突画面 + 强 SFX）
- B：悬念台词开场（一句想知道后文的话）
- C：动作开场（无台词，纯动作音画）

**节奏工具：**
- 快切（2 秒内换镜）：混乱、紧张、动作感
- 长停（5 秒+）：压迫、沉默、悬念
- 突然静音：转折与震惊
- BGM 变速：情绪换挡

**结尾 Cliffhanger 类型：**
- 问题悬停
- 危机揭露
- 身份反转
- 时间压迫

**质量检查点：**
> 最后一秒是否足够让观众点击下一集？

---

## STEP 6.5｜平台适配与版本化输出

**目标**：同一剧情生成多平台可投放版本，减少二次改稿。

**执行方法：**
1. 抖音/快手：优先 9:16，前 3 秒冲突前置，台词更短。
2. 视频号：信息更清楚，旁白可略增，节奏中速。
3. B站/YouTube：可保留更多铺垫与世界观信息。
4. 每次交付至少提供“主版本 + 轻改版本”两档：
   - `Fast Cut`：短句快切，冲突密度高
   - `Story Cut`：信息更完整，情绪停顿更长

**质量检查点：**
> 是否已明确标注“平台、画幅、时长、版本差异”？

---

## STEP 7｜音效与配乐标注

**目标**：每场都具备可制作的声音设计。

**标注规范：**
- `[BGM]`：背景音乐情绪（紧张/温柔/压迫/反转/史诗）
- `[BGM-STOP]`：音乐突停（重大揭露前后）
- `[SFX-环境]`：风雨、人群、空调、电流等环境声
- `[SFX-动作]`：脚步、拔剑、关门、撞击
- `[SFX-情绪]`：心跳、耳鸣、低频轰鸣
- `[配音-语气]`：冷静/颤抖/咬牙/大喊/压声

**质量检查点：**
> 是否存在“有画面但无声音设计”的空场？

### A. 拍摄剧本格式（MODE_NOVEL_TO_SCRIPT / MODE_SCRIPT_POLISH 默认）

**硬约束：**
1. 幕标题必须是 `【第X幕：标题】`
2. 场景头必须是 `内/外。地点 - 时间`
3. 动作描写必须以 `△` 开头
4. 对话格式必须是 `角色名（情绪/动作/潜台词）“台词”`
5. 心理活动改为 `(OS)` / `(内心OS)` 或 `△ 回忆：`
6. 可加入镜头词：`△ 镜头特写：`、`△ 镜头推进：`、`△ 镜头缓缓拉高：`

**改编逻辑（必须体现）：**
1. 开场先 `△` 环境置景（光影/声音/气味）
2. 感官转化为动作反应（刺痛 -> 蜷缩、皱眉、捂鼻）
3. 大段背景用 `△ 回忆` 或 `OS` 前置交代
4. 推理线索要给关键道具特写

**输出模板：**
```text
【第X幕：标题】
内/外。地点 - 时间
△ 环境描写（光影、声音、氛围）。
△ 角色（年龄，状态）动作与神态。

角色A（情绪/动作）“台词。”
△ 角色A反应动作。
角色B（潜台词）“台词。”

△ 镜头特写：关键物件细节。
角色A（OS）“内心或画外音。”
```

### C. 广告剧本格式（MODE_AD_SCRIPT）

**广告脚本必须包含：**
1. 品牌/产品/目标受众
2. 卖点（USP）可视化
3. 开场钩子
4. 明确 CTA
5. 画幅与投放平台约束
6. 至少提供 2 个开场 Hook 变体（A/B），便于投放测试

**输出模板：**
```text
【广告脚本】标题
时长：6s / 15s / 30s / 60s
平台：抖音 / 快手 / 视频号 / B站 / 小红书
画幅：9:16 / 16:9 / 1:1
目标受众：__
核心卖点（USP）：__
开场钩子：__
转化动作（CTA）：__

【场次 01】
内/外。地点 - 时间
△ 视觉钩子画面（0-3秒，必须强冲突或强反差）。
△ 镜头与动作描述。
[BGM] __
[SFX-动作] __
角色A（[配音-语气]）"__"

【场次 02】
...

【收口】
△ 产品/品牌露出与记忆点强化。
△ CTA 画面与口播同步。
旁白（OS）"立即____。"
```

**广告优化补充：**
- 输出末尾追加 `A/B Hook 变体`（每个 1-2 句）。
- 若用户要求转化优先，先保留卖点密度再做文案修辞美化。

### D. 看图转剧本格式（MODE_IMAGE_TO_SCREENPLAY）

**适用条件：**
- 用户上传图片或给出明确画面意向，要求改编为可拍剧本。

**执行规则：**
1. 先列“可见事实”，再列“合理推断”，禁止越界脑补为硬事实。
2. 输出至少 2 场连续场景（含承接关系）。
3. 每场必须含 `△` 动作、至少 1 句对话或 OS、至少 1 个镜头指令。

**输出模板：**
```text
【看图转剧本】
可见事实：
1. ...
2. ...

合理推断（已标注）：
1. ...
2. ...

【第一幕：__】
内/外。地点 - 时间
△ 基于可见事实的环境与动作描述。
角色A（情绪）“台词。”
△ 镜头特写：与图片强相关的道具/表情。

【第二幕：__】
内/外。地点 - 时间
```

## 常见错误与修正

| 错误类型 | 表现 | 修正方法 |
|---|---|---|
| 视听冗余 | 旁白重复画面内容 | 删除旁白，交给画面 |
| 景别单调 | 全篇几乎都是中景 | 情绪峰值强制改特写/大特写 |
| 台词过长 | 单句超过 20 字且信息重复 | 拆句并删除重复信息 |
| 钩子失效 | 开场平铺、结尾无悬念 | 前 3 秒冲突化，末尾问题化 |
| 特效滥用 | 每场都抖动/速度线 | 仅在峰值场次使用特效 |
| 角色失真 | 台词读不出角色差异 | 给每个角色固定 2-3 个句式习惯 |

## 快速参考：情绪 × 景别 × 运镜 × 音效

| 情绪目标 | 推荐景别 | 推荐运镜 | 推荐 SFX |
|---|---|---|---|
| 震慑/威压 | 仰拍全景 -> 近景 | 缓慢推镜 | 低频轰鸣 |
| 恐惧/脆弱 | 大特写（眼部） | 静止/轻抖 | 心跳 + 耳鸣 |
| 愤怒爆发 | 近景 -> 大特写 | 快速推镜 + 抖动 | 爆破重击 + BGM-STOP |
| 深情/温柔 | 中近景柔光 | 缓慢横移/固定 | 轻柔弦乐 + 风声 |
| 反转揭露 | 特写 -> 拉全景 | 拉镜 | 突然静音 + 重音 |
| 追逐逃亡 | 快切多机位 | 横移 + 手持晃动感 | 急促脚步 + 喘息 |
