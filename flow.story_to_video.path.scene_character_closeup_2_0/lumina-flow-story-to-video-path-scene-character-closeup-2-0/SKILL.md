# story_to_video path scene character closeup 2.0

## 路径定位

- 本路径面向 Seedance 2.0 / 2.0 类多模态视频模型。
- 本路径不是导演总规范入口；剧情弧线、暗线、穿插叙事、台词节奏和镜头功能已由上游剧本、故事板和 shot_split 决定。
- 本路径只负责把已确认的镜头意图落成可执行的参考图组合、必要局部特写和短视频提示词。
- 主路径是参考图生视频，不是先把人物和场景溶成普通首帧再生视频。
- 本路径和 blend_2_0 的区别：不追求完整主分镜溶图，只用角色图、场景图和少量特写锁定事实，让视频模型完成自然同框和表演。
- 图片负责锁定稳定事实：人物身份、服装、无人场景空间、关键道具、必要局部特写。
- 文本负责轻量控制：谁在什么场景、主要动作方向、主运镜、情绪落点和声音气质。
- 提示词要短、清楚、可拍，给模型自然发挥空间；过度精确会限制表演，制造噪点、僵硬动作和参考图拉扯。
- 轻量不等于流水账；先给出一段有镜头连续性的“画面与动作”叙事，再用少量时间节拍控制重心。

## 输入职责模型

- 文本：定义主题、动作方向、主镜头、节奏、光线、声音和情绪落点，不微管逐帧姿态。
- 人物图：锁定人物身份、脸、发型、年龄阶段、体态、服装，不承担场景。
- 场景图：锁定空间布局、时间、光线、材质和陈设，必须尽量无人。
- 特写图：锁定某一步必须看清的局部证据或转折点，例如眼神、嘴唇、手、剑身、文件袋、伤口、道具归属、情绪矛盾爆发瞬间。
- 参考图不是连续分镜播放顺序，而是同一个视频镜头的约束集合。

## 视频模式规则

- 默认优先 reference_images。
- first_frame 只在参考图生视频不可用、或确实只有一张可独立承接动作的图时使用。
- first_last_frame 极少使用；只有结尾硬结果不可拆、且不锁尾帧会造成剧情误读时才用。
- text_to_video 只用于纯空镜、抽象过场或没有任何可用参考图的低风险镜头。
- 不把“场景图 + 人物图 + 人物图”规划成普通双人同框首帧溶图。

## 分镜图规划总则

- 每个镜头先判断需要哪些参考职责，而不是先生成一张大而全的同框图。
- 标准组合是：无人物场景图 + 主要人物干净图 + 次要人物干净图 + 必要步骤特写；没有必要特写时不要硬补。
- 普通对话、同框站位、轻微走位、表情变化、推镜、回头、坐下、递物，交给视频提示词语言控制。
- 只有观众必须看清、主画面自然生成不可靠，或情绪/关系矛盾需要明确转折点时，才规划特写图。
- 每张图只解决一个主要风险：身份、服装、空间、道具、动作局部、表情局部、状态证据、转折点。
- 不规划动作过程图，不把每个时间片变成一张图；复杂蒙太奇只保留现实场景、关键人物、关键道具和最强情绪记忆锚点。

## 人物图规范

- 人物图必须是干净底图：单人、无背景或极简中性背景、无其他角色、无复杂场景陈设。
- 人物图可以通过图生图或溶图生成，用来更新服饰、发型、年龄阶段、伤势、仙侠/现代状态。
- 即使人物图需要参考场景的光线、时代或服饰，也只能输出干净人物底图，不把人物放进具体客厅、天门、街道等场景。
- 多角色镜头中，每个关键角色应有独立干净人物图，避免身份串位。
- 人物正侧背三视图只用于身份、服装和轮廓，不要求视频复刻三视图排版。
- 不用文生图生成已绑定人物的新脸；已有角色必须从绑定人物图出发做图生图或溶图。

## 场景图规范

- 场景图必须优先是无人物场景图。
- 场景图负责空间布局、景别基底、时间、光线、家具/建筑/道具陈设，不负责人物动作。
- 已有场景绑定图时，不要文生图重造场景；需要夜晚、破败、冷光、战损等状态变化时，用场景绑定图做图生图覆写。
- 场景图中如果原本有人，应在规划中要求去人化或重新生成无人物版本。
- 场景图和人物图不要溶成普通同框图；视频阶段用参考图和语言把人物放回场景。
- 场景一致性优先来自已有场景图，不能靠文字凭空生成另一个相似空间。

## 特写图规范

- 特写图只补必要步骤，不补普通过程。
- 必要特写包括：眼部反应、嘴唇吐字、捏袖口的手、擦剑反光、文件袋、玄天玉、伤口、血迹、符纸、门牌、道具归属、状态材质、情绪矛盾突出时的转折点。
- 转折点特写用于关系或情绪发生不可逆变化的瞬间，例如玩笑卡住、眼神从回避落向对方、手指停止动作、呼吸停顿、笑意消失、恐惧落下。
- 转折点特写必须对应一个具体可见局部，不写成抽象“悲伤特写”或“震撼氛围图”。
- 特写图可以用人物图、道具图、场景图做图生图或溶图，但输出必须仍是单一局部特写。
- 特写图不同时承担人物全身、场景全貌和多个道具。
- 道具或部位特写需要场景一致性时，可以参考场景图的光线和材质，但不要把整个场景塞进特写。
- 准确中文文字不作为生成成功条件；文件、纸张、牌匾只生成材质、留白、模糊痕迹或可后期贴字区域。

## 溶图规则

- 允许溶图生成干净人物图：例如用人物绑定图 + 服装/状态参考，输出无场景的当前阶段人物底图。
- 允许溶图生成必要特写：例如人物手部 + 道具 + 场景光线，输出手握道具特写。
- 多图溶成特写时，提示词开头必须先写清图片职责：`图片1` 锁人物或手部身份，`图片2` 锁道具形态，`图片3` 锁场景光线或材质。
- 特写溶图的输出目标必须是单一局部证据，例如手握玉佩、嘴唇吐字、眼神落下、血迹滴落；不要输出拼贴图、过程组图或普通同框镜头。
- 允许用场景绑定图做图生图生成无人场景状态图。
- 禁止溶图生成普通双人中景、普通同框首帧、普通站位建立、普通对话首帧。
- 禁止把人物图和场景图溶成“一张看起来完整”的主图后再视频生成；这会让参考图路径退化为旧的首帧路径。
- 有可用绑定图时不要用文生图替代；文生图只在完全缺失且不断镜必须兜底时使用。

## 图片提示词写法

- 图片提示词使用完整自然语言，不写关键词标签串、权重括号、SD 风格参数或 provider 私有语法。
- 先写图片用途，再写画面内容；每张图只服务一个主要职责：干净人物、无人物场景、道具/部位特写、状态变体或缺失场景兜底。
- 文生图公式：主体 + 动作或静态状态 + 环境 + 构图/景别 + 光影/色调 + 材质细节 + 风格/比例。
- 图生图和溶图公式：参考图职责 + 保留什么 + 改变什么 + 输出目标 + 构图/光影/材质 + 禁止项。
- 多图输入必须在开头写清：`图片1` 负责身份或场景，`图片2` 负责服装/道具/姿态，`图片3` 负责光线/材质/局部证据；不要写“参考上图”“保留图里的人”。
- 人物一致性要写具体：保持同一脸型、五官比例、眼距、鼻形、下颌线、年龄阶段、发型和稳定标志物；只改变本次需要的服装、伤势、表情或状态。
- 人物状态编辑要隔离“不变”和“可变”：身份、脸和体态不变；服饰、战损、发光、泪痕、红眼、发丝凌乱等按剧情变化。
- 表情图不要只写“开心/悲伤/震惊”，要写可见变化：嘴角、眼眶、眉头、瞳孔、呼吸、泪痕、肌肉紧绷；同时强调不换脸、不美化、不改年龄。
- 无人物场景图必须写清空间布局、时间、光源、色调、材质、陈设和空镜氛围；禁止出现人物、人体剪影或无关角色。
- 缺失场景兜底图只作为镜头级临时关键帧，不沉淀成项目资产；提示词要明确“无人场景”或“临时场景锚点”。
- 特写图必须写成单一局部：Macro lens shot 或特写构图，主体占画面主要区域，背景只保留必要光线和材质；不要同时生成全身、全景和多个道具。
- 道具、纸张、牌匾、文件等不要依赖准确中文小字；可生成材质、留白、模糊痕迹或后期贴字区域。
- 图片提示词末尾可加短约束：人物身份一致、无多余人物、无拼贴、无文字水印、构图干净、主体清晰。
- 自检：图片是否只解决一个风险？参考职责是否清楚？保留/变化是否分开？是否避免了同框主图、过程帧、拼贴和无关装饰？

## 图片数量和顺序

- Seedance 2.0 支持最多 9 张图片；本路径常规使用 2-4 张，复杂镜头优先控制在 5-6 张。
- 接近 7 张以上时先检查是否应拆镜、合并参考职责，或改用 blend_2_0；不要把 reference_images 变成资产清单。
- 图数由职责决定，不为了凑数，也不为了少图丢掉关键人物、关键证据或关键转折点。
- 推荐顺序：无人物场景图在前，主要人物干净图随后，次要人物干净图再后，关键道具或步骤特写最后。
- 每张图片说明只写一个短职责，例如“图片1锁定无人客厅冷光空间”，“图片2锁定林照月白衣女修身份”，“图片4锁定文件袋特写”。
- 不使用“上图、图1、参考图1、Reference image 1”等混乱称呼；统一使用“图片1、图片2、图片3”。

## 分镜规划工作法

1. 先判断镜头主题和运动：谁、在哪里、动作从哪里开始、镜头怎么动、情绪落到哪里。
2. 再列参考职责：需要哪些干净人物图、哪张无人物场景图、哪些必要特写。
3. 对每个已绑定人物，优先复用或图生图生成干净人物底图。
4. 对每个已绑定场景，优先复用或图生图生成无人物场景图。
5. 只在关键证据看不清，或情绪/关系转折必须被观众看见时补特写图。
6. 选择 reference_images；除非能力不可用或镜头极简单，才退到 first_frame。
7. 最后检查是否误生成了普通同框溶图；如果是，改回人物图 + 无人物场景图 + 必要特写。

## 视频提示词写法

- 用“固定参数 + 画面与动作自然段 + 少量时间节拍”的短格式，不输出冗长栏目。
- visualPrompt 不要重复固定参数；画风、时长、比例、分辨率会由编译器放在开头。
- reference_images 镜头必须能在最终视频提示词里看到参考图职责，例如：`参考: 图片1=无人病房冷光空间，图片2=林照月白衣身份，图片3=玉佩手部特写。`
- 正文只在需要消除歧义时短句绑定：场景以图片1为准，人物以图片2为准，道具或局部证据以图片3为准；不要每个时间段反复点名所有图片。
- 图片编号只代表参考职责，不代表时间顺序；不要写“先出现图片1，再出现图片2”，除非镜头叙事本身确实要切到对应主体。
- 如果输入是原始剧本、对白、旁白或分场文稿，先提取固定参数，再改写成镜头化时间段，不要原样堆剧情说明。
- visualPrompt 开头优先写一段连续的画面叙事：从哪个现实锚点动起来，镜头如何移动，情绪如何递进，最终落到哪个可见画面。
- 时间段只补节拍，不要替代画面叙事；每段按“时间 + 景别/运镜 + 主体 + 主动作 + 场景/光线 + 可见情绪信号”组织。
- 9 秒以上默认拆成 2-3 个时间段；13-15 秒可以拆成 3-4 个时间段。时间段用于控制镜头重心，不对应新增图片。
- 每个时间段只指定一个主动作和一种主运镜，让模型补足自然表演、衣料、发丝、呼吸和微表情。
- 只有关键证据和转折点需要精确，例如手攥护身符、眼圈变红、刀刃反光；普通走位、表情和同框关系不要写得过细。
- 台词和声音由编译器追加；视频正文只为关键台词预留说话动作、停顿、听者反应或情绪承接点，不改写原文台词。
- visualPrompt 不直接写“台词：”“声音：”“BGM：”栏目，但必须让画面里有这些声音的来源和节拍，例如雷击、脚步、衣料摩擦、床沿轻响、呼吸停顿。
- 如果是闪回或蒙太奇，不要平铺每张图；写清现实锚点、转场动机、2-3 个代表性记忆锚点和最终情绪落点。
- 正文按时间写镜头，格式接近：

```text
【画面与动作】
视频从夜晚客厅的无人空间自然动起来，镜头从门口缓慢推进，低噪的室内光线压住空间。沙发上的黑袍男人处于画面深处，身体几乎不动，只在听到画外声时轻轻抬眼；结尾切到他放在膝盖上的手，手指停住，暗示他已经意识到危险。

0.0-4.0秒。夜晚客厅全景固定镜头，空间安静，窗外城市光很远。
4.0-6.0秒。缓慢推镜，黑袍男人在沙发上抬眼，动作克制。
6.0-7.0秒。手部特写固定镜头，手指短暂停住，情绪落到警觉。
```

- 如果是连续对话，也可以写成自然镜头句：

```text
【画面与动作】
镜头从三人同框的中景开始，中间男子先开口，身体微微前倾；左侧女子没有马上回应，先低头停顿半秒，再抬眼看向他，气氛从克制转向对抗。最后镜头回到三人同框，右侧角色沉默旁观，空间里的压力明显变紧。
```

- 对短镜头可用更紧凑格式：

```text
0.0-1.0秒。清晨眼部特写固定镜头，林照月猛然睁开眼，视线落向画外。
1.0-3.0秒。嘴唇特写固定镜头，她短暂停顿后嘴唇微动，情绪落在冰冷决断上。
```

## 台词和声音锚点

- 运行时会把台词和声音拼接到最终 prompt；本路径必须在 visualPrompt 和 audioDirectives 里给它们留足画面动机。
- 对白戏要在画面里安排：说话人开口动作、听者反应、停顿、视线落点、说完后的余韵；不要让最终台词悬浮在画面之外。
- VO 旁白要安排可承载旁白的画面：主角回忆、物件证据、空镜、背影、慢推或凝视；不要让 VO 覆盖无关动作。
- audioDirectives 必须补足三类声音：环境音、动作音、BGM 气质和强度。声音词要具体，如床沿轻响、金属剑身细颤、雷击屏障闷响、衣料摩擦、短促呼吸。
- BGM 气质要服务段落弧线，不只写“温柔/紧张”；优先写成“轻喜悬念”“史诗感中的温柔回收”“压抑后决堤”“灾难快切后收束为心跳”。
- 声音变化要对应画面变化：闪回进入、证据特写、情绪爆点、回到现实，都要有可听的转折来源。

## 视频提示词约束

- 核心写主题 + 运动，不写长篇散文化解释。
- 一段镜头只指定一种主运镜：固定、缓慢推近、轻微跟拍、平稳横移等。
- 动作写方向和过渡，不写逐帧姿态；优先写缓慢、连续、可衔接的小动作，避免把表演锁死。
- 情绪写可见身体信号，例如眼神回避、嘴角僵住、呼吸停顿、肩背不动，不只写“悲伤、愤怒、震惊”。
- 台词只放在对应时间段的关键位置，不把整段画面写成念白。
- 对白、VO、OS 保留为台词信息，不生成字幕或画面文字。
- 声音可短写环境音、动作音和 BGM 情绪；不要把声音扩写成大段说明。
- 结尾稳定性约束保持短：人物身份稳定、动作自然、画面无闪烁、无字幕水印。

## Seedance 方法论落地

- 固定参数和正文分离：画风、时长、比例由编译器放开头，visualPrompt 不重复。
- 画面叙事优先：先写一段连续的画面与动作，再用时间轴校准节拍。
- 主公式是：时间 + 镜头/运镜 + 主体 + 主动作 + 场景/光线 + 可见情绪信号 + 短约束。
- 镜头语言克制：一个时间段一个主景别或主运镜，不同时堆推、拉、摇、移。
- 动作可拍：用停顿、抬眼、侧头、呼吸、手指停住、物品滑落等小动作表达情绪转折。
- 参考图职责明确：每张图只锁身份、空间、道具、特写或转折点，不把参考图当作逐帧动作。
- 语言控制克制：只锁主动作、方向和情绪落点，不追求每秒一动作或精确到每个微表情。
- 台词和声音不能只靠最终栏目；画面必须有对应的口型、反应、动作音来源和 BGM 转折点。
- 约束词简短：只保留人物身份稳定、面部稳定、动作自然、无字幕水印等高收益约束。

## 质量标准

- 规划结果能看出：人物图是干净底图，场景图无人，特写图对应具体步骤、证据或转折点。
- 普通同框关系不依赖溶图，靠参考图 + 语言控制完成。
- 有绑定人物/场景时，不用文生图重造身份或空间。
- 多图溶成特写时，图片1、图片2、图片3各自职责清楚，输出仍是单一局部证据。
- reference_images 视频提示词有清楚的图片职责绑定，不把参考图当连续帧或故事板播放顺序。
- visualPrompt 不重复画风、时长、比例、分辨率等固定参数。
- visualPrompt 有完整“画面与动作”叙事，不只是机械时间段或参考图清单。
- 台词有画面锚点，声音有事件来源，BGM 气质和强度明确。
- 视频提示词短、清楚、有时间和镜头，语言轻控制，不把自然表演逐帧写死。
- 每张图职责明确，不是海报、拼贴、过程帧或完整分镜组。
- 首尾帧极少出现；大多数镜头由 reference_images 或必要时 first_frame 承接。

## 失败回退

- 画面漂移：优先补干净人物图或无人物场景图，不补普通同框溶图。
- 人物串位：为每个关键角色保留独立干净人物图，并在视频提示中写清站位关系。
- 场景不一致：回到已有场景图做无人物图生图，不用文生图造新空间。
- 局部证据或情绪转折看不清：只补一张局部特写，不扩成多张过程图。
- 台词或 BGM 效果弱：不要把更多台词塞进 visualPrompt；补说话/听者反应锚点，并用 audioDirectives 明确环境音、动作音、BGM 气质和强度。
- 输出像时间流水账：先改成一段连续的“画面与动作”叙事，再保留 2-4 个时间节拍。
- 提示词过长：删掉装饰、心理解释、重复形容词、过细姿态和每段重复图片职责，只保留时间、景别/运镜、主体、主动作、场景和情绪落点。
- 画面太僵：减少精确动作链和微表情约束，让参考图锁身份与空间，语言只给运动方向。
