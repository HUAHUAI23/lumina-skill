# story_to_video path scene character closeup 2.0

## 路径定位

- 本路径面向 Seedance 2.0 / 2.0 类多模态视频模型。
- 本路径不是导演总规范入口；剧情弧线、暗线、穿插叙事、台词节奏和镜头功能已由上游剧本、故事板和 shot_split 决定。
- 本路径只负责把已确认的镜头意图落成可执行的参考图组合、必要物证锚点和短视频提示词。
- 主路径是参考图生视频，不是先把人物和场景溶成普通首帧再生视频。
- 本路径和 blend_2_0 的区别：不追求完整主分镜溶图，只用角色图、场景图和关键道具锁定稳定事实，让视频模型完成自然同框、运镜和表演。
- 图片负责锁定稳定事实：人物身份、服装、无人场景空间、关键道具、不可替代的物证状态。
- 文本负责轻量控制：谁在什么场景、主要动作方向、主运镜、情绪落点和声音气质。
- 运行时会在单镜头分析后、提示词包生成前执行 Reference Binder：只在同一 `identity=` 候选池中选择最匹配 `asset=` 的锚点图，并由代码强制绑定到 `selectedImageRefs`。
- 提示词包阶段必须信任已绑定的 `selectedImageRefs` 顺序；不要重新编号、不要假设图像顺序和文本描述不同。
- 提示词要短、清楚、可拍，给模型自然发挥空间；过度精确会限制表演，制造噪点、僵硬动作和参考图拉扯。
- 轻量不等于流水账；先用 `shotSynopsis` 概括整段镜头，再用少量 `microShots` 写自然镜头节拍。

## 输入职责模型

- 文本：定义主题、动作方向、主镜头、节奏、光线、声音和情绪落点，不微管逐帧姿态。
- 人物图：锁定人物身份、脸、发型、年龄阶段、体态、服装，不承担场景。
- 场景图：锁定空间布局、时间、光线、材质和陈设，必须尽量无人。
- 特写图：只锁定不可替代的局部物证或状态证据；眼神、嘴角、口型、呼吸、视线等表演信号默认由视频提示词完成。
- 参考图不是连续分镜播放顺序，而是同一个视频镜头的约束集合。
- 视频 prompt 中的 `@图片N` 只代表最终真实传入视频节点的输入清单，不代表上游图片任务的源参考顺序。
- 如果某个 `storyboardFrame` 需要先图生图或溶图，视频阶段只能把生成出来的帧标成 `@图片N = 生成参考帧/物证特写参考`；源图的“图片1/图片2”只属于图片任务内部，不得在视频 prompt 里继续冒充视频参考图。
- `activeReferenceRefs` 可以引用原始 `img_x` 或 `frame:start/ref_1`，但编译器会映射到最终视频 manifest；写作时不要自创新编号。

## 参考图与关键帧边界

- `reference_images` 不是时间轴关键帧；每张图都是模型需要同时满足的视觉事实。
- 不用多张同主体局部状态图表达“先笑、再怕、再喘息”这类时间变化；时间变化交给 `microShots`、shot 边界或后期剪辑。
- 如果确实需要锁定首尾状态，优先评估 `first_last_frame` 或拆成两个 shot；不要在本路径里用多张特写图模拟关键帧。
- 零特写图是本路径默认优解，不是信息缺失；只有不可替代物证会被提升为 1 张物证特写图。
- `[特写]`、`[反应镜头]`、`[插入镜头]` 是镜头语言，不自动等于 storyboardFrames 里新增 closeup image。
- 同一主体的多状态参考会显著增加硬切风险；如果参考图之间表达了不同时间点、不同表情、不同空间状态，优先删图或拆 shot，而不是让同一段视频同时满足所有状态。

## 资产元数据读取

- 项目资产摘要中的 `identity=` 是唯一身份锚定字符串；判断人物、场景、道具身份时优先看它。
- 项目资产摘要中的 `asset=` 是这张资产的一句话用途说明；判断当前镜头应该选哪张状态图、视角图或道具状态图时优先看它。
- `tier=anchor` 才能作为人物身份、干净场景或干净道具的主参考；`tier=auxiliary` 只能辅助生成局部图，不可当人物完整身份锚点。
- 不依赖固定 `variantKey` 词表做决策；同一人物的“外出三视图、居家三视图、睡觉三视图、受伤三视图”等都通过 `asset=` 描述识别。
- 如果镜头文字出现某个人物的明确状态，优先选择 `asset=` 与该状态匹配的三视图/主视图；找不到时回退该人物基础身份三视图。
- 如果镜头中有人物参与，但没有任何同身份 `tier=anchor` 人物三视图/主视图候选，这个镜头应视为缺少人物身份资产，不能用局部脸、手部或文生图临时替代。
- 场景和道具同理：优先选择 `asset=` 与镜头时间、天气、状态、视角或道具状态匹配的干净锚点图；只有不可替代的物证状态才在本镜头里图生图派生。

## 视频模式规则

- 默认优先 reference_images。
- first_frame 只在参考图生视频不可用、或确实只有一张可独立承接动作的图时使用。
- 多图 `first_frame img2img` 只用于精确首帧构图锁定：例如手机屏幕照亮半张脸、手中道具交接、门口接触、床边开场这类“源图必须先合成同一张开场视觉事实”的情况。
- 精确首帧溶图成立时，生成出来的首帧是视频节点的唯一 `first_frame` 输入；人物、场景、道具源图只服务图片任务，不再作为视频 reference_images 重复下沉。
- first_last_frame 极少使用；只有结尾硬结果不可拆、且不锁尾帧会造成剧情误读时才用。
- text_to_video 只用于纯空镜、抽象过场或没有任何可用参考图的低风险镜头。
- 不把“场景图 + 人物图 + 人物图”规划成普通双人同框首帧溶图。

## 分镜图规划总则

- 每个镜头先判断需要哪些参考职责，而不是先生成一张大而全的同框图。
- 参与实体必须先完整列入 shotSpec / requiredRefs：直接出场、说话、被旁白指代、被观看、发生动作的人物不能漏。
- 人物锚点选图由 Reference Binder 在同身份候选池里完成；分镜图规划只决定是否需要额外的 img2img 物证图、临时锚点图或直接复用图。
- 标准组合是：无人物场景图 + 主要人物干净图 + 次要人物干净图 + 可选关键道具；没有不可替代物证时不要硬补特写图。
- 普通对话、同框站位、轻微走位、表情变化、推镜、回头、坐下、递物，交给视频提示词语言控制。
- 只有观众必须看清具体物证、且主画面自然生成不可靠时，才规划物证特写图；情绪/关系转折默认用镜头语言和可见表演完成。
- 每张图只解决一个主要风险：身份、服装、空间、道具或状态证据；不要用图片锁定动作过程、表情过程或时间片。
- 不规划动作过程图，不把每个时间片变成一张图；复杂蒙太奇只保留现实场景、关键人物、关键道具和最强情绪记忆锚点。

## 人物图规范

- 人物图必须是干净底图：单人、无背景或极简中性背景、无其他角色、无复杂场景陈设。
- 人物图可以通过图生图或溶图生成，用来更新服饰、发型、年龄阶段、伤势、仙侠/现代状态。
- 即使人物图需要参考场景的光线、时代或服饰，也只能输出干净人物底图，不把人物放进具体客厅、天门、街道等场景。
- 多角色镜头中，每个关键角色应有独立干净人物图，避免身份串位。
- 人物正侧背三视图只用于身份、服装和轮廓，不要求视频复刻三视图排版。
- 不用文生图生成已绑定人物的新脸；已有角色必须从绑定人物图出发做图生图或溶图。

## 场景图规范

- 场景图必须优先是无人物场景图。
- 场景图负责空间布局、景别基底、时间、光线、家具/建筑/道具陈设，不负责人物动作。
- 已有场景绑定图时，不要文生图重造场景；需要夜晚、破败、冷光、战损等状态变化时，用场景绑定图做图生图覆写。
- 场景图中如果原本有人，应在规划中要求去人化或重新生成无人物版本。
- 场景图和人物图不要溶成普通同框图；视频阶段用参考图和语言把人物放回场景。
- 场景一致性优先来自已有场景图，不能靠文字凭空生成另一个相似空间。
- 单个视频镜头默认只下沉 1 张主场景图；同一位置的基准图、破窗状态图、窗外城市图不能同时作为 reference_images 竞争空间事实。
- 同一地点的玻璃炸裂、停电、冷风灌入、窗外白光、灾变夜空等事件优先写进 `microShots.visualBeat`、`spatialHint`、`physicalHint`、`audioDirectives` 和光线描述，不额外补第二张场景图。
- 只有空间身份真的变化、时间/天气/形态状态需要跨多个镜头复用，或复杂空间需要可复用多角度轴线时，才在场景资产链路里生成派生图；同一段视频里仍只选当前 shot 的一个主场景参考。
- 窗外城市、楼下人群、远处灾变如果不是当前画面主空间，默认由文字、声音、窗光和主体反应承载，不作为第二场景图下沉。

## 特写图规范（默认零特写）

- 默认不规划任何特写图；常规表情、口型、眼神、视线、呼吸、停顿、笑意消失、瞳孔收缩等，都写进 `microShots.cameraHint`、`visualBeat` 和 `emotionHint`。
- 单镜头 `storyboardFrames` 中最多保留 1 张 `mode=img2img/txt2img` 的物证特写图；同一主体不允许多张特写图互相竞争。
- 只有以下 4 个条件同时满足，才允许规划 1 张特写图：
  1. 这是关系或情绪发生不可逆变化的转折点；
  2. 转折点对应一个具体可见物证或状态证据，例如血滴、玉佩裂纹、文件袋归属、符纸印记、剑身崩裂、伤口状态；
  3. 现有人物 anchor、干净场景图或道具 anchor 不能直接表达该证据；
  4. 视频模型只看 anchor + 场景 + 文本时，几乎肯定会漏掉或误解该证据。
- 表情类特写一律不生成图：眼睛、嘴唇、口型、嘴角、瞳孔、眉头、呼吸、视线、泪光、笑意、恐惧反应都由文本表演和运镜完成。
- 手部特写只有在手里承载具体物证时才保留，例如手中玉佩碎裂、血滴落在掌心、文件袋被夺走；单纯“手指停住、捏袖口、抓紧手腕”默认转写为 `emotionHint`。
- 候选特写之间如果存在状态矛盾，例如笑意特写和恐惧瞳孔特写同时出现，全部删除，让视频模型自然完成表演过渡。
- 物证特写图只服务当前镜头，不沉淀为人物/场景/道具资产库图片。
- 物证特写图的 `selectedImageRefs` 只放最相关的 1 张种子图；不要叠加人物三视图、完整场景图和多个道具图。
- 特写图不同时承担人物全身、场景全貌和多个道具；输出必须仍是单一局部证据。
- 准确中文文字不作为生成成功条件；文件、纸张、牌匾只生成材质、留白、模糊痕迹或可后期贴字区域。

## 特写自检

- 是否只有眼神、嘴角、口型、呼吸、视线、表情变化？是则删除特写图，转入 `microShots`。
- 同一主体是否有 2 张及以上特写图？是则只保留最强物证 1 张；若互相矛盾则全部删除。
- 物证是否已由道具 anchor 或人物状态 anchor 覆盖？是则删除特写图。
- 特写是否需要多张种子图才能勉强成立？是则优先拆镜或改用文本，不靠多图溶图硬塞。
- 自检完成后，单镜头特写图数量必须为 0 或 1。

## 已拆分特写镜头处理

- 上游分镜里已经写成 `[特写]`、`[插入镜头]`、`[反应镜头]` 时，不等于本镜头要生成 closeup reference frame；先把它当作 cameraHint 和 visualBeat 的镜头语言。
- 特写镜头已经是独立 shot 时，reference_images 仍只放稳定锚点：人物状态 anchor、干净场景 anchor、关键道具 anchor；不要再为这个 shot 额外补一张同主体表情/眼部/嘴唇图。
- 同一主体的“中景受伤状态图 + 眼部恐惧特写图 + 嘴唇发白特写图”属于多状态约束，容易硬切；优先只保留受伤状态人物 anchor，恐惧、喘息、嘴唇发白写进 `visualBeat` 和 `emotionHint`。
- 如果上游已经把 INSERT 拆成单独镜头，后期剪辑层的切换由 shot 边界承担；本路径不要在同一段 reference_images 里模拟剪辑切换。
- 只有当 INSERT 的核心是独立物证（例如黑色硬盘、血纱布、倒计时屏幕、碎裂玉佩）且现有道具/人物状态 anchor 无法表达时，才允许该 shot 保留 1 张物证特写图。

## 溶图规则

- 允许溶图生成干净人物图：例如用人物绑定图 + 服装/状态参考，输出无场景的当前阶段人物底图。
- 允许溶图生成必要物证特写：例如关键道具 + 必要光线，输出手中玉佩碎裂或血滴落下；不为普通表情和口型生成特写图。
- 多图溶成物证特写时，提示词开头必须先写清图片职责；但本路径优先只使用 1 张最相关种子图，只有光线或材质确实不可替代时才追加第 2 张。
- 特写溶图的输出目标必须是单一局部物证，例如手握玉佩、硬盘塞入掌心、血迹滴落、纱布被血浸透；不要为嘴唇吐字、眼神落下或恐惧反应生成特写图。
- 允许用场景绑定图做图生图生成无人场景状态图。
- 禁止溶图生成普通双人中景、普通同框首帧、普通站位建立、普通对话首帧。
- 禁止把人物图和场景图溶成“一张看起来完整”的主图后再视频生成；这会让参考图路径退化为旧的首帧路径。
- 例外：确实需要精确开场构图时可以生成一张首帧，但它必须是“单一开场视觉事实”，后续视频输入只看这张生成首帧，不再把源人物图、源场景图、源道具图重复列为视频锚点。
- 有可用绑定图时不要用文生图替代；文生图只在完全缺失且不断镜必须兜底时使用。

## 图片提示词写法

- 图片提示词使用完整自然语言，不写关键词标签串、权重括号、SD 风格参数或 provider 私有语法。
- 先写图片用途，再写画面内容；每张图只服务一个主要职责：干净人物、无人物场景、关键道具、物证局部、状态变体或缺失场景兜底。
- 文生图公式：主体 + 动作或静态状态 + 环境 + 构图/景别 + 光影/色调 + 材质细节 + 风格/比例。
- 图生图和溶图公式：参考图职责 + 保留什么 + 改变什么 + 输出目标 + 构图/光影/材质 + 禁止项。
- 多图输入必须在开头写清：`图片1` 负责身份或场景，`图片2` 负责服装/道具/姿态，`图片3` 负责光线/材质/局部证据；不要写“参考上图”“保留图里的人”。
- 人物一致性要写具体：保持同一脸型、五官比例、眼距、鼻形、下颌线、年龄阶段、发型和稳定标志物；只改变本次需要的服装、伤势、表情或状态。
- 人物状态编辑要隔离“不变”和“可变”：身份、脸和体态不变；服饰、战损、发光、泪痕、红眼、发丝凌乱等按剧情变化。
- 人物状态图可以写可见状态：脸色惨白、血纱布、伤势、服装破损、发丝凌乱；但不要单独生成眼神、嘴角、瞳孔或口型特写图。
- 无人物场景图必须写清空间布局、时间、光源、色调、材质、陈设和空镜氛围；禁止出现人物、人体剪影或无关角色。
- 缺失场景兜底图只作为镜头级临时空间锚点，不沉淀成项目资产；提示词要明确“无人场景”或“临时场景锚点”，不要把它当作时间轴关键帧。
- 特写图必须写成单一局部：Macro lens shot 或特写构图，主体占画面主要区域，背景只保留必要光线和材质；不要同时生成全身、全景和多个道具。
- 道具、纸张、牌匾、文件等不要依赖准确中文小字；可生成材质、留白、模糊痕迹或后期贴字区域。
- 图片提示词末尾可加短约束：人物身份一致、无多余人物、无拼贴、无文字水印、构图干净、主体清晰。
- 自检：图片是否只解决一个风险？参考职责是否清楚？保留/变化是否分开？是否避免了同框主图、过程帧、拼贴和无关装饰？

## 图片数量和顺序

- Seedance 2.0 / 2.0 类模型可以接收多张参考图，但质量预算应独立于硬上限；本路径常规使用 2-4 张。
- 默认组合：无人物场景图 + 主要人物 anchor + 可选关键道具 anchor + 至多 1 张物证特写。
- 5 张及以上视为高风险，必须先检查是否应拆镜、合并参考职责，或改用更适合关键帧的路径；不要把 reference_images 变成资产清单。
- 图数由职责决定，不为了凑数，也不为了少图丢掉关键人物、关键道具或真正不可替代的物证。
- 推荐顺序：无人物场景图在前，主要人物干净图随后，次要人物干净图再后，关键道具或唯一物证特写最后。
- `@图片N` 编号来自最终视频输入 manifest；如果首帧由 img2img 生成，`@图片1` 应是“生成首帧”，而不是源场景/源人物/源道具中的任何一张。
- 图片任务内部的“图片1/图片2/图片3”只用于说明图生图源图职责；最终视频正文必须重新按视频 manifest 写锚点，不能复用图片任务编号。
- 每张图片说明只写一个短职责，例如“图片1锁定无人客厅冷光空间”，“图片2锁定林照月白衣女修身份”，“图片4锁定文件袋特写”。
- 不使用“上图、图1、参考图1、Reference image 1”等混乱称呼；统一使用“图片1、图片2、图片3”。
- 图片生成说明和视频正文分层：图片阶段可以写完整生成提示词；视频阶段只写短职责绑定，例如“参考: 图片1=无人街道空间，图片2=周行睡衣身份，图片3=银白飞剑形态，图片4=垃圾桶命中证据”。
- 不写“按图片1至图片4顺序读取参考”这类容易被误解为按时间播放参考图的表达；改写成“图片编号只代表职责，不代表画面出现顺序”。
- 如果图片列表已经逐张说明职责，最终视频正文不要再完整复述一遍；只保留一行短职责绑定，避免“参考素材段 + 参考: 图片N=...”双重重复。
- 身份图和当前状态图可以同时存在，但要写清分工：身份图锁脸和体态，状态图锁服装、发型变化、手持物或伤势。当前状态图已经稳定包含手持物时，额外道具特写只在关键笑点、命题证据或生成易变形时保留。

## 分镜规划工作法

1. 先判断镜头主题和运动：谁、在哪里、动作从哪里开始、镜头怎么动、情绪落到哪里。
2. 对容易空间倒置、穿模或运动反向的镜头，先建立静态锚点：入口、墙面、床沿、桌角、柱子、明暗交界线或画面中心参考物。
3. 再列参考职责：需要哪些干净人物图、哪张无人物场景图、哪些关键道具或不可替代物证。
4. 对每个已绑定人物，优先复用或图生图生成干净人物底图。
5. 对每个已绑定场景，优先复用或图生图生成无人物场景图。
6. 只在关键物证看不清且无法用 anchor + 文本表达时补 1 张物证特写图；情绪/关系转折写入表演和运镜。
7. 选择 reference_images；除非能力不可用或镜头极简单，才退到 first_frame。
8. 最后检查是否误生成了普通同框溶图或多张同主体特写；如果是，改回人物图 + 无人物场景图 + 关键道具。

## 示例分镜落地：血纱布与黑色硬盘

以下 7 个短镜头已经由上游拆成独立 shot，因此 `[特写]` 只表示镜头景别，不表示要额外生成 closeup reference frame。

```text
镜头1 门开扑入屋内，右眼血纱布。
镜头2 陈眠看向血纱布并追问。
镜头3 许燃靠墙喘息，嘴唇发白。
镜头4 陈眠僵住追问看见什么。
镜头5 许燃剩下的眼里全是恐惧。
镜头6 许燃掏出黑色硬盘塞进陈眠手心。
镜头7 许燃盯着她说明硬盘内容。
```

推荐参考职责：

- 全段共享：夜晚室内无人场景 anchor。
- 许燃：优先使用“右眼血纱布、脸色惨白”的人物状态 anchor；如果只有基础人物图，先生成干净受伤人物状态图，不生成眼部特写图。
- 陈眠：使用人物身份 anchor；惊恐、声音发紧、后背僵直写进 `visualBeat` / `emotionHint`。
- 黑色硬盘：使用道具 anchor；镜头6 可激活硬盘道具 anchor，但默认不额外生成手心硬盘特写图，除非硬盘外形或归属无法看清。

逐镜建议：

| 镜头 | 参考图策略 | 特写处理 |
|---|---|---|
| 1 | 场景 + 许燃受伤状态人物 anchor | 血纱布是人物状态，不另生成眼部 closeup |
| 2 | 陈眠 anchor + 许燃受伤状态 anchor，必要时加场景 | `[特写]` 由 cameraHint 固定到视线/血纱布，不补特写图 |
| 3 | 许燃受伤状态 anchor + 场景 | 嘴唇发白、喘息写进 emotionHint |
| 4 | 陈眠 anchor + 场景 | 反应镜头只写后背僵直、声音卡住 |
| 5 | 许燃受伤状态 anchor | 恐惧眼神是表演信号，不生成眼部 closeup 图 |
| 6 | 许燃 anchor + 陈眠 anchor + 黑色硬盘 anchor | 硬盘是关键道具；只有归属/形态看不清才允许 1 张物证特写 |
| 7 | 许燃受伤状态 anchor + 黑色硬盘 anchor | 台词信息不靠画面文字，不生成倒计时/视频画面特写 |

## 空间轨迹控制

- 本路径的提示词要把文学动作转成几何动作：少写“进门、躲开、转身离开、冲过去”，多写相对锚点、屏幕方向、纵深变化和可见反馈。
- 复杂运动先定义一个不动空间锚点，例如“以图片1门框为中心参照”“以图片1右侧墙面为终点平面”“以床沿和窗光交界线锁定前后关系”。
- 光影可以充当空间边界：冷色暗区作为起点，暖光亮区作为终点，人物必须跨越明暗交界线；不要让明暗关系在动作中反向或漂移。
- 需要表达向内移动时，写成“重心向画面深处平移，身体在门框内逐渐变小，被门框或室内陈设遮挡”；需要表达向外逼近时，写成“沿画面纵深靠近镜头，身体遮挡背景，占画面比例逐渐增大”。
- 左右移动用屏幕坐标写清楚：从画面左侧阴影切入、推向画面右侧墙面、沿桌沿从右向左横移。不要只写“过去”“躲开”“上前”。
- 上下运动只在必要时写 Y 轴或高低层级，例如抬手到胸口高度、身体下沉到桌面以下、从台阶上方向下退半步；避免同时写复杂旋转和纵深位移。
- 短镜头里的动作按“预备 -> 爆发 -> 阻尼”组织：先给脚尖、肩膀、视线或重心的起势，再给主位移，最后给墙面、衣料、发丝、纸张、道具或光影的惯性反馈。
- 同一个 3 秒片段不要同时要求转身、走远、挥手、回头和说话；先原地旋转或反应，再进入 Z 轴位移。
- 碰撞、推搡、压墙、摔倒、扑倒、开门穿行等高风险动作必须写清不透明物体边界：人物不能穿过门板、墙面、桌子或其他身体，只能绕开、推开、撞上或被遮挡。
- 反向约束只在高风险动作里使用，短而具体，例如“禁止向镜头前方反向移动”“禁止人物穿过不透明门板”“保持墙面和门框固定不漂移”。

## 视频提示词写法（自然节拍原子化）

输出形态是 `shotSynopsis` + `microShots`，但心智模型不是填表，而是把一个镜头拆成几句自然可拍的镜头节拍。结构只保证可编译，真正的镜头理解仍由自然语言完成。

- `shotSynopsis` 是整段镜头的一句话总览，写清从哪里动到哪里、情绪落到哪里；可以有少量叙事情绪词，不要复述每拍。
- `microShots` 是镜头节拍数组。每拍最重要的是 `visualBeat`，它是一句自然镜头句，可以同时包含主体、动作、视线、口型、运镜和情绪落点。
- `cameraHint` 是可选自然镜头提示，不是枚举选择器。可以写“中近景固定”“从门外缓慢推近到面部反应”“低角度轻手持跟随”。
- 每拍只需要填写有信息增益的字段：普通表情、口型、停顿和正反打不需要强行写 `spatialHint`、`physicalHint` 或 `constraintHints`。
- 时间窗要连续覆盖镜头时长，但切片数量服从镜头自然节奏：短镜头可以 1-2 拍，7-10 秒通常 3-4 拍，复杂动作再增加。
- `activeReferenceRefs` 是注意力锚点子集，不是重新编号，也不代表画面出现顺序。它引用真实 `img_x`；编译器会再映射为 `@图片N`。
- 每个 micro-shot 默认只激活 1-2 张真正有信息增益的参考；单人反应拍只激活该人物，单道具插入拍只激活道具或生成物证帧，双人同框才同时激活两个人物。
- 不把所有可用锚点默认塞进每个 micro-shot；全量激活会让人物、场景、道具、生成帧互相抢权重。
- close-up 景别优先激活人物或道具锚点，大全景优先激活场景锚点；只有物证材质或光线确实不可替代时才额外引用场景图，不要无意义全量引用。
- `primaryReferenceRef` 只在多图参考里存在明显主图时填写；没有明确主图时可以留空。
- HDVP/SDV 只服务高风险空间动作：进出门、靠近/远离镜头、推撞、压墙、摔倒、追逐、穿行或绕行。普通表情和对白不要硬塞矢量。
- 高风险位移再填写 `spatialHint`：写清起点、方向、终点和不可穿透边界；有碰撞或余震时再补 `physicalHint`。
- `beatRole` 是叙事提示，不是必填字段。只有诱导、转折、揭示、余震、关系不可逆变化明显时再用自然短语。
- `emotionHint` 只在有助于生成时填写，例如嘴角僵住、呼吸停顿、手指停住、指节泛白；这些只是文本表演提示，不生成对应特写图。
- `constraintHints` 写自然短语，例如“禁止穿过不透明墙面”“禁止位移方向反转”“明暗交界线稳定”；不要写 enum key。
- 对白、VO、OS 通过 `dialogueLineIndices` 绑定到能承接口型、听者反应、旁白画面或情绪落点的节拍。
- 缺席角色（玻璃后人影、门后声音、楼上脚步）优先通过 `audioDirectives.sfx` 和当前主体视线承载，不要硬塞进画面动作。

## HDVP 使用边界

- 静态锚点用于高风险空间，不用于所有镜头。门框、墙面、桌沿、床边、明暗交界线都可以作为锚点。
- 光影深度墙只在冷暖区或明暗线参与空间方向时使用；普通灯光氛围交给项目风格和 `visualBeat`。
- 空间矢量是风险提示，不是每拍标签。可以写 X+/X-/Y+/Y-/Z+/Z-，也可以直接写中文方向；固定对话和表情不用写。
- `physicalHint` 用来证明受力和惯性，例如发丝前飘、衣袖滞后、纸张晃动、身体短暂反弹；没有受力就不要制造反馈。
- 反向约束越短越好，只在必要时写成自然短语，例如“禁止向镜头前方反向移动”“禁止人物穿墙”“明暗线不漂移”。

## 自然节拍示例

以下示例里的“近景/特写”都只表示 `cameraHint` 和 `visualBeat` 的镜头景别；`activeReferenceRefs` 仍只引用稳定锚点，不新增表情、眼部、嘴唇或手指特写图。

8s 大门戏（参考图：img_1=场景:小区门，img_2=人物:小美，img_3=道具:门禁红灯）：

```text

microShots:
  1. 0.0–2.0s
     cameraHint: 全景，镜头从门外缓慢推向铁门
     visualBeat: 冷蓝夜色下的小区门口空旷安静，镜头缓慢推向半掩铁门和闪烁门禁。
     activeReferenceRefs: ['img_1', 'img_3']
     beatRole: 建立等待
  2. 2.0–5.5s
     cameraHint: 中近景固定
     visualBeat: 小美停在铁门外，双手抱胸微微前倾，先看保安室玻璃，再把视线勾向玻璃后的人影。
     activeReferenceRefs: ['img_2', 'img_1']
     primaryReferenceRef: 'img_2'
     dialogueLineIndices: [0]
     beatRole: 诱导推进
  3. 5.5–8.0s
     cameraHint: 近景，缓慢推到面部反应和指尖手势
     visualBeat: 镜头推近到小美脸侧和指尖，小美食指轻点嘴边，笑意加深后留下半拍停顿。
     activeReferenceRefs: ['img_2']
     dialogueLineIndices: [1]
     beatRole: 揭示诱饵

audioDirectives:
  ambience: ['远风', '门禁电流']
  sfx: ['高跟鞋', '玻璃后呼吸']    # 缺席的姜夜由 sfx 暗示
  bgmMood: '轻喜悬念'
  bgmIntensity: 'low'

```

## 复杂叙事场景

| 场景 | 落地方式 |
|---|---|
| 正反打对白 | 两个 micro-shot 交替写“过肩看 A / 中近景看 B”；activeReferenceRefs 在 A/B 角色 anchor 间切换；dialogueLineIndices 分别绑定双方台词 |
| 闪回 | 优先拆成独立 shot；若必须留在同镜头内，只写一个短 micro-shot，activeReferenceRefs 指向稳定记忆锚点，不把参考图当作时间关键帧 |
| 内心独白 | dialogue.lines[].kind = inner_monologue + 对应 micro-shot 用 close-up 自然镜头句；emotionHint 写呼吸停顿/眼眶湿润；不写嘴部动作 |
| 画外音先行（声画错位） | dialogue.lines[].kind = offscreen + 在更早的 micro-shot 上挂 dialogueLineIndices；该 micro-shot 的 visualBeat 写听者反应 |
| 蒙太奇 | 优先拆成多个 shot 或后期剪辑；若同镜头内短节拍串联，activeReferenceRefs 各只挂 1 张代表性稳定锚点，不新增表情特写图 |
| 隐喻镜头 | 单独插入镜头；visualBeat 写实际动作（雨滴沿玻璃滑落），不写"象征"；靠 beatRole + 时间相邻关系构建隐喻 |
| 缺席角色暗示 | 不进 visualBeat 的主体动作；由 audioDirectives.sfx + 当前主体的视线方向承载 |

## 台词和声音锚点

- 运行时会把台词和声音拼接到最终 prompt；本路径要在 visualBeat 和 audioDirectives 里给它们留足画面动机。
- 对白戏要在画面里安排：说话人开口动作、听者反应、停顿、视线落点、说完后的余韵；不要让最终台词悬浮在画面之外。
- VO 旁白要安排可承载旁白的画面：主角回忆、物件证据、空镜、背影、慢推或凝视；不要让 VO 覆盖无关动作。
- 画外音 offscreen 要保留声源方向或空间关系，例如门外、电话里、走廊尽头、屏幕后方；visualBeat 只写角色听见后的反应和环境变化，不把画外音误写成画内角色开口。
- 如果画外音内容要留到下一镜，不要把“冷冷的女声从楼上传来、内容留到下一镜揭示”写成引号台词；当前镜只写声音压入、声源方向和角色僵住/抬眼/继续逃的反应。
- 台词栏只放角色真正说出口或旁白真正念出的文字；说明性文字不进入引号，不生成字幕，也不要求模型读出来。
- 内心 OS / inner_monologue 不表现为口型或对镜说话；要用停顿、凝视、手部动作、呼吸、背影或物件特写承载。
- 正反打对白要给出说话人和听者的可见反应链：建立关系镜头 -> 说话人中近景/过肩 -> 听者反打 -> 情绪落点或双人同框回收。
- audioDirectives 只补 ShotSpec.audio 不足的声音动机、动作音或 BGM 气质。声音词要具体，如床沿轻响、金属剑身细颤、雷击屏障闷响、衣料摩擦、短促呼吸。
- BGM 气质要服务段落弧线，不只写“温柔/紧张”；优先写成“轻喜悬念”“史诗感中的温柔回收”“压抑后决堤”“灾难快切后收束为心跳”。
- 声音变化要对应画面变化：闪回进入、证据特写、情绪爆点、回到现实，都要有可听的转折来源。
- audioDirectives 输出前要去重：同类环境底噪、脚步、衣料、塑料袋、破风、金属碰撞和 BGM 只保留一次；重复词会消耗提示词预算并削弱镜头控制。

## 视频提示词约束

- 核心写主题 + 运动，不写长篇散文化解释。
- 一段镜头只指定一种主运镜：固定、缓慢推近、轻微跟拍、平稳横移等。
- 动作写方向、锚点和过渡，不写逐帧姿态；优先写缓慢、连续、可衔接的小动作，避免把表演锁死。
- 对空间倒置风险高的动作，必须写清“相对哪个锚点、向屏幕哪一侧或纵深哪一端移动、终点被什么遮挡或阻挡”。
- 情绪写可见身体信号，例如眼神回避、嘴角僵住、呼吸停顿、肩背不动，不只写“悲伤、愤怒、震惊”。
- 台词只放在对应时间段的关键位置，不把整段画面写成念白。
- 对白、VO、OS 保留为台词信息，不生成字幕或画面文字。
- 声音可短写环境音、动作音和 BGM 情绪；不要把声音扩写成大段说明。
- 结尾稳定性约束保持短：人物身份稳定、动作自然、画面无闪烁、无字幕水印。
- 最终给模型的正文删除导演表格字段和解释性内容，例如叙事目的、关键确认、黄金三秒、观众理解、焦段、光圈和镜头型号；这些可以存在于内部分析，不进入 video prompt。

## Seedance 方法论落地

- 固定参数和正文分离：画风、时长、比例由编译器放开头，microShots 不重复。
- 画面叙事优先：先用 shotSynopsis 把整体镜头说顺，再用 microShots 写自然节拍。
- 自然镜头句通常包含：时间感、空间锚点、镜头提示、主体、运动方向、场景/光线、可见情绪信号或物理反馈；不需要每拍填满。
- 镜头语言克制：一个时间段一个主景别或主运镜，不同时堆推、拉、摇、移。
- 动作可拍：用停顿、抬眼、侧头、呼吸、手指停住、物品滑落等小动作表达情绪转折。
- 空间可验：门框、墙面、桌沿、床边、明暗交界线等锚点保持固定，人物运动围绕锚点发生，不让空间随主体折叠。
- 运动可验：进出、靠近、离开、推撞、跌倒、绕行等动作要能从体型比例、遮挡关系、重心方向和惯性反馈看出来。
- 参考图职责明确：每张图只锁身份、空间、道具或不可替代物证，不把参考图当作逐帧动作或时间关键帧。
- 语言控制克制：只锁主动作、方向和情绪落点，不追求每秒一动作或精确到每个微表情。
- 台词和声音不能只靠最终栏目；画面必须有对应的口型、反应、动作音来源和 BGM 转折点。
- 约束词简短：只保留人物身份稳定、面部稳定、动作自然、无字幕水印等高收益约束。

## 质量标准

- 规划结果能看出：人物图是干净底图，场景图无人，关键道具稳定；如有特写图，只能对应具体物证。
- 普通同框关系不依赖溶图，靠参考图 + 语言控制完成。
- 有绑定人物/场景时，不用文生图重造身份或空间。
- 如确实多图溶成物证特写，图片1、图片2各自职责清楚，输出仍是单一局部证据；普通表情和反应不得走多图溶图。
- microShots 时间窗连续覆盖镜头时长；每拍有清楚的镜头重心和自然 visualBeat。
- microShots 不堆导演心理解释；必要情绪优先写成可见动作、视线、口型、停顿或身体信号。
- close-up 景别、局部证据和插入节拍优先减少场景图干扰；全景、远景和建立镜头优先包含场景锚点。确有光线/材质需要时可以例外。
- activeReferenceRefs 长度 ≥2 且有明确主图时填写 primaryReferenceRef；没有明确主图时不要硬填。
- 高风险动作优先用 spatialHint、physicalHint、constraintHints 覆盖；beatRole 只在叙事转折明显时使用。
- 跨光影区动作的 spatialHint 写出冷暖区起止，并在 constraintHints 写“明暗交界线稳定”。
- 台词通过 dialogueLineIndices 挂在能承载口型/反应的 micro-shot 上，offscreen 挂听者反应拍。
- audioDirectives 三类（ambience/sfx/bgmMood）去重；缺席角色由 sfx 暗示。
- 每张图职责明确，不是海报、拼贴、过程帧或完整分镜组。
- 首尾帧极少出现；大多数镜头由 reference_images 或必要时 first_frame 承接。

## 失败回退

- LLM 仍输出旧式长段散文视频提示词：检查 schema 是否正确加载；改写为 shotSynopsis + microShots，保留自然镜头句到 visualBeat。
- visualBeat 承载了多个相互抢权重的动作：保留主镜头重心，把次要动作拆到下一拍或删除。
- close-up 或局部镜头包含场景图导致脸漂：优先移除场景图，仅留人物 anchor；如确实需要环境光线参考，缩短场景引用职责。
- micro-shot 时间窗有间隙：找最近邻 micro-shot 扩展，或新增过渡 micro-shot；不允许"留白"。
- 空间倒置：补 spatialHint 写明 SDV 句式和屏幕方向，constraintHints 写“禁止位移方向反转”。
- 穿模：constraintHints 写“禁止穿过不透明物体”；余震拍写"撞上/被遮挡/反弹"而不是"穿过"。
- 跨光影区漂移：spatialHint 写冷暖区起止，constraintHints 写“明暗交界线稳定”。
- 多图身份串脸：activeReferenceRefs ≥2 且有明确主脸/主服装时补 primaryReferenceRef；特写拍优先只引 1 张人物 anchor。
- 局部物证看不清：补一个短插入节拍（≤1.2s），activeReferenceRefs 只引该证据图；情绪转折看不清时改写表演、运镜和 emotionHint，不补表情图。
- 画外音被误当台词：dialogue.lines[].kind = offscreen，挂在听者 micro-shot；visualBeat 写听者反应。
- 参考图重复：anchor 注册表是单一真理源，micro-shot 只引用 imageRef；不允许在 spatialHint 或 visualBeat 自创新编号。
- 声音重复：按 ambience/sfx/bgmMood 三类去重；audioDirectives 每类 ≤3 条。
- 镜头氛围词侵蚀：发现 microShots 出现"压抑/克制/氛围"等词，整拍重写为可见信号。
- 画面太僵：减少 constraintHints 数量；让参考图锁身份与空间，micro-shot 只给运动方向，不锁微表情链。
