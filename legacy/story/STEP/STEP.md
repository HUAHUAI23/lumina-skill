## 前置声明
本文件是 `story` skill 的完整执行规范。必须先完成模式路由，再按步骤执行；禁止跳步，禁止先出成稿后补结构。

---

## STEP 0｜模式路由与输入契约

### 模式路由
- `MODE_AD_COPY`
- `MODE_AD_SCRIPT`
- `MODE_IMAGE_TO_STORY`
- `MODE_STORY_SHORT`
- `MODE_NOVEL_LONG`
- `MODE_VIDEO_PROMPT`
- `MODE_POLISH`

### 快速触发
- 用户只说“开始 / 需要 / 先来一版 / 给我首轮”时，触发 `MODE_FIRST_ROUND_BUNDLE`。
- 默认假设：通用消费品 + 城市场景 + 25-35 岁受众。

### 输入契约
1. 任务目标
2. 受众画像
3. 主体信息
4. 风格方向
5. 平台与规格
6. 输出形式

### 强制规则
1. 角色、任务、硬约束、输出格式前置
2. 先出结构，不先堆文采
3. 图片任务先列事实，再列推断
4. 视频提示词默认中文主稿
5. 只重写低分片段

**质量检查点：**
> 是否已经明确“目标 + 受众 + 主体 + 平台 + 形式”？

## STEP 1｜素材解构与任务卡

**目标**：把用户输入转成可执行任务结构。

### 输出任务卡
```text
任务卡
- 模式：
- 主目标：
- 受众：
- 成功标准：
- 必须包含：
- 必须避免：
```

### 规则
- 图片只先提取可见事实
- 商业任务必须写转化目标或传播目标
- 故事任务必须写情绪承诺

**质量检查点：**
> 是否同时明确了“目标、受众、成功标准”？

---

## STEP 2｜人物与叙事蓝图

**目标**：先锁人物与内容骨架。

### 人物引擎
- 共性：外显身份
- 个性：内在驱动
- 矛盾：冲突点
- 和谐：矛盾成立的原因

### 蓝图选择
- 广告：钩子 -> 问题 -> 方案 -> 证据 -> CTA
- 短故事：开场钩子 -> 冲突升级 -> 反转或情绪落点
- 长篇：世界观 -> 主线 -> 支线 -> 章尾钩子
- 视频提示：氛围 -> 主体动作 -> 镜头动作 -> 场景变化 -> 收束

**质量检查点：**
> 是否能用 3 句话说明“为什么目标受众会继续看/读下去”？

## STEP 3｜按模式创作

### A. `MODE_AD_COPY`

**要求：**
- 至少 3 个版本
- 每版都含标题、钩子、正文、CTA
- 默认附 A/B Hook 与 A/B CTA

**模板：**
```text
【广告文案包】
产品/品牌：
受众：
核心卖点：

【版本A】
标题：
首屏钩子：
正文：
CTA：
```

### B. `MODE_AD_SCRIPT`

**要求：**
- 0-3 秒必须抓人
- 结尾必须可执行 CTA
- 每场必须有画面、镜头、台词/旁白、声音

### C. `MODE_IMAGE_TO_STORY`

**规则：**
1. 先列可见事实
2. 再列合理推断
3. 输出微故事、短故事、剧情版三档
4. 推断不得伪装成事实

### D. `MODE_STORY_SHORT`

**规则：**
- 三段式优先
- 每 150-250 字至少有一个情绪推进点
- 不用大段抽象说明代替场景

### E. `MODE_NOVEL_LONG`

**规则：**
- 先做世界观与章节规划
- 再写章节目标卡
- 每章都要检查设定、人物、时间线一致性

### F. `MODE_VIDEO_PROMPT`

**核心升级：**
- 默认中文主提示词
- 英文只作平台增强版
- 采用“主体动作 + 镜头动作 + 场景动作”三分法
- 情绪必须转写为微表情和生理反应

**中文模板：**
```text
氛围：...
主体动作：...
镜头动作：...
场景动作：...
光影与色彩：...
强约束：...
```

**可选英文增强版模板：**
```text
Mood: ...
Subject motion: ...
Camera motion: ...
Scene motion: ...
Lighting and color: ...
Constraints: ...
```

### G. `MODE_POLISH`

**要求：**
1. 先指出问题
2. 再给优化稿
3. 最少 2 种风格版本

**质量检查点：**
> 当前模式的关键字段、目标和格式是否全部完整？

## STEP 4｜草稿生成与局部增强

**目标**：先出可控稿，再精修。

### 流程
1. 结构草稿
2. 语言精修稿
3. 可选增强：
- 情绪增强
- 画面增强
- 转化增强

**质量检查点：**
> 草稿是否已经满足“结构完整 + 信息齐全 + 可继续精修”？

---

## STEP 5｜质量闸门

### Rubric（每项 1-5 分）
1. 目标对齐
2. 结构清晰
3. 情绪牵引
4. 语言质感
5. 可执行性

### 模式专项检查
- 广告文案：卖点清楚、CTA 明确、存在 A/B 变体
- 广告脚本：开头抓人、节奏递进、画面可拍
- 图片故事：遵守事实边界
- 长篇小说：设定自洽、章节可连载
- 视频提示词：有动作、有镜头、有光影、有强约束

规则：总分低于 20 必须局部重写。

## 常见错误与修正

| 错误 | 表现 | 修正 |
|---|---|---|
| 目标漂移 | 写得好看但不解决需求 | 回到任务卡 |
| 情绪空词 | 只写“开心/生气” | 改成动作与微表情 |
| 图片脑补过度 | 把推断写成事实 | 先拆事实与推断 |
| 视频词空泛 | 只有形容词堆叠 | 补动作、镜头、场景变化 |
| 广告无证据 | 只有喊口号 | 补证据场景 |
