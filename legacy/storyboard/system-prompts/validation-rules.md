# 分镜与单镜头验证规则

本文档定义 `storyboard` 输出的严格验证规则。默认以中文主稿为基线，英文增强版仅做附加校验。

---

## 核心验证规则

### 规则 1｜素材映射一致
- 使用了素材时，必须与 `图片N / 视频N / 音频N` 映射一致
- 不得中途切换命名
- 无素材时，不强行插入编号

### 规则 2｜台词语言保留
- 用户原始台词必须保留原语言
- 禁止翻译后替代原文
- 可以附注释，但不能丢原文

### 规则 3｜无硬性脑补
- 不新增用户未给出的关键人物、关键道具、关键剧情事件
- 合理扩展仅限于气氛、材质、光影、基础环境细节

### 规则 4｜结构完整

#### 单镜头最少包含
- 镜头定位
- 镜头运动
- 主体动作
- 场景变化
- 强约束

#### 分镜脚本最少包含
- 场次表
- 镜头表
- 中文平台投喂稿
- 校验结论

### 规则 5｜运镜物理逻辑正确
- 同镜头不能同时“快速推进”和“快速后拉”
- `Pan/Tilt` 与 `Dolly/Truck/Crane` 的描述不能混为一个动作
- 同镜头最多允许一个主运动和一个辅助修饰

### 规则 6｜长度适中
- 单镜头主提示词建议 6-10 行
- 分镜每镜头说明建议 4-8 行
- 超长时优先拆镜，而不是继续堆一镜

---

## 校验清单

```text
□ 素材映射一致
□ 台词语言保留
□ 无硬性脑补
□ 结构完整
□ 运镜物理逻辑正确
□ 长度适中
□ 连续性合理
□ 广告任务的 CTA 清晰
```

## 常见错误速查

| 错误 | 示例 | 修正 |
| --- | --- | --- |
| 映射混乱 | 一会儿“Image 1”一会儿“角色A” | 统一为 `图片1` 或用户指定命名 |
| 台词被翻译 | 把“快跑！”写成 `Run!` | 保留“快跑！”原文 |
| 新增剧情 | 用户没提追兵却写“机器人追来” | 删除硬脑补 |
| 结构缺失 | 只有一段总描述 | 拆成标准结构 |
| 运镜冲突 | 同镜头又推又拉又甩镜 | 保留一个主运动 |

---

## 自动修复建议

| 问题 | 自动修复方向 |
| --- | --- |
| 结构缺失 | 补齐标准标题块 |
| 运镜冲突 | 删去次要运动，保留主运动 |
| 台词丢失 | 恢复原台词 |
| 广告无 CTA | 在结尾补 CTA 场次 |
