## 前置声明
本文件定义 `image_prompt` skill 的完整执行流程。收到任务后，必须先完成模式路由与输入补齐，再输出提示词。默认中文输出；英文仅作为可选增强层，不得替代中文主稿。

---

## STEP 1｜任务解析与模式路由

**目标**：把需求稳定路由到正确模式，避免“一条提示词包打天下”。

### 路由优先级
1. 明确“图生图/重绘/修图/放大/局部替换” -> `MODE_I2I`
2. 明确“九宫格/16 格/25 格/宫格叙事/连续画面” -> `MODE_GRID_STORY`
3. 明确“角色设定/定妆/三视图/角色卡” -> `MODE_CHARACTER`
4. 明确“空镜/环境图/概念场景/不要人物” -> `MODE_ENVIRONMENT`
5. 明确“广告图/KV/产品主视觉/投放图” -> `MODE_AD`
6. 明确“同主体多风格/矩阵风格测试” -> `MODE_STYLE_BATCH`
7. 其他图片需求 -> `MODE_T2I`

### 最少输入字段
1. 目标：海报、广告、设定、叙事、社媒、封面
2. 主体：人物、产品、空间、道具、事件
3. 风格：写实、插画、国风、动漫、广告、CG
4. 比例：`1:1`、`9:16`、`16:9`、`4:5`
5. 参考素材：无 / 单图 / 多图
6. 硬约束：必须保留、禁止出现、可变范围

### 缺失处理
- 缺比例：人物默认 `9:16`，广告与环境默认 `16:9`，宫格默认 `1:1`
- 缺风格：默认“高质感电影写实”
- 缺年龄信息且涉及性感边界：默认“成年人”
- 缺参考图但用户要求“保持一致”：先声明“仅能做文字锚定，无法做严格外观锁定”

**质量检查点：**
> 是否已经明确“模式 + 目标 + 主体 + 风格 + 比例 + 硬约束”？

## STEP 2｜最新提示词工程原则（强制）

**目标**：把 2025-2026 官方最佳实践固化为稳定规则，减少玄学调参。

### 2.1 结构化而不是散文化
所有主提示词默认按以下顺序组织：
1. 任务目标
2. 主体锚点
3. 场景锚点
4. 构图与镜头
5. 光影与色彩
6. 材质与细节
7. 强约束清单
8. 参数建议

### 2.2 正向约束优先
- 多写“必须包含什么”和“必须保持什么”。
- 少写长串“不要什么”；仅对文字污染、畸形、错字、额外肢体等高频缺陷补充短负面词。
- 对 FLUX、Imagen 等不依赖负面词的平台，改用正向限制与构图限制。

### 2.3 先简单验证，再加细节
- 第 1 轮先验证主体、构图、方向、比例。
- 第 2 轮再加材质、皮肤、布料、反射、微表情、商业精修。
- 不要一开始就堆满 30 个风格词。

### 2.4 多图任务先锁不可变项
当存在参考图或连续画面时，先写：
- 不可变项：脸型、发型、瞳色、服装、产品形态、品牌色、空间结构
- 可变项：姿态、镜头远近、局部道具、天气、光位强弱

### 2.5 强约束与软风格分离
- 强约束负责“不能跑偏”
- 风格词负责“长成什么样”
- 参数负责“怎么生成”
- 三者不能混写成一团

**质量检查点：**
> 提示词是否已经明确区分“锚点、风格、强约束、参数”四层？

## STEP 3｜模型与平台选择

**目标**：根据任务目标选择更合适的模型或平台写法。

### 通用路由
- `5.0 Lite`：时效检索、复杂知识语义、信息密集型画面
- `4.5`：写实人物、商业产品、高保真编辑
- `4.0`：快速探索、低成本草图
- `3.1`：艺术风格化、电影插画
- `3.0`：文字排版、海报文字需求

### 平台化补偿
- OpenAI / GPT Image：适合多轮编辑与连续修图，优先使用“上一轮图像继续编辑”策略
- Runway Gen-4：适合自然语言整句写法，避免重度负面词
- Imagen：支持从短提示迭代到长提示，适合逐步加细节
- Midjourney：当要求更贴提示时，降低风格自由度；需要风格稳定时使用风格码/样式参考

**质量检查点：**
> 模型和平台的选择理由，是否与“任务目标”一一对应？

## STEP 5｜按模式生成提示词

**目标**：根据模式产出可直接投喂的中文提示词包。

### 5A. `MODE_T2I` 文生图

**构建公式：**
```text
任务目标 + 主体锚点 + 场景锚点 + 构图镜头 + 光影色彩 + 材质细节 + 强约束
```

**中文模板：**
```text
请生成一张用于[用途]的图片。主体是[主体描述]。场景位于[地点/时间/环境]。画面采用[景别/机位/构图]，强调[光影/色彩]，突出[材质/细节]。必须保留[不可变项]，必须出现[关键元素]，禁止出现[禁项]，允许变化[可变范围]。整体风格为[主风格]，画幅为[比例]。
```

**强约束：**
- 禁止只写“电影感、氛围感、高级感”这类空词，必须落到镜头、光位、材质、色彩。
- 禁止在主提示词里混入互相冲突的风格，例如“极简平涂 + 超写实毛孔”。

### 5B. `MODE_I2I` 图生图

**中文模板：单图编辑**
```text
以参考图为唯一主体依据，在不改变主体身份、构图主体关系和核心光影方向的前提下，执行以下编辑：[编辑目标]。必须保留[脸部/产品形态/服装/品牌色/空间结构]，重点优化[材质/清晰度/局部细节]，清除[文字/水印/HUD/脏污]，输出干净、自然、无拼接痕迹的结果。
```

**中文模板：双图融合**
```text
将参考图1作为主体身份与光影基准，将参考图2作为姿态与构图基准。最终画面必须保留参考图1中的[不可变项]，同时采用参考图2中的[姿态/镜头/构图关系]。禁止新增未提及角色、道具和文字元素，输出高保真、统一风格的结果。
```

### 5C. `MODE_CHARACTER` 角色设定

**强约束：**
- 全身
- 自然站立
- 轮廓清晰
- 无剧情动作
- 背景极简
- 外貌与服装描述必须具体

**中文模板：**
```text
请生成角色设定图。角色为[姓名/身份]，年龄为成年人，外貌特征为[发型/脸型/肤色/瞳色/体型]，服装为[服装细节]，材质为[材质]。画面要求全身、自然站立、正面视角、无剧情动作、表情平静、背景纯净简洁、轮廓清晰，重点展示角色外形与服装结构，画幅为9:16。
```

**中文模板：三视图**
```text
请生成角色三视图设定图，同一角色保持完全一致的脸型、发型、服装与配色，依次展示正面、侧面、背面。画面为设定稿用途，站姿中性，背景纯白或浅灰，无地面杂物，无戏剧化光影。
```

### 5D. `MODE_ENVIRONMENT` 场景空镜

**强约束：**
- 绝对无人
- 不出现文字牌匾以外的界面元素
- 必须交代空间层次

**中文模板：**
```text
请生成一张场景空镜图，地点为[地点名]，时间为[时间]，环境包含[建筑/自然/陈设细节]。画面采用[全景/远景/广角]，强调[光线、空气感、天气、色温]。必须保持无人、无主角、无界面文字污染，重点展示空间层次、景深和氛围建立，画幅为16:9。
```

### 5E. `MODE_GRID_STORY` 叙事宫格

**统一规则：**
- 必须先写“序列头”
- 必须连续编号 `格1` 到 `格N`
- 每格至少包含“动作 + 镜头 + 环境变化”
- 画面中禁止编号、字幕、水印、HUD

**中文模板：**
```text
序列头：风格[主风格]；题材[题材]；画幅[比例]；总格数[9/16/25]；连续性要求[保持角色与场景一致]。

第1行（铺垫）
格1：……
格2：……
格3：……

第2行（推进）
……

参数：画幅[比例]；连续性优先；禁止文字污染。
```

**25 格节奏：**
- 第 1 行：建立
- 第 2 行：推进
- 第 3 行：高潮
- 第 4 行：反应
- 第 5 行：收束

### 5F. `MODE_AD` 广告图

**必填项：**
- 品牌/产品
- 卖点
- 目标人群
- 留白区域
- Logo/标题/CTA 位置

**中文模板：**
```text
请生成一张[品牌/产品]的广告主视觉。主体为[产品名称与外观]，核心卖点是[卖点]，目标人群为[受众]，使用场景为[场景]。画面采用[构图]，突出[材质/光影/品牌色]，必须预留[标题区/卖点区/CTA区]，禁止杂乱背景、错误品牌色、变形包装和无关道具，整体呈现高级商业广告质感，画幅为[比例]。
```

### 5G. `MODE_STYLE_BATCH` 风格批量

**规则：**
- 固定主体与构图
- 只换风格层
- 每版必须复用同一主体锚点

**输出要求：**
- 先给基础母提示词
- 再给 4、9 或 25 条风格分支
- 每条风格分支只改风格，不改人物身份、场景结构和主构图

**质量检查点：**
> 当前模式的硬约束是否都已写进最终提示词？

## STEP 6｜质量闸门与回退

**目标**：在交付前拦截高概率失败点。

### 必检项
1. 是否存在主体漂移风险
2. 是否存在文字污染风险
3. 是否存在风格冲突
4. 是否遗漏必须元素
5. 是否误加未提及角色/物体/剧情

### 平台兼容策略
- FLUX / Imagen：优先正向约束，不依赖长负面词
- Midjourney：要求更贴提示时，降低风格自由度
- OpenAI / GPT Image：优先多轮编辑，沿用上一轮结果做局部修正

### 失败回退
1. 先减少冲突风格词
2. 再收紧构图与主体锚点
3. 再拆为两轮：构图稿 -> 精修稿
4. 最后再切换更适合的平台或模型

**质量检查点：**
> 是否已经给出“主提示词 + 关键约束 + 参数建议 + 回退方向”？

## STEP 8｜首轮默认包（用户只说“来一版/开始”时）

**默认设定：**
- 风格：高质感电影写实
- 角色：成年人
- 比例：人物 `9:16`，环境/广告 `16:9`，宫格 `1:1`
- 流程：先构图稿，再精修稿

**必须一次给全：**
1. 文生图
2. 图生图
3. 宫格叙事
4. 广告 KV

**首轮示例要求：**
- 示例必须为中文完整句
- 示例必须包含硬约束
- 示例必须能直接复制投喂

## 常见错误与修正

| 错误 | 表现 | 修正 |
|---|---|---|
| 模式混淆 | 把空镜写成人物海报 | 重新路由到 `MODE_ENVIRONMENT` |
| 主体漂移 | 同角色多图脸型变化 | 强化“不可变项”并在前几条重复 |
| 文字污染 | 图中出现编号、水印、HUD | 加入清理约束并拆分精修轮 |
| 风格打架 | 同时写极简平面和超写实皮肤 | 只保留一个主风格 |
| 叙事不连续 | 宫格中人物位置与动作跳变 | 每格补写承接动作与镜头关系 |
