# flow.story_to_video.shot_prompt_package 2.0 溶图 skill

## 适用场景

- 上游分镜规划采用 2.0 溶图策略。
- 当前镜头需要通过图生图或溶图完成首帧、特写或状态分镜。
- 视频生成主要依赖首帧，并可结合少量明确职责的参考图。

## 负责什么

- 为图生图、溶图和文生图分镜写自然语言提示。
- 为视频生成写与分镜图一致的画面提示。
- 说明图片之间的融合职责：角色、场景、道具、构图、动作或状态。
- 在首尾帧不得不用时，写清连续动作和硬性结果。

## 不负责什么

- 不把每个动作阶段都写成独立图片。
- 不为了丰富画面添加上游没有规划的参考图。
- 不使用 provider 私有语法或标签串。
- 不把字幕、音效、BGM 直接塞进画面提示。

## 判断原则

- 首帧为主。能靠首帧加语言自然推进的镜头，不强化尾帧。
- 首尾帧只有在必须要的时候才用。首尾帧很难控制从首帧到尾帧的画面连续性和一致性，尽量不用首尾帧。
- 分镜图提示要服务溶图结果：先保留参考图中的稳定元素，再改变当前镜头需要改变的动作、站位、状态或光效。
- 多张图参与溶图时，必须明确每张图的职责，但最终只生成一张当前镜头画面。
- 状态异变要写清“保留什么”和“覆写什么”，避免模型把病弱、破损、战损、枯死等恢复成默认漂亮状态。
- 视频提示词要围绕已生成分镜图，描述一个连续镜头的动作、情绪、镜头和节奏。

## 工作方法

- 先按上游分镜规划逐张确认需要生成的画面。
- 对首帧图，压实主体、场景、动作起点、构图和当前状态。
- 对溶图，先写参考图职责，再写最终单张画面，不写拼贴或多图并列。
- 对特写，聚焦一个局部证据，例如脸、手、伤口、道具、污渍、光效或文字承载面。
- 写视频提示时，保留语言空间：说明动作方向和情绪落点，不逐帧限定姿态。
- 最后复核提示词是否和上游分镜数量一一对应。

## 质量标准

- 图生图和溶图提示能看出每张参考图承担什么职责。
- 分镜图都是单一静帧，不是动作过程或多画面拼贴。
- 视频提示词和首帧图一致，不凭空切换空间或人物。
- 首尾帧使用克制，只服务硬性结果。
- 角色身份、场景布局、道具形状和当前状态不漂移。

## 失败回退

- 溶图关系过多时，删掉重复职责，只保留角色、场景和关键道具。
- 状态难以稳定时，改写成更具体的表面证据和默认特征缺失。
- 视频提示过细时，删掉逐帧姿态，只保留动作方向、情绪落点和镜头节奏。
