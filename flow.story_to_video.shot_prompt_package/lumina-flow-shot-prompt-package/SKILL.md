# flow.story_to_video.shot_prompt_package

## 适用场景

- 已经有镜头说明和资源规划，需要继续生成可执行提示词。

## 负责什么

- 为需要生成的分镜图输出对应 prompt。
- 为当前镜头输出最终视频 prompt。
- 保持画面、台词、氛围和运镜信息在图片/视频阶段一致。
- 对需要生成的 img2img / txt2img frame 生成分镜图提示词，并对整个镜头生成视频提示词。

## 不负责什么

- 不回头改前一阶段的资源规划。
- 不决定任务节点结构。
- 不把 provider 特定语法抬到上游规划层。
- 不把字幕要求写入最终视频 prompt；字幕可保留结构化数据供消息渲染或剪辑层使用。

## 判断原则

- 只为需要生成的图生图或文生图关键帧撰写分镜图提示词。
- reuse_bound 的 frame 直接跳过，不补 prompt。
- 视频 prompt 继承镜头的画面动作、台词剧本、声音层和节奏设计。
- 分镜图提示词要与前一阶段确定的关键帧一一对应，不多写也不少写。
- 不要为未被规划为“需要生成”的关键帧补写提示词。
- 不要改写前一阶段已经确定的资源选择和视频模式。
- 对 `reference_images`，视频 prompt 必须按图片输入顺序描述图1、图2、图3……各自职责。

## 视频提示词要求

视频 prompt 需要包含：

- 主体、空间、动作和镜头运动。
- 多参考图序说明：参考图1中的{元素/职责}，参考图2中的{元素/职责}，参考图3中的{元素/职责}……
- 图序一致性说明：按照图1至图n的参考顺序组织画面，保持人物身份、服装、空间布局、关键道具和构图关系一致。
- 台词按剧本发生：{角色A}以{语气}说“……”，{角色B}以{语气}回应“……”。
- 环境声、音效、BGM 氛围和强度。
- 节奏：慢、中、快，以及镜头推进方式。

## 字幕处理

- 字幕字段可以保留在结构化输出中，用于最终消息渲染、剪辑层或 UI 展示。
- 视频 prompt 不主动生成字幕指令，不写“底部字幕”、不写“气泡字幕”、不写“不要生成字幕”。
- 只强化台词剧本一致性和说话顺序，不把字幕作为画面文字要求。

## 多参考模板

- `参考图1中的{元素/职责}，参考图2中的{元素/职责}，参考图3中的{元素/职责}……`
- `按照图1至图n的参考顺序组织画面，保持人物身份、服装、空间布局、关键道具和构图关系一致。`
- `台词按剧本发生：{角色A}以{语气}说“……”，{角色B}以{语气}回应“……”。`

## 工作方法

- 先核对前一阶段里哪些关键帧需要生成。
- 再逐帧生成 prompt，确保一一对应。
- 最后统一整理视频 prompt，保证图序参考、关键动作和台词顺序不打架。

## 质量标准

- 提示词与关键帧对齐，不漏不多。
- `reference_images` 视频 prompt 显式包含图1到图n的职责描述。
- 视频 prompt 能体现镜头对白、氛围、BGM、音效与运镜节奏，但不包含字幕生成要求。
- 项目风格约束被自然融合，不喧宾夺主。

## 失败回退

- 信息不足时，优先保证镜头主目标和关键动作，不硬补细枝末节。
- 分镜图无法稳定生成时，退回更保守、更直接的表述。
