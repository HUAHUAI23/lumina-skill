# flow.story_to_video.shot_prompt_package

## 适用场景

- 已经有镜头说明和资源规划，需要继续生成可执行提示词。

## 负责什么

- 为需要生成的分镜图输出对应 prompt。
- 为当前镜头输出供编译器使用的视频画面 prompt。
- 保持画面、台词、氛围和运镜信息在图片/视频阶段一致。
- 对需要生成的 img2img / txt2img frame 生成分镜图提示词，并对整个镜头生成视频提示词。

## 不负责什么

- 不回头改前一阶段的资源规划。
- 不决定任务节点结构。
- 不使用 provider 特定语法、SD 标签串或命令行参数。
- 不在 `visualPrompt` 中直接写字幕生成要求；字幕可保留结构化数据供消息渲染、剪辑层或最终编译器使用。
- 不在 `visualPrompt` 中重复改写结构化台词和音频；最终视频任务会由编译器把画面、台词、字幕和声音拼装成统一 prompt。

## 判断原则

- 只为需要生成的图生图或文生图关键帧撰写分镜图提示词。
- reuse_bound 的 frame 直接跳过，不补 prompt。
- `visualPrompt` 继承镜头的画面动作、构图、运镜、光线、情绪和节奏设计。
- 分镜图提示词要与前一阶段确定的关键帧一一对应，不多写也不少写。
- 不要为未被规划为“需要生成”的关键帧补写提示词。
- 不要改写前一阶段已经确定的资源选择和视频模式。
- 自动化流程主要依赖首帧叙事和多参考叙事完成画面；两者都是有效主路径。`visualPrompt` 要围绕输入图片展开，不要把提示词写成需要额外切镜才能成立的分镜脚本。
- 如果视频模式是 `first_last_frame`，`visualPrompt` 必须把图片1到图片2写成同一镜头内的连续动作演化：保持同一空间、同一主体、同一运镜逻辑，明确中间动作如何发生，避免把首帧和尾帧描述成两个分离画面。
- 如果视频模式是 `first_frame`，`visualPrompt` 应强调从首帧开始自然推进的动作过程和叙事结果，让模型靠首帧锚点完成后续演化，不再暗示需要切到另一个固定结果画面。
- 对 `reference_images`，必须按图片输入顺序描述 `图片1`、`图片2`、`图片3`……各自职责。
- 对 `reference_images`，应把多参考写成同一镜头的身份、空间、道具、动作构图和特效约束，而不是多张图依次切换播放。
- 提示词必须使用自然语言，优先正向描述画面中应该出现什么、如何运动、如何变化。
- 禁止使用 `1girl`、`1boy`、`masterpiece`、`best quality`、`--neg`、权重括号、逗号标签堆叠等 SD 风格写法。
- 禁止把英文参考占位写成 `Reference image 1`；统一使用中文 `图片1`、`图片2`、`视频1`。
- 需要负向约束时只写在专门的负向约束位置，并保持简短；不要把负向参数混进主 prompt。
- 角色身份、性别呈现、年龄阶段和服装必须来自项目资产或镜头上下文，不能把男角色写成女角色，不能把成年人写成儿童。

## 视频提示词要求

`visualPrompt` 需要包含：

- 主体：谁在画面中，身份和外观如何保持一致。
- 空间：在哪里，关键陈设、时间、光线、色调和气氛是什么。
- 动作：主体正在做什么，动作如何开始、持续和结束。
- 运镜：固定、推近、拉远、横摇、跟拍、俯冲、特写切换等。
- 情绪：人物表情、肢体停顿、冲突张力或喜剧反应。
- 节奏：慢、中、快，以及镜头推进方式。
- 多参考图序说明：`图片1` 中的{元素/职责}，`图片2` 中的{元素/职责}，`图片3` 中的{元素/职责}……
- 图序一致性说明：按照 `图片1` 至 `图片n` 的参考顺序组织画面，保持人物身份、服装、空间布局、关键道具和构图关系一致。
- 首尾帧连续性说明：当存在首帧和尾帧时，描述“画面从图片1的起始状态连续推进到图片2的结果状态”，并写清主体动作、特效/物体变化、镜头运动和节奏如何贯穿全镜头。
- 首帧叙事说明：当只有首帧或主分镜时，写清“从该画面起始状态继续发生什么”，让动作和结果都从首帧自然延展。

最终视频任务会由编译器把 `visualPrompt` 与结构化台词、字幕、环境声、音效和 BGM 合成统一视频 prompt，以兼容支持音视频联合生成的模型，也兼容只消费画面 prompt 的模型。

## 分镜图提示词要求

- 分镜图 prompt 也使用自然语言，不写标签堆叠。
- `img2img` prompt 必须说明保留参考图中的哪些稳定特征，再说明要新增或改变什么；如果参考图包含已绑定角色或场景，必须明确保留角色身份、性别呈现、年龄阶段、服装、脸部特征、场景布局和关键陈设，再生成新的动作、站位、表情、道具或光效。
- 多张参考图参与 `img2img` 时，开头先写 `图片1`、`图片2`、`图片3` 各自负责的人物、场景、道具或构图职责。
- 多人同框溶图时，不写“保留图片中的两人”这类模糊表达；应写成“夏青卓以图片2为准，乙之以图片3为准，客厅以图片1为准”这类明确绑定。
- 如果同一张溶图同时使用角色图和场景图，先锁定场景图片，再逐个锁定角色图片，最后描述新动作、站位、表情和特效。
- `txt2img` prompt 必须能独立生成画面，包含主体、空间、动作、构图、光线和情绪。
- `txt2img` prompt 只适合没有可用绑定图的临时画面、纯缺失道具、纯特效示意或无绑定主体空镜；如果当前帧涉及已绑定角色/场景，不应把它写成孤立的新人物或新空间。
- 为缺失资产生成临时关键帧时，要明确它是镜头级可见元素，不要声称它已经是项目绑定资产。
- 分镜图要服务后续视频输入，不做海报、封面、立绘或孤立插画，除非 frame 职责明确需要。

## 字幕处理

- 字幕信息可以保留在结构化输出中，用于最终消息渲染、剪辑层或 UI 展示。
- `visualPrompt` 不主动生成字幕指令，不写“底部字幕”、不写“气泡字幕”、不写“不要生成字幕”。
- 只强化台词剧本一致性和说话顺序，不把字幕作为画面文字要求。
- 如果最终模型需要原生字幕，交给编译器按结构化字幕统一追加；不要在本阶段散写。

## 多参考模板

- `图片1` 用于{元素/职责}，`图片2` 用于{元素/职责}，`图片3` 用于{元素/职责}……
- 按照 `图片1` 至 `图片n` 的参考顺序组织画面，保持人物身份、服装、空间布局、关键道具和构图关系一致。
- 如果 `reference_images` 中既有 `reuse_bound` 绑定图又有生成参考图，必须说明生成参考图如何继承前面绑定图的人物身份、服装、空间和道具关系，避免把后续参考图当成新角色或新场景。
- 如果图片序列中包含 `start` 主分镜，把它说明为当前镜头的起始主画面和降级兜底；后续 `ref_n` 是对这个主画面的身份、空间、动作、道具或特效补充，不是切到新镜头。
- 多参考职责应服务同一个镜头：身份图锁角色，场景图锁空间，动作构图图锁站位和动作，特效/道具图锁局部细节；不要写成“先显示图片1，再切图片2，再切图片3”。
- 如果参考的是视频，使用 `视频1` 的{动作/运镜/特效}作为参考，不写模糊的“参考上面素材”。

## 通用视频模型写法

- 使用清楚的自然语言句子，少用抽象形容词，多写可见动作和可听声音。
- 先写主体和动作，再写空间与镜头，最后写情绪、节奏和风格。
- 用正向表达替代否定堆叠，例如写“镜头稳定、人物面部清晰、动作连贯”，不要堆大量“不要模糊、不要变形”。
- 一条 prompt 只强调当前镜头最重要的视觉目标，避免把整集设定塞进单镜头。
- 运镜要具体，例如“镜头从门口缓慢推近到人物肩部近景”，不要只写“电影感运镜”。
- 对首尾帧镜头，运镜要写成单次连续运镜，例如“镜头从中景持续推近，动作在同一视角中完成，雷光/烟尘/人物反应把起始状态自然带到结果状态”；不要把画面写成先出现图片1、再切到图片2。
- 对首帧和多参考镜头，运镜同样保持单次连续镜头；如果需要结果画面，写成动作自然发展或余波显现，不写“切到结果画面”。
- 台词、字幕、音效和 BGM 由结构化信息承载；本阶段只确保画面能承接这些声音信息。

## 工作方法

- 先核对前一阶段里哪些关键帧需要生成。
- 再逐帧生成自然语言 prompt，确保一一对应。
- 检查每条 prompt 是否能独立回答“谁、在哪、做什么、镜头怎么拍、情绪是什么”。
- 对 `img2img` frame，检查 prompt 是否明确继承了所选图片的身份、服装和空间；如果只是重新描述一个新画面，应补上“保留图片中的稳定特征”。
- 对 `first_last_frame` 的 `visualPrompt`，检查它是否像一个连续镜头，而不是两个镜头说明；如果读起来像硬切，应改成首帧叙事或多参考连续动作表达。
- 对 `first_frame` 和 `reference_images` 的 `visualPrompt`，检查是否充分利用了输入图片的职责；如果只是泛泛描述剧情，应补充图片职责和一致性约束。
- 最后统一整理 `visualPrompt`，保证图序参考、关键动作、情绪承接和后续台词不打架。

## 质量标准

- 提示词与关键帧对齐，不漏不多。
- `reference_images` 的 `visualPrompt` 显式包含 `图片1` 到 `图片n` 的职责描述。
- `visualPrompt` 能体现主体、动作、空间、构图、光线、情绪与运镜节奏，但不直接混写字幕和音频。
- 首尾帧提示词必须能看出“从起点到结果”的连续动作链条，不能只列两个状态。
- 首帧和多参考提示词必须能看出“输入图片如何约束最终画面”，不能只复述剧情。
- 已绑定角色/场景参与的生成帧，提示词必须体现参考图继承关系，避免身份漂移和场景漂移。
- 项目风格约束被自然融合，不喧宾夺主。
- 不出现 SD 标签串、英文参考占位、provider 参数或与角色资产冲突的性别/年龄描述。
- 缺失资产的文生图 prompt 足够具体，能生成可用于画布继续执行的临时关键帧。

## 失败回退

- 信息不足时，优先保证镜头主目标和关键动作，不硬补细枝末节。
- 分镜图无法稳定生成时，退回更保守、更直接的表述。
- 多参考关系过复杂时，减少参考数量，保留 `图片1` 主体、`图片2` 空间、`图片3` 关键道具或动作构图。
