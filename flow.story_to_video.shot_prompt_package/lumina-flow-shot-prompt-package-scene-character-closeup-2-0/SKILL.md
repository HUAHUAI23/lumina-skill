# flow.story_to_video.shot_prompt_package 2.0 场景人物加特写 skill

## 适用场景

- 上游分镜规划采用角色加场景主图和必要特写的轻量策略。
- 当前镜头不需要大量过程分镜，只需要稳定人物、空间和关键局部。
- 需要让视频模型通过较宽松的语言控制自然发挥。

## 负责什么

- 为角色加场景主图写清晰但不过度限定的分镜图提示。
- 为必要特写写聚焦局部证据的提示。
- 为视频生成写简洁自然的画面提示，保留动作和表演空间。
- 保持角色身份、服装、场景关系和关键道具一致。

## 不负责什么

- 不写大量动作过程图提示。
- 不把视频提示写成逐帧姿态说明。
- 不用过度精确的语言限制模型发挥。
- 不为装饰性细节额外补图。

## 判断原则

- 首帧为主。角色加场景主图是大多数镜头的核心输入。
- 首尾帧只有在必须要的时候才用。首尾帧很难控制从首帧到尾帧的画面连续性和一致性，尽量不用首尾帧。
- 图片只需要角色加场景图，加某些特写镜头。
- 语言控制要轻。只写主体、动作方向、情绪落点、镜头节奏和不能误读的关键点。
- 越精确的连续姿态越会限制模型发挥；避免逐帧描述手脚位置和每一步动作。
- 特写图只用于剧情理解必须看清的局部，不作为装饰。

## 工作方法

- 先写主图：角色是谁、位于哪个场景、当前站位或动作起点、镜头景别和气氛。
- 如果需要特写，只写一个局部目标，例如眼神、手、伤口、道具、证据、文字承载面或光效。
- 主图提示保留角色和场景一致性，但不把后续动作全写死。
- 视频提示使用简洁句子：从主图开始，人物顺着动作方向运动，情绪到达目标落点，镜头保持稳定节奏。
- 对台词和声音，只让画面承接语气，不复述台词内容。
- 最后删掉过度精确的过程词和不必要细节。

## 质量标准

- 主图提示可以生成稳定的角色加场景画面。
- 特写提示只解决一个关键理解问题。
- 视频提示词简洁、有方向、有情绪，但不过度限定模型。
- 不出现大量参考图职责说明或逐帧过程控制。
- 首尾帧极少出现，且只用于不可拆的硬结果。

## 失败回退

- 提示过长时，删掉次要装饰和逐帧动作，只保留主体、场景、动作方向、情绪和镜头。
- 特写太多时，只保留最影响剧情理解的一张。
- 模型容易跑偏时，加强主图里的身份和场景锚点，而不是增加过程图。
