# flow.story_to_video.shot_prompt_package 2.0 溶图场景人物加特写 skill

## 适用场景

- 上游分镜规划采用角色加场景主图、人物身份参考和按需关键特写的职责清晰策略。
- 当前镜头不需要大量过程分镜，只需要稳定人物、空间和关键局部。
- 需要让视频模型通过较宽松的语言控制自然发挥。
- 分镜文本可能带有时间段、VO、OS、对白或固定画风，但图片输入仍应保持克制。

## 负责什么

- 为角色加场景主图写清晰但不过度限定的分镜图提示。
- 为按需关键特写写聚焦局部证据的提示。
- 为视频生成写“主题 + 运动”式画面提示，保留动作和表演空间。
- 保持角色身份、服装、场景关系和关键道具一致。
- 把台词、旁白和 OS 转化为画面情绪、停顿、视线、姿态和节奏承接。

## 不负责什么

- 不写大量动作过程图提示。
- 不把视频提示写成逐帧姿态说明。
- 不用过度精确的语言限制模型发挥。
- 不为装饰性细节额外补图。
- 不把时间轴中的每一小段都改写成单独图片提示。
- 不把 VO、OS、对白原文塞进画面提示。
- 不写平台参数、命令行参数、标签串或私有引用语法。

## 判断原则

- 首帧为主。角色加场景主图是大多数镜头的核心输入。
- 首尾帧只有在必须要的时候才用。首尾帧很难控制从首帧到尾帧的画面连续性和一致性，尽量不用首尾帧。
- 图片类型集中在角色加场景图、人物身份参考和关键特写镜头，不限定为低数量组合。
- Seedance 2.0 最多支持 9 张图片；多人物和多个关键特写可以使用多图，重点是每张图职责清楚，避免职责混杂和普通过程帧。
- 语言控制要轻。只写主体、动作方向、主运镜、情绪落点和不能误读的关键点。
- 核心是主题加运动：谁在什么场景里，从什么状态开始，朝哪个动作方向推进，镜头如何跟随，情绪落到哪里。
- 越精确的连续姿态越会限制模型发挥；避免逐帧描述手脚位置、每一步动作和每一秒的表情。
- 特写图只用于剧情理解必须看清的局部，不作为装饰。
- 对短剧对白和仙侠戏，画面提示优先写可见动作和身体信号，例如猛然睁眼、嘴唇微动、手指惨白、宵夜滑落、平静回望。
- 对 VO、OS、旁白，只写画面如何成为旁白证据，不复述旁白内容。
- 对准确文字、招牌、纸面内容，主图或特写只提供材质、留白、模糊痕迹或遮挡区域，不要求模型生成可读长句。

## 图片顺序和编号规范

- 所有参考图片都按输入顺序称为 `图片1`、`图片2`、`图片3`，一直到 `图片n`。
- 不根据角色重要性、场景重要性或提示词表达习惯重排图片编号；上游给出的顺序就是最终说明顺序。
- 不写 `图1`、`图2`、`Reference image 1`、`上图`、`前一张图` 这类不稳定指代。
- 图生图或溶图提示开头先说明图片职责，例如 `图片1` 用于场景空间，`图片2` 用于林照月身份和服装，`图片3` 用于黑袍男人外观。
- 视频提示开头也要用一句短句声明图片职责，例如：`图片1` 为角色加场景首帧，`图片2` 为林照月眼部特写，`图片3` 为黑袍男人手部特写。
- 如果角色加场景主图由多张图溶成，先按 `图片1` 到 `图片n` 写清每张图负责什么，再写最终只生成一个当前镜头画面。
- 如果只有一张首帧图，仍称为 `图片1`，并说明视频从 `图片1` 的起始画面自然动起来。
- 如果确实使用首尾帧，`图片1` 是首帧，`图片2` 是尾帧；视频提示要写成同一镜头内从 `图片1` 连续推进到 `图片2`，不能写成两张图切换。
- 如果上游不得不使用多参考视频，仍要在视频提示里按 `图片1` 至 `图片n` 逐一说明职责，但保持轻量，不扩展成大量过程分镜。
- 当图片数量达到 7-9 张时，图片职责说明仍要完整，但每张只写一个短职责，避免让职责说明本身变成冗长噪点。

## Seedance 2.0 风格压缩

- 画风、时长、比例、声音这类固定信息只在视频提示里保留为上层约束，不扩写到每张图片里。
- 用户输入是长时间轴时，保留时间段文字作为视频语言控制；图片提示只写角色加场景主图、人物参考和关键特写，不为普通动作过程额外写图。
- 视频提示可以采用短时间片写法，例如“0.0-1.0 秒：眼部特写固定镜头……”，但每个时间片不对应一张新图。
- 对短剧对白，画面和台词可以同段出现；画面负责动作、表情和镜头，台词保持简短，不把台词转成画面文字。
- 对旁白和 OS，画面承接叙事，不直接生成字幕或可读文字。
- 对仙侠和短剧场景，优先写清“人物 + 场景 + 动作方向 + 关键特写”；如果有多个人物或多个关键特写，可以保留多图，但不要把普通过程帧也混进去。
- 稳定性约束保持简短，例如人物面部稳定、动作自然、画面无闪烁；不要堆过长负面约束。

## 主图提示写法

- 先写画面职责：角色加场景主图、当前镜头首帧、动作起点或关系建立。
- 再写参考继承：`图片1`、`图片2`、`图片3` 各自锁定什么，例如角色身份和服装、场景空间和光线、道具形状。
- 然后写当前瞬间：人物站位、动作起势、表情方向、场景氛围、景别和角度。
- 最后写轻约束：身份稳定、面部清晰、空间关系清楚、不要多余人物或错误道具。
- 主图不要写完整剧情过程，不写“随后、接着、最后”，只写一张能动起来的起始画面。

## 特写提示写法

- 眼部特写：写眼神方向、情绪温度、倒影或灵力变化，不写整段对话。
- 嘴唇特写：写嘴唇微动、吐字力度、面部局部光线，不写长台词。
- 手部特写：写手指形态、握持关系、掉落瞬间、血迹、黑气或法术痕迹。
- 道具特写：写形状、材质、光效、污损、归属关系和画面位置。
- 文字承载面特写：写空白、半模糊、被遮挡、污损或留白区域；准确文字由后续层处理。
- 每张特写只解决一个问题，不同时承担脸、手、道具、环境全貌。

## 视频提示写法

- 开头先声明图片职责，再进入时间轴或连续镜头描述；图片职责要短，不抢占主要画面描述。
- 视频提示应从主图自然动起来，围绕一个连续镜头组织，不写多个剪辑点。
- 使用轻量时间感，而不是逐秒控制；可以写“短暂停顿后”“镜头缓慢推近”“动作顺势继续”，但不要把每一秒都锁死。
- 如果用户已经给了清晰时间轴，可以保留这些时间段和镜头词，但压缩细节，只留下主体、动作、场景、光线、声音和稳定性约束。
- 有对白时，画面承接说话语气：冷声对应嘴唇轻动和眼神压低；震惊对应手中物品滑落和身体停住；无语对应嘴角微抽和视线停顿。
- 有 VO 或 OS 时，画面承接叙事重点：空客厅、门口推进、手部特写、低角度脸部、反应镜头等都作为旁白证据链。
- 只指定一种主运镜，例如固定镜头、缓慢推近、轻微跟拍、低角度近景；不要同时堆推拉摇移。
- 动作写方向，不写逐帧姿态。例如写“她从静止中抬眼看向门口”，不要写每一帧眼皮和头部角度。
- 情绪写可见表现，不只写抽象词。例如写“眼神冰冷、肩背不动、嘴唇轻轻吐字”，不要只写“非常冷酷”。
- 画风、光线和稳定性约束保持简短，服务画面，不喧宾夺主。

## 工作方法

- 先读上游分镜，提取主题和运动：主体、场景、动作方向、主运镜、情绪落点。
- 再写主图：角色是谁、位于哪个场景、当前站位或动作起点、镜头景别和气氛。
- 如果需要特写，每张特写只写一个局部目标，例如眼神、嘴唇、手、伤口、道具、证据、文字承载面或光效。
- 主图提示保留角色和场景一致性，但不把后续动作全写死。
- 视频提示使用简洁句子：从主图开始，人物顺着动作方向运动，情绪到达目标落点，镜头保持稳定节奏。
- 对台词、VO、OS 和声音，只让画面承接语气和叙事功能，不复述原文。
- 如果上游分镜已经按时间段写得很细，要压缩成“主图 + 按需关键特写 + 主题运动”，不要逐段照抄。
- 最后删掉过度精确的过程词和不必要细节。

## 质量标准

- 主图提示可以生成稳定的角色加场景画面。
- 特写提示只解决一个关键理解问题。
- 视频提示简洁、有方向、有情绪，但不过度限定模型。
- 即使用到 7-9 张图片，也只写短职责说明，不扩展成逐帧过程控制。
- 首尾帧极少出现，且只用于不可拆的硬结果。
- 能从提示里看出“主题是谁”和“运动如何发生”。
- 台词、VO、OS 的信息被画面动作承接，而不是被画面文字复述。
- 主图和特写都不变成海报、立绘、分镜拼贴或多动作合集。
- 角色身份、性别呈现、年龄阶段、服装和场景关系不漂移。

## 失败回退

- 提示过长时，删掉次要装饰和逐帧动作，只保留主体、场景、动作方向、情绪和镜头。
- 特写太多时，删除装饰性和重复特写，保留所有影响剧情理解的关键特写，总数控制在 9 张以内。
- 模型容易跑偏时，加强主图里的身份和场景锚点，而不是增加过程图。
- 首尾帧连续性不稳时，退回首帧叙事，或把硬结果交给下一镜头首帧。
- 画面过于静态时，不补新图，先把语言里的动作方向写清楚。
