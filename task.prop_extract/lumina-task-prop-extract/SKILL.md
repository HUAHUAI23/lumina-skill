# task.prop_extract

## 适用场景

- 从文本、剧本、故事段落中抽取稳定道具资产。

## 负责什么

- 优先把当前文本中的道具提及映射到已存在的道具资产。
- 找出会影响后续镜头连续性、剧情推进或视觉识别的关键道具。
- 为每个道具给出稳定称呼和简明描述。
- 合并同一道具的不同叫法，避免重复沉淀。
- 为后续图生图保留稳定物体锚点，例如类别、材质、轮廓、颜色、尺寸、磨损状态、标记纹样和使用方式。

## 不负责什么

- 不抽取人物和场景。
- 不把一次性背景杂物当成稳定道具资产。
- 不扩写道具来历或剧情意义，除非原文明确支持。
- 不把有行动意志、可反复出现的妖怪、怪兽、动物伙伴、灵兽抽成道具资产；它们应进入人物资产。

## 判断原则

- 优先抽灵物、法器、武器、信物、关键文件、反复出现且需要视觉连续的物件。
- 同一道具保持一个稳定称呼，不发明多个别名。
- 描述聚焦形状、材质、颜色、年代感、尺寸感、发光/纹理等识别特征。
- 不确定但明显重要的道具放入待解析，不强行新建。
- 衣服、鞋、首饰如果主要服务人物外观，优先写入人物 costume_style / costume_colors / signature_props；只有作为独立剧情物件反复出现时才新增道具。
- 植物、树苗、农作物如果主要构成地点氛围，优先作为场景 key_elements；只有被单独操作、反复特写或承载剧情线索时才作为道具。
- 动物、妖怪、灵兽如果有名字、行动意志、反复出现或会影响镜头连续性，按人物资产处理；只有尸骸、标本、雕像、图腾等物体化存在才可作为道具。
- 普通背景杂物、消耗品、一次性工具、泛称物品通常不新增；但如果它反复出现并承载情感或行动连续性，可以新增。

## 状态、文字面与使用痕迹

- 道具资产要记录“稳定形状”和“使用后的表面状态”。如果道具的意义来自磨损、血迹、泥污、裂纹、字迹模糊、烧焦、褪色、泡烂、边缘卷曲或多年使用，必须写入 wear_state、damage_state、history_traces 或 texture。
- 同一道具跨阶段变化时，不要默认拆成多个道具；优先用 state_variants 记录，例如“新挂上时字迹清楚，风雪泥水后字迹模糊”“锄头边缘从磨秃到发亮”“作业本最后一页有歪斜遗言”。
- 文本、手写字、符号、编号和标记要拆成两类信息：key_markings 记录内容意图，text_surface 记录承载材质与留白/笔迹形态。不要把长篇准确中文可读性当成稳定生图能力。
- 如果后续必须准确阅读文字，应在属性中提示“准确文字适合后期贴图/字幕层叠加”；道具参考图只需稳定纸张、木牌、墨迹、刻痕、留白区域和年代质感。
- 对会被模型默认美化的道具，应记录排他性缺失：不是崭新、不是光滑、不是干净、不是现代塑料感、不是完整无损，具体以原文事实为准。

## 工作方法

- 先判断文本里的道具是否已经能复用到现有道具。
- 只有无法复用、且值得长期维护的道具才新增。
- 优先补充这些稳定属性：prop_type、material、shape、color、size_scale、age_style、wear_state、damage_state、history_traces、state_variants、key_markings、text_surface、texture、usage、glow_effect。
- attributeEntries 只保留稳定可见特征和叙事用途，不把一次性摆放位置、临时动作或镜头效果写成稳定属性。
- 如果道具由人物长期使用或继承，要记录与人物关系和手感痕迹，例如“被长期握持磨亮的木柄”“被父亲装进口袋的旧作业本”；这些信息帮助后续镜头建立情感连续性。
- 最终保留短而准、可复用的道具说明；新增道具会由下游链路继续生成图片任务规划。

## 质量标准

- 道具列表干净，不重复。
- 名称稳定，描述足以支撑后续视觉和镜头连续。
- 每个道具资产应能独立生成清晰单体参考图，并能在后续分镜里被准确引用。
- 关键道具应能回答“它经历过什么、表面留下什么痕迹、哪些默认干净/崭新特征不能出现、文字是否需要后期精确覆盖”。

## 失败回退

- 边界模糊时，宁可少收也不误收一次性杂物。
- 不臆造文本里没有出现的品牌、机关或隐藏结构。
- 如果物件只是人物服装或场景陈设的一部分，优先不新建道具资产。
