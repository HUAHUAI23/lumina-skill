# task.prop_extract

## 适用场景

- 从文本、剧本、故事段落中抽取稳定道具资产。
- 为后续道具干净单体参考图、必要状态/角度图、分镜连续性和视频镜头复用建立道具库。

## 负责什么

- 优先把当前文本中的道具提及映射到已存在道具资产，合并同一道具的别名、简称和描述性称呼。
- 找出会影响后续镜头连续性、剧情推进、情感伏笔或视觉识别的关键道具。
- 为每个道具保留可生图锚点：类别、轮廓、结构、材质、颜色、尺寸比例、磨损状态、纹理、发光效果、标记和使用痕迹。
- 识别同一道具的稳定状态变化，例如崭新/旧损、完整/断裂、干净/带血、字迹清晰/字迹模糊、未激活/发光激活。
- 为后续判断是否需要单体基准图、状态图或角度图留下依据；材质/手持/局部特写下沉到 story_to_video 镜头阶段。

## 不负责什么

- 不抽取人物和场景。
- 不把一次性背景杂物、消耗品、摆设噪点当成稳定道具资产。
- 不扩写道具来历、品牌、机关或隐藏能力，除非原文明确支持。
- 不把有行动意志、可反复出现的妖怪、怪兽、动物伙伴、灵兽抽成道具资产；它们应进入人物资产。
- 不把人物衣服、首饰、武器佩戴方式重复拆成道具；只有它们被单独操作、交付、特写或跨场景复用时才独立沉淀。

## 判断原则

- 优先抽灵物、法器、武器、信物、钥匙、文件、药剂、遗物、证据、交通工具、机械装置、反复出现且需要视觉连续的物件。
- 同一道具保持一个稳定称呼，不因“长剑、飞剑、本命飞剑”这类语境变化拆成多个物体；若确有成套关系，应记录成套数量、统一形制和差异点。
- 描述聚焦可见识别特征：形状、结构、材质、颜色、尺寸、重量感、纹理、年代感、损坏、污渍、标记、发光或能量效果。
- 普通背景杂物通常不新增；如果它反复出现、被人物操作、承载情感或作为关键证据，才成为道具资产。
- 植物、树苗、农作物如果主要构成地点氛围，优先归入场景；只有被单独操作、反复特写、作为证据或承载剧情线索时才作为道具。
- 尸骸、标本、雕像、图腾、画像等物体化存在可以作为道具；有行动意志的生物不作为道具。
- 文本只给抽象意义时，要转成可见承载物。例如“护身符代表爱意”应记录符纸、材质、折痕、金光、随身磨损，而不是只写“重要信物”。

## 状态、文字面与使用痕迹

- 道具资产要记录稳定形状，也要记录会影响后续镜头的表面状态。
- 如果道具的意义来自旧、破、脏、血迹、泥水、裂纹、烧焦、褪色、泡烂、边缘卷曲、字迹模糊或多年使用，这些状态必须作为主体特征保存。
- 同一道具跨阶段变化时，不要默认拆成多个道具；优先记录为状态变化，例如“新挂上时字迹清楚，风雪泥水后字迹模糊”“刀刃从完整到崩口”“符纸从暗淡到金光激活”。
- 对会被模型默认美化的道具，应显式记录缺失项：不是崭新、不是光滑、不是干净商品图、不是现代塑料感、不是完整无损。
- 文本、手写字、符号、编号和标记要分清“内容意图”和“可生成承载面”。准确长中文、小字、复杂文件内容不应依赖生图模型稳定生成；道具资产只需记录纸张、木牌、刻痕、墨迹、留白区域和粗略书写痕迹。
- 如果后续必须准确阅读文字，应提示适合后期贴图或字幕层覆盖，不把准确可读文字当作道具参考图的唯一识别点。

## 资产线索沉淀

- 抽取阶段必须把后续图生图道具状态资产需要的线索沉淀到 summary 或 attributeEntries，不直接写图片提示词。
- 默认只为同一道具留下一个基准资产需求；不要因为文本里出现轻微角度、手持、污渍、局部特写或一次性动作，就倾向生成多个图。
- attributeEntries 不要求固定 key；优先用少量自然语言说明资产需求，例如“需要干净单体锚点”。只有后续确实会复用不同状态/角度时，才记录“激活发光状态图”“破损带血状态图”等派生需求。
- 每条资产线索都要能回答：唯一道具身份是什么、这张资产用于什么状态/角度、保持哪些道具锚点不变、只改变什么可见内容、什么时候复用。
- 材质微距、手持、刻痕、符文、血迹、墨迹、磨损、裂纹或开合局部只作为镜头级需求记录，不作为资产库图片长期维护。
- 新增或增量抽取时，如果状态变化只在当前镜头短暂出现，优先更新同一道具 summary；只有它会跨镜头复用、改变道具识别、或必须稳定生成同一状态，才沉淀为后续图生图派生资产。

## 多图克制规则

- 先做“是否需要多图”的判断，再决定 attributeEntries；不要把 variants 当默认产物。
- 一张基准单体图足以覆盖：常规展示、轻微手持、普通摆放、临时反光、一次性污点、镜头内局部特写。
- 只有三类情况才建议多图：需要看清体积/背面/开合结构的多角度；同一道具存在会复用的多种形态或状态；同一道具在剧情中发生稳定阶段变化且后续镜头需要保持。
- 如果某个状态是道具首次出现且之后一直维持的主状态，例如“旧损带血的匕首”从头到尾都是这个状态，就把它写进基准图，不再额外生成“干净匕首 + 带血匕首”两张。
- 增量抽取遇到已有道具的新状态时，更新 existingEntityUpdates，记录“基于已有基准图做图生图派生”的需求；不要新建同名道具，也不要重复生成新的基准图。

## 边界规则

- 人物长期穿戴但不单独行动的衣服、鞋子、首饰，优先作为人物服饰或标志物；被摘下、交付、遗失、特写或成为证据时，才考虑道具。
- 场景内固定陈设如床、门、墙、牌匾、树木，优先作为场景元素；被人物搬动、隐藏、打开、反复特写或独立承载线索时，才考虑道具。
- 武器如果只是人物战斗态的一部分，可作为人物随身标志；如果武器本身有名称、属性、传承、成套关系或独立特写需求，应作为道具。
- 成套道具要记录“套装身份”和单体识别方式，例如三十六柄飞剑、同款金属剑身、统一排列、冷冽反光，而不是生成三十六个独立道具资产。

## 工作方法

- 先判断文本里的物体是否已经能复用到现有道具。
- 决定新增前先问：它是否会复用？是否被操作、交付、隐藏、破坏、特写或作为证据？是否有足够稳定可见特征？
- 抽取时按“类别 + 轮廓结构 + 材质 + 颜色 + 尺寸比例 + 表面状态 + 标记/文字面 + 使用关系 + 状态变化”检查。
- 只保留稳定可见特征和叙事用途，不把一次性摆放位置、临时动作、镜头效果写成稳定属性。
- 如果道具由人物长期使用或继承，要记录与人物关系和手感痕迹，例如被长期握持磨亮的木柄、贴身折皱的护身符、沾着泥水的旧作业本。
- 最终道具说明要短而准：说明是什么、为什么需要复用、长什么样、表面状态是什么、是否需要单体、状态或角度资产。

## 质量标准

- 道具列表干净、不重复，名称稳定。
- 每个关键道具都能独立生成清晰单体参考图，并能在后续分镜里被准确引用。
- 关键道具能回答：轮廓是什么、主要材质是什么、尺寸比例如何、经历过什么、表面留下什么痕迹、哪些默认干净/崭新特征不能出现。
- 涉及文字面的道具不会把准确长文本当作唯一识别方式。
- 成套道具能保持套装关系，不被拆散成无关物体。

## 失败回退

- 边界模糊时，宁可少收也不误收一次性杂物。
- 信息不足时，优先保住名称、类别、轮廓和核心材质，不强加复杂花纹、品牌、机关或奇幻效果。
- 如果物件只是人物服装或场景陈设的一部分，优先不新建道具资产。
