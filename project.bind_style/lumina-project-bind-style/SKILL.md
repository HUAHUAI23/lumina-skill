# project.bind_style

## 适用场景

- 用户希望为整个项目绑定长期稳定的视觉风格，用于后续人物、场景、生图和生视频任务。
- 输入可能是纯文字、参考图、参考图加文字说明，或对已有风格的补充/替换。
- 用户可能是在首次定义风格，也可能是在补充、修正或替换已有风格。

## 负责什么

- 提炼适合长期复用的项目视觉方向。
- 同时沉淀生图风格和生视频风格两套约束。
- 把媒介、色彩、材质、镜头、光影、构图、运动节奏和整体气质压实成可投喂的风格表达。
- 从参考图中反推风格，而不是复制图里的主体、场景、构图偶然性或剧情动作。
- 当项目已有风格时，做合并与收束，而不是机械覆盖。

## 不负责什么

- 不绑定人物、场景、道具身份。
- 不把单张参考图里的具体角色、构图或剧情动作误当成项目长期风格。
- 不创建图片任务、视频任务或其他结构化工作流。
- 不扩写世界观、剧情设定或角色关系。
- 不写平台专属参数、权重语法、模型名或工具调用方式。
- 不把某位艺术家、在世创作者、影视 IP、品牌视觉或受保护角色当成项目风格捷径；要转写为可执行的通用视觉语言。

## 判断原则

- 优先抽象稳定风格语言，而不是复述参考图表层内容。
- 文字说明优先决定方向，参考图负责校准质感和视觉落点。
- 生图风格强调单帧画面质量、构图与材质表达。
- 生视频风格强调镜头语言、运动节奏、时序氛围和动态一致性。
- 风格应长期可复用，不能过度依赖某一张图的偶然细节。
- 风格提示词要具体但不臃肿；优先保留 4-8 个最能改变结果的风格锚点。
- 正向描述优先。不要依赖大量“不许、不要、禁止”式表达来定义风格。

## 核心反推流程

1. 先判断用户的目标：新建风格、加强已有风格、替换已有风格，还是只从参考图提炼风格。
2. 读取文字意图：题材方向、审美关键词、平台用途、用户明确想保留或排除的点。
3. 读取参考图风格：媒介/渲染方式、色彩体系、明暗关系、光源、镜头/景别、构图秩序、材质纹理、线条边缘、细节密度、后期质感、情绪氛围。
4. 剥离非风格内容：具体人物身份、物件、地点、剧情动作、单张图姿势、偶然遮挡、文字水印、logo、一次性构图。
5. 合成长期风格：用少量高影响锚点描述“以后所有画面都应如何看起来”，而不是描述“这张图里有什么”。
6. 分成生图和生视频两套表达：生图写静帧视觉，生视频在同一视觉基础上补运动、镜头和节奏。

## 参考图反推规范

- 单张参考图只提供风格证据，不自动提供项目主体。提取“看起来像什么”，不要提取“画了谁、在哪里、发生什么”。
- 多张参考图要找共同分母：重复出现的色彩、光影、材质、镜头、线条、颗粒、构图和情绪才适合进入项目风格。
- 如果多张参考互相冲突，优先用户文字说明；没有明确说明时，保留最稳定的主方向，并把冲突点放入参考备注。
- 如果参考图是人物图，风格提取聚焦画风、打光、色彩、质感、景深和镜头；不要把人物脸、服装、姿势或身份写成全局风格。
- 如果参考图是场景图，风格提取聚焦空间质感、光线、色彩、透视、材质和氛围；不要把具体地点或陈设写成全局风格。
- 如果参考图有明显水印、字幕、界面截图、社媒边框或压缩噪点，这些通常是污染，不进入风格。

## 文字描述反推规范

- 把抽象词翻译成可见视觉语言：
  - “高级” -> 克制色彩、干净构图、低噪点、精确材质、柔和主光。
  - “电影感” -> 电影剧照构图、自然肤色、方向性光线、浅景深或分层景深、轻微胶片颗粒。
  - “压抑” -> 低照度、低饱和、冷暖弱对比、阴影占比高、人物空间压迫感。
  - “仙侠” -> 中国奇幻语汇、飘逸衣料、体积光、云雾层次、玉石/金属/丝绸材质。
  - “短剧感” -> 真实人物比例、清晰面部、室内外生活化光线、近景对话友好。
- 用户只给风格词时，不强行添加复杂镜头或世界观；先形成清晰、简短、可长期复用的风格总纲。
- 用户描述很细时，压缩为稳定锚点，去掉只服务单个画面的内容。

## 生图风格写法

- 生图风格建议覆盖这些方面中的关键项：画面媒介、渲染/摄影方式、色彩体系、光影、构图、镜头/景深、材质纹理、细节密度、后期质感。
- 生图 prompt 适合写成一段短语或短句集合，例如：
  - `3D国漫奇幻角色与场景风格，冷暖对比的月光与法术体积光，清晰面部与服饰材质，玉石、金属、丝绸和雾气层次丰富，电影剧照式构图，低噪点，高一致性。`
- 生图负向约束只写长期通用风险：低清晰度、过度噪点、文字水印、风格混乱、廉价塑料感、脸部崩坏等。
- 不把“某角色站在某处、某道具在画面中央、镜头从左到右”写进项目生图风格。

## 生视频风格写法

- 生视频风格要继承生图视觉锚点，再补充镜头和时间感：主运镜倾向、运动速度、剪辑节奏、动态模糊、景深变化、光影随时间的变化、动作自然度和连续性。
- 视频 prompt 要偏正向视觉描述，少用负向句。把“不要乱动”改成“固定机位、主体动作克制、镜头稳定”；把“不要闪烁”改成“光线连续、画面稳定、运动自然”。
- 如果项目主要做短剧/对白，视频风格应强调近景/中景友好、面部稳定、自然对话节奏、轻微推镜或固定镜头。
- 如果项目主要做奇幻/动作，视频风格应强调慢速推进、体积光和粒子随动作流动、衣料和发丝自然摆动、镜头运动不要过度复杂。
- 不把完整分镜时间轴写进项目风格；项目风格只定义全局运动气质和镜头偏好。

## 合并已有风格

- 用户说“换成/改成/不要原来的”时，按替换处理，旧风格只保留不冲突的通用质量要求。
- 用户说“加一点/更像/偏向”时，按补充处理，保留已有主方向，只吸收新输入里最明确的差异。
- 用户上传新参考图但没有文字说明时，默认把它作为校准参考，不完全覆盖已有风格。
- 合并后要删除内部冲突，例如同时要求“写实电影摄影”和“扁平赛璐璐线稿”时，必须确定主风格，另一个只能作为轻微特征或舍弃。

## 工作方法

- 先判断用户想保留的核心风格方向是什么。
- 再区分哪些属于项目长期风格，哪些只是参考图里的局部内容。
- 对图文混合输入，综合文字意图与图像气质，收束成统一风格表达。
- 对已有项目风格，优先延续主方向，只吸收用户这次明确补充的差异。

## 质量标准

- 风格摘要一句话能说明项目整体视觉方向。
- 风格表达稳定，不会频繁漂移。
- 生图与生视频的风格约束既一致又各有侧重。
- 语言直接、具体、可执行，避免空泛审美词堆砌。
- 生图提示词能直接并入人物、场景、道具、分镜图任务。
- 生视频提示词能直接并入视频任务，且不会把普通镜头变成复杂多镜头脚本。
- 参考备注能说明哪些输入被当作风格来源，哪些内容被剥离为非风格。
- 最终结果能被后续人物、场景、图片任务、视频任务持续复用，并让同一项目的不同画面看起来属于同一视觉体系。

## 失败回退

- 用户意图模糊时，优先保留最明确的风格信号。
- 参考图信息复杂时，宁可收敛成更稳的主方向，也不要拼贴互相冲突的风格。
- 无法确认的局部细节不要强行写成长期项目约束。
