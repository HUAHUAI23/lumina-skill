# 分镜脚本工作流（模式 B / B-Review / B-Ad）

本工作流用于把【小说章节 / 剧本片段 / 多模态素材】转成可执行分镜；或对现有动画分镜进行导演级优化。未指定平台时，默认按 Seedance 2.0 交付。

---

## 0. 工作流总览

```text
输入契约检查
  -> 任务分支判断（B-Create / B-Review / B-Ad）
  -> 素材标准化（含九宫格识别）
  -> 动态检索与平台规则同步
  -> 场次规划
  -> 镜头拆解
  -> 视听填充 + 平台化 Prompt
  -> 验证与回路修订
  -> 最终交付
```

### 分支定义
- `B-Create`：从文本/图片/视频参考创建完整分镜
- `B-Review`：在不推翻原剧情与人设的前提下优化现有分镜（偏动画制作）
- `B-Ad`：广告/品牌片分镜设计，强调卖点表达、品牌一致性与 CTA 转化

### B-Ad 触发词
- 广告、品牌片、产品片、宣传片、种草、带货、口播、转化、CTA、卖点

## 1. 输入契约（必须先满足）

### 1.1 必填字段

| 字段 | 说明 | 缺失处理 |
| --- | --- | --- |
| 任务类型 | 新建分镜 or 优化分镜 | 必问 |
| 核心叙事目标 | 本场戏要传达什么 | 必问 |
| 目标平台 | Seedance 2.0/Sora/Runway/Pika/通用 | 未指定默认 Seedance 2.0 |
| 素材清单 | 文本、图、视频、音频 | 必问 |
| 时长与画幅 | 例如 15s / 16:9 | 必问 |
| 制作限制 | 周期、预算、作画量、禁用项 | 必问 |
| 项目品类 | 写实影片/TV 动画/剧场版/短视频 | 建议问 |
| Seedance 入口类型 | 首尾帧 / 全能参考 | Seedance 任务必问 |
| 品牌/产品与卖点 | 品牌名、USP、利益点 | 广告任务必问 |
| 广告资产与合规 | Logo、口播、字幕、CTA、禁用词 | 广告任务必问 |

### 1.2 缺失信息提问模板

```text
为保证分镜可执行，请补充：
1) 任务类型：新建分镜 或 优化现有分镜
2) 目标平台与时长/画幅
3) 制作限制（周期、预算、作画量）
4) 本场戏核心叙事目标
5) （广告任务）品牌卖点、必带文案与 CTA
```

## 2. 素材标准化（Asset Normalization）

### 2.1 文本标准化
- 抽取：人物、动作、地点、时间、冲突、情绪、关键道具
- 标记：内心戏、象征意象、必须保留台词

### 2.2 图片/视频编号映射
- 统一映射：`Image 1...N`、`Video 1...N`、`Audio 1...N`
- 在分镜主稿中保持同一主体编号一致

### 2.3 Seedance `@素材` 映射（默认必做）
- 输出前必须生成素材用途声明：
1. `@图片N`：首帧 / 角色参考 / 场景参考 / 风格参考
2. `@视频N`：动作参考 / 运镜参考 / 节奏参考 / 待延长或待编辑对象
3. `@音频N`：BGM 参考 / 音色参考 / 节奏卡点参考
- 若素材用途冲突，按优先级处理：角色一致性 > 动作 > 运镜 > 音频

### 2.4 Seedance 输入约束检查（默认必做）
- 图片 `<= 9`
- 视频 `<= 3`
- 音频 `<= 3`
- 混合输入总数 `<= 12`
- 生成时长 `4-15s`（延长任务按“新增部分时长”填写）
- 若超限，先产出“降载方案”（删减低优先素材 + 分段生成）

### 2.5 九宫格/网格图处理

```text
G0 网格检测 -> G1 意图判断 -> G2 虚拟拆分 or 回退模式A
```

- `G1` 关键词优先：
1. 分镜词（分镜/脚本/storyboard/按顺序） -> 走 B
2. 参考词（参考图/表情/姿态/角色设定） -> 回退 A

- `G2` 虚拟拆分规则：
1. 行优先、从左到右、从上到下
2. `[row,col] -> Image N`
3. 若叙事顺序明显冲突，可重排并在备注说明原因

## 3. 动态检索与规则同步（必须执行于“最新/平台限定”场景）

### 3.1 触发条件
- 用户要求“最新/趋势/最佳实践”
- 提到特定平台（尤其 Seedance 2.0）
- 平台限制可能变化（输入上限、时长、素材格式）

### 3.2 检索优先级
1. 官方文档/官方帮助中心
2. 官方 GitHub/SDK 文档
3. 可信行业一手资料

### 3.3 检索结果落地
- 只保留可执行规则，不抄长文
- 合并到本次“平台适配参数表”
- 在最终输出附来源链接

参考：
- `../system-prompts/platform-prompts.md`
- `../system-prompts/seedance-2.0-guide.md`

## 4. Step B1：场次规划（Scene Planning）

### 4.1 通读与定调
- 识别题材：动作/悬疑/爱情/科幻/动画
- 定义情绪基调：压抑、温暖、紧张、史诗等
- 抽取关键意象：例如霓虹、雨、破碎镜面、棋盘

### 4.2 场次划分
按地点、时间、事件目标切分 `Scene`，每场定义：
- Dramatic Objective（剧情目的）
- Emotional Beat（情绪变化）
- Visual Motif（视觉母题）

### 4.3 结构选择
- 3-5 镜头：三幕式
- 5-9 镜头：蒙太奇或短叙事
- 7-12 镜头：完整故事式

### 4.4 产出工件 A（场次表）

| 场次 | 地点-时间 | 剧情目标 | 情绪曲线 | 关键意象 | 建议时长 |
| --- | --- | --- | --- | --- | --- |

### 4.5 B-Ad 广告策略工件（广告任务必做）

| 项目 | 内容 |
| --- | --- |
| 品牌主张 | 一句话价值主张 |
| 目标受众 | 人群与使用场景 |
| 核心卖点 | 1-3 条，不做冲突表达 |
| 证据镜头 | 卖点对应的可视化证据镜头 |
| CTA | 行动号召文案与出现场次 |
| 品牌资产位 | Logo/口播/字幕/包装露出位 |

---

## 5. Step B2：镜头拆解（Shot Breakdown）

### 5.1 拆解原则
- 不把一整段文字塞进一个镜头
- 动作戏：短镜头、多切换、强调动势
- 感情戏：长镜头、强调停顿和表演
- 内心戏转译：VO / 微表情特写 / 象征镜头
- 广告片：卖点先于炫技，每个关键卖点至少有一个“证据镜头”

### 5.2 单镜字段（必须完整）
- 镜号
- 景别（XLS/LS/MS/CU/ECU）
- 运镜（Dolly/Truck/Pan/Arc/POV 等）
- 画面描述（动作、光影、构图、空间关系）
- 音效/台词（SFX/BGM/对白/OS）
- 导演备注（叙事目的、隐喻、表演重点）
- 时长（秒）
- 转场（Cut/Dissolve/Fade/Match Cut 等）

### 5.3 产出工件 B（镜头计划表）

| 场次 | 镜号 | 景别 | 运镜 | 画面描述 | 音效/台词 | 导演备注 | 时长 | 转场 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |

参考：
- `../system-prompts/storyboard-guide.md`
- `../system-prompts/knowledge-base.md`
- `../system-prompts/transitions.md`

## 6. Step B3：视听填充与平台化 Prompt

### 6.1 运镜物理逻辑（硬规则）
- `Pan/Tilt` 是机位不动的转动
- `Dolly/Truck/Crane` 是机位移动
- 禁止同一镜头写互相冲突的运动命令

### 6.2 Prompt 生成顺序
1. 先导演分镜（中文可执行稿）
2. 再生成平台投喂 prompt（中/英）
3. 必要时按平台导出不同版本
4. 广告任务追加：输出“镜头-文案-CTA”对齐表

### 6.3 平台化导出建议
- `Sora`：场景先概括，再给镜头/动作/光影；复杂动作按时间段拆分
- `Runway`：主体动作 + 场景动作 + 镜头动作，描述简洁明确
- `Seedance 2.0`：先写 `@素材` 用途映射，再写镜头主线；涉及延长/编辑时必须写清目标片段与新增秒数

### 6.4 每镜头 Prompt 包
- `[运镜技法名称]`
- `CN 解析`
- `EN Prompt`（运镜关键词前置）

### 6.5 Seedance 2.0 执行包（默认输出）
```text
[入口]
- 首尾帧 / 全能参考

[素材映射]
- @图片1: ...
- @视频1: ...
- @音频1: ...

[任务类型]
- 新建 / 延长 / 编辑

[时长与画幅]
- 生成时长: ...s
- 画幅: 16:9 / 9:16 / 1:1

[提示词]
- 主提示词（中文）
- 可选英文增强词
```

## 7. Step B4：验证与反馈回路（Feedback Loop）

### 7.1 基础验证（全部必须通过）
- Image 编号一致
- 台词语言保留
- 无越界脑补（不擅自改剧情核心）
- 结构完整
- 长度合理

### 7.2 分镜专属验证
- 轴线/空间连续性
- 人物走位连续性
- 转场合理性
- 节奏起伏是否支持叙事目标

### 7.3 动画制作可行性验证（B-Review 强制）
- 原画工作量是否超预算
- 中期（合成/特效）是否超周期
- 是否存在可替代低成本镜头

### 7.4 广告有效性验证（B-Ad 强制）
- 品牌露出是否清晰且时长足够
- USP 是否有对应证据镜头
- CTA 是否明确且位于收束段
- 口播、字幕、画面是否同频不冲突
- 合规表达是否通过（无越界宣称）

### 7.5 回路机制

```text
若任一校验失败：
  定位失败项 -> 回退到 B1/B2/B3 对应步骤修订 -> 重新验证
直到全部通过后再输出终稿
```

参考：`../system-prompts/validation-rules.md`

## 8. 最终交付格式

## 8.1 B-Create（新建分镜）

### 1) 总体分析
- 核心基调
- 关键意象

### 2) 分镜表（按场景输出）

**场景标题：[场景号] [地点] - [时间] (氛围描述)**

| 镜号 | 景别 | 运镜 | 画面描述 (Visual) | 音效/台词 (Audio) | 导演备注 (Note) |
| :--- | :--- | :--- | :--- | :--- | :--- |

要求：每个场景通常 `5-10` 镜头（按情节密度浮动）。

### 3) 平台投喂 Prompt（可选）
- 按镜头附 `CN 解析 + EN Prompt`

## 8.2 B-Review（优化现有分镜）

### 一、导演全局评估
1. 原分镜核心亮点
2. 核心问题（<=3）
3. 整体优化方向与创作基调

### 二、逐镜分镜修改优化建议
- 【原分镜内容】
- 【现存问题】
- 【修改方案】
- 【增益效果】
- 【制作提示】

### 三、镜头组接与节奏优化
1. 镜头顺序/剪辑频率/时长优化
2. 转场优化与叙事作用
3. 全场节奏起伏设计

### 四、视听与表演整体升级建议
1. 人物表演统一策略
2. 光影/色彩/特效方案
3. 配音/音效/配乐同步建议

### 五、制作落地全流程提示
1. 核心难点与规避
2. 作画量与周期优化
3. 原画/作监/后期执行注意项

## 8.3 B-Ad（广告分镜设计）

### 一、广告策略摘要
1. 品牌主张
2. 目标受众
3. 核心卖点（USP）
4. CTA 与转化目标

### 二、广告分镜表
| 镜号 | 时长 | 景别/运镜 | 画面（卖点证据） | 音频/口播 | 字幕/屏显 | 品牌露出 | CTA |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |

### 三、执行包
1. Seedance `@素材` 映射
2. 主提示词（中文）
3. 英文增强词（可选）
4. 合规与禁用词检查结果

---

## 9. 失败与回退策略

- 用户输入不足 -> 停止生成，先补信息
- 九宫格意图不明 -> A/B/B-Review/B-Ad 四选一确认
- 平台约束冲突 -> 先给“通用版”分镜，再给“平台受限版”
- 制作不可落地 -> 保留叙事目标，提供低成本替代镜头
- Seedance 输入超限 -> 先输出“降载计划”，再生成可执行版本

## 10. 执行检查清单（SOP Checklist）

```text
□ 已完成输入契约检查
□ 已完成任务分支判断（B-Create / B-Review / B-Ad）
□ 已完成素材编号与九宫格处理
□ 已同步最新平台规则（如触发）
□ 已产出场次表与镜头计划表
□ 已完成视听填充与平台化 prompt
□ 广告任务已完成 USP/CTA/合规检查
□ 已通过验证规则与可行性检查
□ 已按目标格式交付
```
