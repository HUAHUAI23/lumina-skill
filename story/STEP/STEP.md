## 前置声明
本文件是 `story` skill 的完整执行规范。  
必须先完成模式路由，再按 STEP 顺序执行；禁止跳步、禁止先出成稿后补分析。  
本技能覆盖：广告文案、广告脚本、图生故事、短篇与长篇小说、视频提示词。

---

## STEP 0｜模式路由与输入契约

### 模式路由
- `MODE_AD_COPY`
- `MODE_AD_SCRIPT`
- `MODE_IMAGE_TO_STORY`
- `MODE_STORY_SHORT`
- `MODE_NOVEL_LONG`
- `MODE_VIDEO_PROMPT`
- `MODE_POLISH`

### 快速触发（首轮默认包）
- 若用户只说“开始 / 需要 / 先来一版 / 给我首轮”，直接触发 `MODE_FIRST_ROUND_BUNDLE`。
- `MODE_FIRST_ROUND_BUNDLE` 不做追问，基于“通用消费品 + 城市场景 + 25-35岁受众”假设先出首轮草案。
- 交付后附“可替换变量清单”（产品名、受众、平台、语气、预算），供下一轮精修。

### 输入契约（缺失先补问，或假设并标注）
1. 任务目标：转化/传播/叙事/品牌心智
2. 受众画像：年龄、性别、场景、痛点、消费层级
3. 主体信息：产品/人物/事件/图片内容
4. 风格方向：写实/温情/悬疑/幽默/热血/科幻等
5. 平台与规格：抖音/小红书/B站/公众号；字数或时长
6. 输出形式：文案、脚本、故事、提示词、组合包

### 新版提示词工程规则（强制）
1. 指令前置：角色、任务、硬约束、输出格式放最前
2. 分解执行：`大纲 -> 场景卡 -> 草稿 -> 格式化 -> 质检`
3. 角色分工：`Planner -> Writer -> Editor -> QA`
4. 迭代策略：仅重写低分片段，不全量推翻
5. 缓存友好：静态前缀固定，变量后置，便于复用

## STEP 1｜素材解构与目标映射

**目标**：把用户输入转为可执行任务结构。

**执行方法：**
1. 提取目标对象：产品、人、事件、地点、时间。
2. 标记任务核心：销售转化 / 品牌传播 / 情绪体验 / 连载推进。
3. 输出“任务卡”：

```text
任务卡
- 模式：
- 主目标（1句话）：
- 受众：
- 成功标准（如点击率/停留时长/阅读完成率）：
- 必须包含：
- 必须避免：
```

4. 若用户给图片：先提取可见事实，不做超出画面的硬推断。

**质量检查点：**
> 是否明确“目标 + 受众 + 成功标准”三要素？

## STEP 1.5｜首轮默认四套内容包（仅快速触发时执行）

**目标**：在信息极少时也能先给用户“可用草案”，减少空转。

**输出清单（固定四套）：**
1. 商家广告文案包（3风格 + A/B Hook + A/B CTA）
2. 广告脚本包（15s，含分场、音效、镜头、A/B开场）
3. 图转故事包（微故事/短故事/剧情版三档）
4. 电影感视频提示词包（2条，CN/EN 双语）

**默认假设（需显式标注）：**
- 产品类型：消费品（可替换）
- 受众：25-35 岁城市用户
- 平台：抖音/小红书优先
- 风格：现代写实、轻情绪张力

**质量检查点：**
> 四套内容是否都可直接复制使用，并且都带可替换变量？

## STEP 2｜人物引擎（量化人格模型）

**目标**：把人物写“活”，避免空洞标签。

### 2.1 变量定义
- `C`（共性）：外显标签（如“冷静律师”）
- `P`（个性）：内在驱动（如“对弱者共情强”）
- `M`（矛盾）：`M = |Value(C,dim) - Value(P,dim)|`
- `H`（和谐）：解释矛盾共存的经历或信念
- `G`（人格倾向）：可映射 MBTI 作为行为参考
- `L`（生命力）：在事件触发下输出行为

### 2.2 执行方法
1. 抽取角色的 `C` 与 `P`（至少各 2 条）。
2. 在 2-3 个维度打分（如情感、纪律、道德）。
3. 找出高矛盾点（建议阈值 `M > 8`）。
4. 为每个高矛盾点配置 `H`（合理化触发条件）。
5. 输出“角色行为预测卡”：

```text
角色行为预测卡
- 角色名：
- C：
- P：
- 高矛盾M：
- H触发器：
- 在事件E下的行为B（选择/台词风格/标志动作）：
```

**质量检查点：**
> 该角色在冲突场景下是否能预测“会怎么做、怎么说”？

## STEP 3｜叙事蓝图设计（DNA -> Blueprint）

**目标**：在写正文前锁定叙事结构。

**执行方法：**
1. 生成项目 DNA（主题、主冲突、情绪承诺、受众快感点）。
2. 选择结构蓝图：
   - 广告：`Hook -> Problem -> Solution -> Proof -> CTA`
   - 短故事：`起 -> 变 -> 冲突 -> 反转/释然`
   - 长篇：`世界观 -> 主线 -> 支线 -> 阶段钩子`
3. 标注节奏策略：快叙事、慢情绪（关键场景留情绪停顿）。
4. 设置信息差策略：
   - 观众先知（爽感）
   - 主角先知（悬念）
   - 反派先知（危机）

**质量检查点：**
> 是否能用 3 句话说清楚“这个内容为什么会让目标受众继续看”？

## STEP 3.5｜短视频叙事节奏协议（SCU / N1 / N2）

**目标**：把“好看”转成可执行节奏模型，适用于广告脚本和剧情型短内容。

### SCU（爽点循环单元）
1. 压力：建立压迫或痛点
2. 引燃：一个动作/一句话预告反击
3. 爆发：核心反转或关键解决
4. 发酵：展示后果、收益、他人反应

### N1 高频模型（转化优先）
- 0-8 秒：黄金钩子
- 8-40 秒：SCU 循环 I
- 40-75 秒：SCU 循环 II
- 75-90 秒：留扣或 CTA 前悬停

### N2 过山车型（传播优先）
- 0-15 秒：快速抬升
- 15-45 秒：危机深潜
- 45-80 秒：反弹爆发
- 80-90 秒：反转留悬

**执行规则：**
- `MODE_AD_SCRIPT` 默认优先 N1（高转化）。
- 叙事类内容可选 N2（高讨论度/高传播）。

**质量检查点：**
> 是否至少完成 1 个完整 SCU，且结尾仍保留驱动下一步的动因（CTA 或悬念）？

## STEP 4｜模式化创作流程

### A. `MODE_AD_COPY`（商家广告文案）

**目标**：先讲人，再讲货；先痛点，再卖点；最后强 CTA。

**执行方法：**
1. 卖点映射：功能卖点 -> 用户收益 -> 使用场景。
2. 文案分层：
   - 标题（吸睛）
   - 首屏钩子（3 秒）
   - 主文案（痛点/解决/证据）
   - 收口 CTA（明确动作）
3. 输出最少 3 个版本：理性款、情绪款、对比款。
4. 每版提供 A/B Hook 和 A/B CTA。

---

### B. `MODE_AD_SCRIPT`（广告脚本）

**目标**：可拍、可投放、可转化。

**执行方法：**
1. 选时长：6s / 15s / 30s / 60s。
2. 每场必须包含：画面动作、台词、音效、镜头目标。
3. 0-3 秒必须强 Hook（反常识/强冲突/高利益）。
4. 结尾必须有强 CTA（点击、咨询、领券、下单）。
5. 提供 A/B 开场和 A/B 收口版本。

---

### C. `MODE_IMAGE_TO_STORY`（根据图片写故事）

**目标**：以图为锚，扩展情节，不脱离视觉事实。

**执行方法：**
1. 先列“可见事实清单”（人物、动作、环境、道具、光影）。
2. 再列“可合理推断”（情绪、关系、事件前后因果）。
3. 输出三档版本：
   - 微故事（80-120字）
   - 短故事（300-600字）
   - 剧情版（800-1500字）
4. 可选广告化：增加品牌植入与消费触发点。

---

### D. `MODE_STORY_SHORT`（短篇）

**目标**：短平快但有记忆点。

**执行方法：**
1. 三段式：开场钩子 -> 冲突升级 -> 反转或情绪落点。
2. 每 150-250 字至少一个情绪推动点。
3. 对话优先可视化，避免纯解释性段落。

---

### E. `MODE_NOVEL_LONG`（长篇小说）

**目标**：可持续连载，人物稳定，冲突可升级。

**执行方法：**
1. 输出世界观、主线、阶段目标、章节目录（先骨架）。
2. 章节执行采用“单章目标卡”：
   - 本章目标
   - 冲突
   - 信息释放
   - 章尾钩子
3. 每章结束做一致性核查（设定、人物、时间线）。

---

### F. `MODE_VIDEO_PROMPT`（电影感视频提示词）

**目标**：把简短画面意图扩写为高质量视频生成提示词（CN/EN）。

**导演级写法约束：**
1. 由浅入深：先整体氛围，再局部特写。
2. 微表情优先：禁止只写“开心/害怕/生气”等空词。
3. 加入生理反应：瞳孔、喉结、呼吸、汗水、肌理。
4. 明确运镜：推/拉/摇/移/景深/Rack Focus/Extreme Close-up。
5. 英文提示词含质量标签：如 `8k`, `photorealistic`, `cinematic lighting`。

**情绪转译（示例）**
- 害羞：咬下唇、眼神躲闪、睫毛颤动；手指绕发丝
- 愤怒隐忍：咬肌绷紧、鼻翼扩张；拳头发白、指甲掐掌心
- 恐惧紧张：瞳孔震颤、喉结滚动；频繁吞咽、手指抠紧
- 震惊呆滞：嘴微张、动作定格；手中物件滑落
- 深思算计：眼神聚焦不动；摩挲戒指或打火机

**输出格式（必须严格）：**
```text
[氛围关键词]
CN: [中文详细描述，包含镜头、光影、动作细节]
EN: [英文提示词，包含艺术风格、镜头术语、细节描述]
```

**平台级优化要点（官方经验）**
1. Sora：提示词尽量具体；描述画面发展过程（可含起承转）；图生视频时优先写“想要的运动变化”。
2. Runway：提示词直接、简洁；图生视频重点写运动，不重复描述图片已给出的静态信息；避免用否定句堆叠。
3. Midjourney：主体 + 场景 + 光影 + 镜头 + 风格 + 参数拆段写，先保证主体清晰再加风格词。

---

### G. `MODE_POLISH`（改写优化）

**目标**：保留原意，提升可读性、转化力和完成度。

**执行方法：**
1. 先指出原文问题：冗余、空洞、逻辑跳跃、无钩子。
2. 再给优化稿：最少 2 个风格版本。
3. 标注改写策略：删减、重排、强化、替换。

## STEP 5｜草稿生成（分解式）

**目标**：先出可控草稿，再做精修。

**执行方法：**
1. 先输出“结构草稿”（不追求文采，追求完整）。
2. 再输出“语言精修稿”（节奏、措辞、记忆点）。
3. 对关键段落做“可选增强包”：
   - 情绪增强
   - 画面增强
   - 转化增强（广告）

**质量检查点：**
> 草稿是否已满足“结构完整 + 关键元素齐全”？

## STEP 6｜质量闸门与评分回路

### 通用评分 Rubric（每项 1-5 分）
1. 目标对齐（是否真正解决用户目标）
2. 结构清晰（起承转合/卖点路径/逻辑链）
3. 情绪牵引（有无持续阅读动力）
4. 语言质感（具体、可视、可记忆）
5. 可执行性（可拍、可投放、可复用）

**规则：**
- 总分 < 20：必须局部重写
- 任一项 <= 2：该项强制重写

### 模式专项检查
- 广告文案：USP 是否清楚、CTA 是否明确、是否有 A/B 变体
- 广告脚本：首 3 秒是否抓人、场景是否可拍、节奏是否递进
- 图片故事：是否遵守可见事实边界、是否强行脑补
- 长篇小说：设定是否自洽、章节钩子是否连续
- 视频提示词：是否包含微表情 + 运镜 + 光影 + 质量词

### DII-Lite（沉浸诊断，可选）
- `S`（安全感）：主角/品牌承诺是否清晰可信
- `E`（效率感）：信息密度是否高、拖沓是否少
- `R`（强化感）：情绪峰值是否被放大与记住

规则：任一维度 <= 2 时，优先重写该维度相关段落。

## 常见错误与修正

| 错误类型 | 表现 | 修正方法 |
|---|---|---|
| 目标漂移 | 写得好看但不转化 | 回到任务卡，重写 Hook+CTA |
| 情绪空词 | 只写“开心/生气” | 强制替换为微表情+动作 |
| 图片过度脑补 | 脱离可见事实 | 先列事实清单，再推断 |
| 广告无证据 | 只喊卖点无证明 | 增加场景证据/用户反馈/参数 |
| 长篇失控 | 章节松散、设定冲突 | 章节目标卡+一致性审查 |
| 提示词无镜头 | 只有形容词堆叠 | 补齐运镜、景别、光影、节奏 |

## 快速参考：镜头与情绪映射

| 情绪目标 | 推荐景别 | 推荐运镜 | 声音建议 |
|---|---|---|---|
| 暧昧心动 | 中近景/特写 | 缓慢推镜 | 轻呼吸+环境弱噪 |
| 隐忍愤怒 | 近景/大特写 | 固定后突推 | 低频轰鸣+短停顿 |
| 恐惧紧张 | 大特写 | 轻微手持抖动 | 心跳+耳鸣 |
| 震惊失语 | 特写 -> 定格 | 快推后静止 | 突然静音 |
| 绝望崩溃 | 中景 -> 拉远 | 缓慢拉镜 | 空旷混响+低音乐 |
