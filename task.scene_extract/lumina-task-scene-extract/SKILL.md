# task.scene_extract

## 适用场景

- 从文本、剧本、故事段落中抽取稳定场景资产。
- 为后续场景空镜参考图、图生图空间锁定、分镜连续性和视频镜头复用建立场景库。

## 负责什么

- 优先把当前文本中的空间提及映射到已存在场景资产，合并同一地点的不同说法。
- 找出值得长期维护的地点和空间：反复出现、承载关键剧情、影响视觉连续性、需要后续生成参考图的场景。
- 为每个场景保留可生图空间锚点：空间类型、布局结构、尺度关系、入口/路径、高低层级、关键陈设/自然物、材质、光线、色调、天气季节和默认缺失项。
- 记录同一空间的关键状态变化，例如白天/夜晚、晴天/雨雪、荒废/复苏、破损/修复、污染/净化、贫穷/富丽、现代/古旧。
- 为下游无人空镜提示词留下足够依据，让场景能作为空间锁定参考，而不是只留下抽象气氛。

## 不负责什么

- 不抽取人物和独立道具。
- 不把一次性镜头背景、纯时间、纯天气、纯情绪当成稳定场景。
- 不直接写最终图片提示词或图片任务结构。
- 不把有行动意志、可反复出现的妖怪、怪兽、动物伙伴、灵兽抽成场景资产；它们应进入人物资产。
- 不为了丰富画面而创造文本没有支持的新地点、地图关系或建筑结构。

## 判断原则

- 身份名优先使用文中稳定地名或空间名，例如“乱石岗”“病房”“村口雪地”“玄清宗剑阵”；不要用“山上、那里、屋里”这类泛称，除非上下文只有这个稳定空间。
- 同一地点不要因为轻微角度、天气、时间或剧情动作变化拆成多个场景；这些通常是同一空间的状态或镜头倾向。
- 场景描述聚焦能被画出来的事实：空间边界、前后左右关系、高低层级、入口出口、主材质、主光源、主色调、典型陈设和缺失物。
- 氛围不能替代空间。不要只写“压抑、温馨、神秘、恐怖”；要转成光线、色调、材质、拥挤/空旷、破败/整洁、冷暖、湿度、尘土、声源等可见信息。
- 默认按无人空镜资产理解场景；除非场景本体依赖人群活动，否则不要把人物动作写进场景核心描述。
- 背景动物群、环境生物或远景生态只在服务空间氛围时记录；有独立身份和行动连续性的生物不要放入场景。
- 医院、街道、工地、村庄等泛地点只在承载重要剧情或后续可能复用时新增；一笔带过的地点保留待解析或省略。

## 空间锚点

- 基础空间身份：室内/室外、现代/古代/仙侠/科幻、住宅/宗门/街道/战场/病房/工地/自然地貌等。
- 布局结构：房间方位、门窗、走廊、床边/床底、桌面/台阶、庭院、山道、街巷、建筑群、阵法中心等。
- 尺度关系：狭窄、开阔、纵深、压迫、低矮、挑高、层级、坡度、人物在远景中显得渺小等。
- 关键元素：建筑、陈设、自然物、地貌、入口、路径、遮挡物、反复出现的空间标记。
- 材质和表面：木、石、金属、玻璃、水泥、泥土、雪地、水面、破墙、潮湿地面、烧焦痕、霉斑、尘土。
- 光线色调：主光源、光线方向、冷暖、明暗、雷光、金光、居家常光、晨光、夜灯、体积光、阴影密度。
- 缺失项：没有人群、没有现代装潢、没有绿叶果实、没有整洁街道、没有温暖灯光等，按剧情事实选择。

## 状态阶段与反常规环境

- 场景资产要记录稳定结构，也要记录会影响后续分镜的一组关键状态：季节、天气、毁坏/修复、荒废/繁盛、污染/净化、冻死/复苏、白天/夜晚、贫穷/富丽。
- 同一空间经历重大状态变化时，通常仍保持一个场景资产，用状态说明记录差异，不默认拆成多个场景。
- 当场景中的核心元素处于反常规状态时，必须显式记录默认特征缺失。例如果园冻死应写光秃开裂树干、没有叶片、没有果实；废弃房屋应写破窗、剥落墙皮、无人维护、没有整洁新装修。
- 如果场景情绪来自规模和压迫感，应记录可生成尺度线索，例如大片乱石坡、梯田层级、巨型建筑阴影、狭窄走廊、低顶空间、远景人物渺小。
- 植物、农作物、建筑群、雪地血迹、泥水木牌等如果主要服务空间气氛和连续性，可作为场景元素；如果需要独立特写、操作或跨场景复用，再交给道具抽取。

## 变体线索沉淀

- 抽取阶段必须把后续图生图空间变体需要的线索沉淀到 summary 或 attributeEntries，不直接写图片提示词。
- 推荐属性 key：`space_anchors`、`layout_structure`、`canonical_state`、`time_variants`、`weather_variants`、`seasonal_variants`、`condition_variants`、`angle_needs`、`detail_anchors`、`negative_defaults`。
- `time_variants` 记录清晨、早餐、傍晚、夜晚、深夜等会改变光源和使用痕迹的时间状态。
- `weather_variants` 记录晴天、雨天、雪天、雾天、雷雨、严寒等会改变表面状态和光线的天气。
- `seasonal_variants` 记录春夏秋冬、枯黄、繁盛、落叶、冻土、雪灾等季节状态。
- `condition_variants` 记录荒废、破损、修复、复苏、污染、净化、贫穷、富丽、古旧、现代化等阶段变化。
- `angle_needs` 和 `detail_anchors` 只记录后续确实需要补足空间锁定的入口、路径、床边、墙面、牌匾、阵法纹理、材质磨损等。
- 每个变体线索都要能回答：保持哪些空间锚点不变，只改变什么时间/天气/季节/状态，以及什么时候复用。

## 工作方法

- 先判断文本里的空间是否已经能复用到现有场景。
- 决定新增前先问：这个地方是否会复用？是否承载关键剧情？是否后续需要空镜图锁定空间？是否有足够可见锚点？
- 抽取时按“空间身份 + 布局结构 + 尺度关系 + 关键元素 + 材质表面 + 光影色调 + 天气季节 + 状态变化 + 缺失项”检查。
- 如果原文出现可复用角度或后续画面需求，只记录为构图倾向，例如全景建立空间、入口路径、床底近景、墙面特写；不要在抽取阶段写成完整图片提示词。
- 对会被模型默认美化的地点，要写清“没有什么”：没有人群、没有现代装潢、没有绿叶果实、没有整洁街道、没有温暖灯光等，具体以剧情事实为准。
- 最终场景说明要短而准：说明是什么空间、核心布局/尺度、关键元素、光影色调、当前状态，以及哪些默认元素不能出现。

## 质量标准

- 场景列表稳定、不重复，地点称呼清楚。
- 每个场景资产能支撑无人空镜参考图，而不只是抽象气氛词。
- 场景资产能回答：这个地方从哪个角度一眼能认出来？哪些元素必须出现？哪些默认美化必须排除？哪些状态变化会影响后续镜头？
- 场景的关键状态不会被宽泛名称冲掉；如果故事需要荒芜、枯死、雪灾、破败或复苏，应有可见证据和排他性缺失。
- 不把人物动作、道具特写或剧情结论误写成场景主体。

## 失败回退

- 边界模糊时，宁可少收也不误收一次性背景。
- 信息不足时，优先保住空间类型、核心布局和主氛围，不臆造地图关系、建筑结构或具体陈设。
- 如果空间和道具、植物、人物状态混在一起，优先只保留空间本身，把对象状态交给对应资产链路。
