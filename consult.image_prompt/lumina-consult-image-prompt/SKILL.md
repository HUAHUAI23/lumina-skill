# consult.image_prompt

## 适用场景

- 用户想写、改、优化图片提示词，但还不准备直接发任务。

## 负责什么

- 把图片创意整理成可直接使用的提示词建议。
- 补足主体、行为/状态、环境、构图、风格、光线、色调、材质、比例和限制条件。
- 帮用户压实视觉目标，减少描述冲突。
- 根据用途调整表达，例如角色参考图、场景空镜、道具单体图、海报、缩略图、产品图或风格迁移图。
- 按用户目标选择文生图、编辑图、多图融合、风格迁移、文字海报、人物一致性等写法。

## 不负责什么

- 不创建图片任务。
- 不读取项目维护资产。
- 不做人物、场景、道具绑定。
- 不输出 SD 标签串、权重括号、命令行参数或 provider 私有语法。

## 判断原则

- 现代图像模型更吃完整自然语言和逻辑清楚的约束；优先写可理解的画面描述，不堆关键词。
- 用户信息不完整时，优先补足主体、用途、风格/氛围、构图/景别、光影/色调、材质和比例。
- 提示词要能直接投喂，不停留在创意散文，也不写成松散标签。
- 约束要真实有效，避免互相打架。
- 专业摄影词、镜头词或材质词可保留常用英文，例如 Macro lens shot、rim light、soft lighting，但不要堆成英文标签流。
- 编辑已有图时优先写“改变什么、保留什么”，不要重写成一条全新文生图 prompt。
- 多图工作流必须明确 `图1/图2/图3` 各自用途，避免“上图”“这张图”这类模糊指代。
- 人物一致性要具体到脸型、五官比例、眼距、鼻形、下颌线、年龄阶段、发型和稳定标志物。
- 如果用户需要严格一致性，先把不变项和可变项分开，再写最终提示词；不变项优先级高于风格美化。

## 工作方法

- 先判断需求类型：文生图、编辑图、多图融合、风格迁移、文字海报、角色参考图、缩略图或产品图。
- 文生图按“主体 + 行为/状态 + 环境 + 构图/景别 + 光影/色调 + 材质细节 + 风格/比例”组织。
- 编辑图按“变化动作 + 变化对象 + 变化特征 + 保持不变项”组织。
- 多图融合先列图像职责：`图1` 锁主体身份，`图2` 锁场景/风格，`图3` 锁姿态/道具/构图，再写最终画面。
- 人物表情变化不要只写“开心/悲伤/震惊”，要写嘴角、眼眶、眉头、瞳孔、呼吸、泪痕、肌肉紧绷等可见变化，并声明不换脸、不美化、不改年龄。
- 场景图默认写清空间类型、布局、入口/路径、关键陈设、光线、季节天气、材质和主色调；若是资产参考图，应默认无人空镜。
- 道具图默认写清轮廓、材质、尺寸比例、纹理、磨损、发光或标记；除非用户明确要求，不加入人物手持或复杂背景。
- 文字海报要把生成文字放进引号，说明字体风格、排版位置、配色和用途；同时提醒复杂小字可能需要后期校对。
- 最终给出一版可直接复制的提示词；复杂需求可先给“素材/参考图职责”再给正文。

## 输出格式

- 简单文生图：直接输出一段自然语言提示词。
- 编辑现有图：输出一条短编辑指令，明确保留项和变化项。
- 多元素或多图：用分点式说明主体、场景、构图、光影、风格、材质、参考图职责。
- 严格人物一致性：把“不变身份”和“可变状态”分开写，必要时加入短负向约束。

## 质量标准

- 具体、可执行、可直接使用。
- 画面目标清晰，风格和主体不冲突。
- 主体、构图、光影、色调、材质、比例至少覆盖关键项。
- 参考图职责清楚，保留/变化分开，没有互相矛盾的指令。
- 关键文字已加引号，人物一致性有具体约束。

## 失败回退

- 用户信息过少时，先给最小可用版本，再指出缺失维度。
- 不用空洞形容词替代具体视觉信息。
- 用户说“更好看”时，转成可执行变化：提高对比度、调整光线、改变色调、强化主体、清理背景或提升材质。
- 多对象同时编辑导致风险高时，建议分步修改：先主体，再背景，再光影或文字。
