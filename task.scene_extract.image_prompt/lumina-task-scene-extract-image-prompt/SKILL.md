# task.scene_extract.image_prompt

## 适用场景

- 已完成场景抽取后，为单个场景整理一条可直接出图的任务提示。

## 负责什么

- 把场景名称、空间属性和环境氛围压实成一条可执行图片任务。
- 让空间结构、关键陈设、光线和气质保持统一。
- 需要时补充克制的反向约束，减少画面跑偏。
- 生成适合后续图生图锁定空间的场景参考图，而不是只追求一次性氛围图。

## 不负责什么

- 不新增地点。
- 不把一个场景拆成多条任务。
- 不改写上游已经确认的场景身份。

## 判断原则

- 空间感和环境特征必须稳定。
- 提示内容优先服务场景设定建立，而不是镜头运动。
- 画面语言直接、具体、可投喂。
- 默认是无人空镜；除非上游明确要求人物，否则不出现人、群像、背影或人体剪影。
- 重点写空间结构、入口/路径/高低关系、关键陈设、材质、光线、天气季节和主色调。
- 不把人物动作、剧情结果或旁白情绪当成场景本体；情绪只能转化为空间氛围和光影。
- 如果场景属性包含荒废、冻死、破败、污染、烧毁、灾后、贫穷、空无或复苏等反常规状态，prompt 必须写出当前状态和默认特征缺失，避免模型把场景恢复成繁茂、整洁、现代、温暖或丰收。
- 场景里的核心植物、建筑、地貌或陈设如果承载剧情反差，应使用排他性描述，例如“光秃开裂的枝干，完全没有叶片和果实”“剥落墙皮和破窗，没有新装修痕迹”。

## 工作方法

- 先判断这个场景最关键的空间气质和辨识物。
- 默认输出 1 张基准空镜图；只有解析事实显示同一场景需要额外角度、入口/路径、关键陈设或材质锚点时，才写入 variants。
- 如果写入 variants，第一张必须是 `canonical_establishing`：基准文生图，优先 wide shot / establishing shot / documentary still，完整展示空间布局。
- 可按事实需要补充 `alternate_angle`：后续图生图变体，保持同一空间身份，从不同角度提供可复用分镜参考。
- 可按事实需要补充 `detail_anchor`：后续图生图变体，保持同一空间身份，表现入口/路径、关键陈设、材质、光线或地形锚点，不要变成新场景。
- 再决定适合作为资产参考的构图：优先能看清布局和空间关系。
- 场景提示词应包含：空间类型、时代/地域质感、主要结构、关键陈设或自然物、光线方向、天气/季节、材质纹理、色彩范围。
- 如果场景的意义来自规模压迫或人物渺小感，基准图优先用 wide shot / establishing shot 展示完整空间关系，而不是只给局部氛围。
- negativePrompt 应排除 people、person、human、character、portrait、face、figure、crowd、group、silhouette、animal，避免场景参考图被人物污染。
- 对反常规场景，negativePrompt 还应排除最容易误补的默认元素，例如 lush greenery、ripe fruit、clean modern interior、pristine building、crowded street 等，具体按剧情状态选择。
- 最终整理成一条场景资产任务；没有必要多图时不要写 variants。只有输出多个 variants 时，系统才会编排成同一空间的“文生图基准图 → 图生图派生图”流程，适合绑定为 scene_lock 或 first_frame_candidate。

## 质量标准

- 一眼能看出场景性质。
- 环境目标明确，没有空泛修辞。
- 结果适合继续沉淀为场景参考图。
- 画面必须能给后续分镜提供稳定空间关系，而不是只有近景碎片或纯氛围色块。
- 反常规状态被锁住，不会因为场景名称触发模型默认美化，例如“果园”自动丰收、“病房”自动干净明亮、“老屋”自动温馨整洁。

## 失败回退

- 信息不够时，优先保住空间类型和核心氛围。
- 不额外捏造原文没有支持的环境细节。
- 如果文本只给了抽象情绪，转写为克制的光线、色彩、材质和空间空旷/拥挤程度，不新增具体建筑结构。
