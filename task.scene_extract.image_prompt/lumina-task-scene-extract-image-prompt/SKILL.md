# task.scene_extract.image_prompt

## 适用场景

- 已完成场景抽取后，为单个场景整理可直接出图的场景参考图任务。
- 需要让后续分镜、图生图和视频镜头稳定复用同一空间。

## 负责什么

- 把场景名称、空间属性、布局结构、关键元素、光影色调和当前状态压实成可生图提示词。
- 生成适合后续图生图锁定空间的场景参考图，而不是只追求一次性氛围图。
- 默认生成无人空镜；除非上游明确要求人物，否则不出现人物、群像、背影、剪影、脸或动物。
- 区分基准图和派生图：基准图建立空间身份，派生图只改变角度、距离、局部焦点或状态，不重造新地点。
- 用克制反向约束减少人物污染、错误美化、现代化补全和无关装饰。

## 不负责什么

- 不新增地点，不改写上游已经确认的场景身份。
- 不把一个场景拆成多条实体任务。
- 不生成剧情分镜图，不把人物动作、对白结果或旁白情绪当成场景主体。
- 不依赖准确小字、招牌长文本或复杂墙面文字作为空间唯一识别点。

## 判断原则

- 使用完整自然语言描述，不写关键词标签流、权重括号、命令行参数或 provider 私有语法。
- 场景资产图优先服务空间锁定：布局清楚、尺度关系清楚、关键元素清楚、材质和主光源稳定。
- 默认无人空镜；不出现人、群像、背影、人体剪影、肖像、动物和与场景身份无关的主体。
- 情绪只能转化为空间氛围和光影，例如冷白灯、暖暗卧室光、金光、雷光、阴影、空旷、压迫、尘土、潮湿、破败。
- 如果使用派生图，必须写清“保持同一空间身份、结构、材质、主色调和关键元素，只改变本次角度、距离或局部焦点”。
- 比例和构图服务用途：空间建立优先横向宽画幅或全景；入口/路径用中远景；材质、墙面、床底、阵法纹路等锚点才用近景或 Macro lens shot。
- 反常规场景必须写出当前状态和默认特征缺失，避免模型把果园变丰收、病房变明亮干净、老屋变温馨整洁、废墟变新建筑。

## 资产变体决策协议

- 每个场景图片任务先判断是否需要多图资产链路；输出中不要写推理过程，只把结果落实到 prompt 和 variants。
- 每个场景至少需要一张 `canonical_establishing` 基准空镜；只有存在可复用角度、时间、天气、季节、破损/修复、荒废/复苏、污染/净化或细节锚点时，才输出更多 variants。
- 同一空间不要因为白天/夜晚、早餐/晚间、晴雨雪、季节或轻微状态变化拆成多个场景；这些应作为同一场景的 variants。
- 如果场景属性或描述中已经出现时间、天气、季节、状态阶段或关键锚点，必须输出对应派生 variants。
- 每个派生 variant 必须写清“保持同一空间身份、布局结构、时代、材质、关键元素和空间比例；只改变本次时间/天气/季节/状态/角度/局部焦点”。
- variants 第一项是文生图基准空镜；后续 variants 默认会使用第一张基准图作为图生图空间参考。
- 每个 variant 必须填写英文小写 snake_case 的 `variantKey` 和清晰 `title`，不要使用 `variant_1`、`style_a`、`image_b` 这类无语义命名。

## 场景 variantKey 词表

- `canonical_establishing`：基准空镜全景，锁定空间结构、入口路径、关键元素、材质、主光源和主色调。
- `morning_establishing`：清晨状态，只改变自然光方向、亮度、空气湿度和早晨生活痕迹。
- `breakfast_state`：早餐时段状态，只在文本支持时加入餐具、桌面使用痕迹、晨光或室内灯光。
- `evening_establishing`：傍晚或黄昏状态，只改变天色、长阴影、窗外光和室内外色温。
- `night_establishing`：夜晚状态，只改变窗外天色、灯光、阴影密度和可见范围。
- `rainy_state`：雨天状态，只改变湿地反光、水迹、雨幕、阴天光和室外表面。
- `snowy_state`：雪天或严寒状态，只改变积雪、霜冻、冷光和被雪覆盖的表面。
- `seasonal_state`：季节状态，例如春夏秋冬、枯黄、繁盛、落叶、冻土。
- `damaged_state`：破损、灾后、荒废、污染、烧毁、倒塌或冻结枯死状态。
- `restored_state`：修复、复苏、繁盛、净化或重新启用状态。
- `alternate_angle`：同空间替代角度，补足入口、路径、床边、街角、庭院等复用视角。
- `detail_anchor`：门口、床底、墙面、牌匾、阵法纹理、破窗、材质磨损等锚点特写。

## 基准空镜图

- 基准图按“空间类型 + 用途/状态 + 布局结构 + 尺度关系 + 关键陈设/自然物 + 光线方向 + 天气/季节 + 材质纹理 + 色彩范围 + 风格/比例”组织。
- 优先使用 wide shot / establishing shot / documentary still 这类能展示空间关系的构图。
- 必须能看清空间如何成立：入口在哪里、主体区域在哪里、关键物如何分布、前中后景如何组织。
- 如果场景意义来自规模压迫或人物渺小感，基准图优先展示完整空间关系，而不是局部氛围。
- 如果是室内空间，写清门窗、床/桌/柜/墙面、走廊、遮挡、地面和主光源；如果是室外空间，写清地形、路径、建筑/自然物、天空/天气和远近层次。

## 派生角度与细节锚点

- 只有同一场景确实需要额外角度、入口/路径、关键陈设、材质、光线或地形锚点时，才补派生图。
- 派生角度用于补足可复用分镜视角，必须保持同一空间身份，不改变结构、时代、材质和关键元素。
- 细节锚点用于床底、门口、墙面、牌匾、阵法纹理、破窗、材质磨损、光源位置等可复用特写，不要变成新场景。
- 近景或 Macro lens shot 只服务材质/陈设/路径/文字承载面，不能把空间裁到完全无法识别。

## 文字面与默认美化

- 如果场景包含牌匾、招牌、文件墙、墙面标语或阵法文字，除非上游要求准确文字，否则只描述留白、模糊字迹、粗略标记或符号感纹理。
- 准确小字适合后期覆盖；场景参考图重点是承载面位置、材质、光线和空间关系。
- 反常规状态要写排他性描述，例如完全没有叶片和果实、没有新装修痕迹、没有人群、没有整洁街道、没有现代灯具、没有温暖软装。

## 负向约束

- 常规排除：people、person、human、character、portrait、face、figure、crowd、group、silhouette、animal、logo、watermark、caption。
- 反常规场景按剧情状态追加排除，例如 lush greenery、ripe fruit、clean modern interior、pristine building、crowded street、warm cozy decoration。
- 室内场景要排除无关人物、路人、宠物和杂乱广告；室外场景要排除与时代/地域不符的现代车辆、招牌或街景元素。

## 质量标准

- 一眼能看出场景性质和空间身份。
- 布局、尺度、关键元素、主材质和主光源清楚，能给后续分镜提供稳定空间关系。
- 结果适合继续沉淀为场景参考图，不是只有近景碎片或纯氛围色块。
- 反常规状态被锁住，不会因为场景名称触发模型默认美化。
- 无人物、无群像、无动物污染，空间身份不会被局部特写或装饰风格冲掉。

## 失败回退

- 信息不足时，优先保住空间类型、核心布局和主光影，不新增具体建筑结构。
- 文本只给抽象情绪时，转写为克制的光线、色彩、材质和空间空旷/拥挤程度。
- 如果无法判断是否需要派生图，默认只做一张基准空镜图，避免无意义多图。
