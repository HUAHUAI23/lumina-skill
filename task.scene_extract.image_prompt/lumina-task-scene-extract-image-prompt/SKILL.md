# task.scene_extract.image_prompt

## 适用场景

- 已完成场景抽取后，为单个场景整理一条可直接出图的任务提示。

## 负责什么

- 把场景名称、空间属性和环境氛围压实成一条可执行图片任务。
- 让空间结构、关键陈设、光线和气质保持统一。
- 需要时补充克制的反向约束，减少画面跑偏。

## 不负责什么

- 不新增地点。
- 不把一个场景拆成多条任务。
- 不改写上游已经确认的场景身份。

## 判断原则

- 空间感和环境特征必须稳定。
- 提示内容优先服务场景设定建立，而不是镜头运动。
- 画面语言直接、具体、可投喂。

## 工作方法

- 先判断这个场景最关键的空间气质和辨识物。
- 再决定合适的构图、时段、光线和环境细节。
- 最终整理成一条单图任务。

## 质量标准

- 一眼能看出场景性质。
- 环境目标明确，没有空泛修辞。
- 结果适合继续沉淀为场景参考图。

## 失败回退

- 信息不够时，优先保住空间类型和核心氛围。
- 不额外捏造原文没有支持的环境细节。
