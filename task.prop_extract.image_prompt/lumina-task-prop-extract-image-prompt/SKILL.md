# task.prop_extract.image_prompt

## 适用场景

- 已完成道具抽取后，为单个道具整理可直接出图的道具参考图任务。
- 需要让后续分镜、图生图和视频镜头稳定识别同一个物体。

## 负责什么

- 把道具名称、类别、轮廓、材质、颜色、尺寸比例和关键识别特征压实成可生图提示词。
- 生成适合后续复用的单体道具参考图；默认一张基准图即可，只有确实需要时才补充状态图或角度图。
- 让道具形状、材质、颜色、纹理、磨损、发光效果、标记和比例关系稳定。
- 区分基准图和派生图：基准图锁定物体身份，派生图只改变角度或表面状态，不生成新道具。
- 用克制的反向约束减少人物、手部、杂物和复杂背景干扰。

## 不负责什么

- 不新增道具，不改写上游已经确认的道具身份。
- 不把一个道具拆成多条实体任务。
- 不生成剧情分镜图，不让人物动作、手持姿势、场景叙事抢主体。
- 不要求模型生成准确长中文、小字文件或复杂可读文本。

## 判断原则

- 使用完整自然语言描述，不写关键词标签流、权重括号、命令行参数或 provider 私有语法。
- 主体必须稳定可辨认，画面只围绕一个核心道具。
- 默认干净单体参考图：单体居中、完整可见、边缘清楚、无遮挡、简单背景。
- 对灵物、法器、能量装置等可加入克制光效，但光效不能遮住本体轮廓、材质和标记。
- 旧损、泥污、血迹、烧焦、裂纹、褪色、泡烂、字迹模糊、长期握持痕迹必须作为主体特征，不要生成崭新商品图。
- 成套道具要突出统一形制、数量关系、排列关系和材质一致性，不要把套装变成随机杂物堆。
- 后续派生图必须写清：保持同一道具的轮廓、材质、颜色、尺寸比例和稳定标记，只改变本次角度、局部焦点或表面状态。

## 资产变体决策协议

- 每个道具图片任务先判断是否需要多图资产链路；输出中不要写推理过程，只把结果落实到 prompt 和 variants。
- 默认只输出一张基准单体图；只有证据明确且后续复用价值足够时，才输出更多 variants。
- 多图必须满足至少一个条件：需要可复用多角度来理解体积/背面/开合结构；存在会复用的破损、污损、血迹、激活、打开/关闭、修复或阶段状态；同一道具在多个镜头中需要保持不同稳定形态。
- 同一道具不要因为完整/破损、干净/污损、未激活/激活、打开/关闭、崭新/旧损而拆成多个道具；这些应作为同一道具的 variants。
- 如果破损、污损、血迹、激活、开合、修复或必要角度只是当前镜头的一次性表现，或可以由基准图 + story_to_video 镜头级 img2img 承担，不输出长期 variants。
- 如果某种状态就是道具的默认主状态，例如“旧损带血的护身符”首次出现后一直如此，则基准图直接采用该状态，不额外生成干净版。
- 每个派生 variant 必须写清“保持同一道具的轮廓、结构、材质、颜色、尺寸比例和稳定标记；只改变本次视角、表面状态、激活状态或开合状态”。
- variants 第一项是文生图基准单体图；后续 variants 默认会使用第一张基准图作为图生图物体参考。
- 增量抽取已有基准图时，新增状态 variants 默认使用现有基准图作为图生图参考；不要重新输出新的基准单体图，除非原基准图质量或身份锚点明显不可用。
- 每个 variant 必须填写 `assetDescription`，这是结构化 schema 的必填字段：一句话说明唯一道具身份是谁、这张资产是基准单体还是哪个状态/角度图、它供 story_to_video 在什么镜头决策中使用。
- `variantKey` 只是技术去重标识，不使用固定词表；可以用简短 snake_case，但决策信息必须写在 `assetDescription`。

## 基准单体图

- 基准图按“道具类别 + 轮廓结构 + 材质 + 颜色 + 尺寸比例 + 纹理/磨损 + 稳定标记 + 干净背景 + 风格/比例”组织。
- 构图优先完整展示物体，不裁切关键边缘，不遮挡主体，不加入手、人物、场景杂物或包装广告。
- 背景使用 plain / simple / neutral / clean background；如果项目风格需要真实质感，可用克制台面或地面，但不能变成复杂场景。
- 尺寸比例要写相对尺度，例如掌心大小、长剑尺度、文件纸张大小、吊坠尺寸、箱体体量；没有依据时不要硬给数值。

## 角度与状态图

- 只有当同一道具需要体积、背面结构、开合结构、破损、污损或激活状态时，才补派生图。
- 三分之四角度用于补足体积和视频运动参考，要保持同一物体身份，不改变轮廓和材质。
- 材质微距、手持、刻痕局部、血迹局部等交给 story_to_video 用干净道具图做镜头级 img2img，不作为资产库长期图片。
- 派生图要保留基准图的主体身份、颜色、材质和稳定标记，只改变视角或主体状态。
- 派生图数量保持克制：常规道具 1 张基准图；确有需要时 1 张基准 + 1-2 张派生图；超过 3 张必须能说明每张不可由镜头级派生或同一状态图替代。

## 文字面与标记

- 如果道具包含长文本、手写遗言、文件、符纸、牌匾、编号或复杂符号，不要求模型生成准确可读文字。
- 可以写“有模糊手写痕迹、墨迹、刻痕、符号感纹样、留白区域、磨损纸面”，让承载面稳定。
- 必须准确阅读的文字适合后期贴图或字幕层覆盖；道具参考图重点是纸张/木牌/金属刻面/墨迹位置和年代质感。
- 标志性符号、法纹、发光线条可以描述形态和位置，但不要写过长、过密、互相矛盾的纹样要求。

## 负向约束

- 常规排除：people、person、human、hand、hands、holding、clutter、busy background、multiple objects、cropped、blocked、logo、watermark。
- 旧损道具还应排除 pristine、brand new、glossy、plastic、clean surface 等容易误补的商品化质感。
- 成套道具要排除随机杂物、数量混乱、不同风格物件混入。

## 质量标准

- 一眼能看出是什么道具。
- 轮廓、边缘、主要材质和尺寸比例清楚，不能靠文字说明才能理解。
- 多角度或特写之间是同一个物体，不因光效、背景或局部展示变成另一件道具。
- 磨损、泥污、血迹、字迹模糊、年代感不会被默认清洁或商品化。
- 提示词可直接生成，没有空泛修辞和互相冲突的材质要求。

## 失败回退

- 信息不足时，优先保住名称、类别、轮廓和核心材质。
- 道具细节不足时，生成朴素单体参考，不强加复杂花纹、品牌、机关或奇幻效果。
- 如果文字面无法稳定生成，优先给留白、模糊笔迹和材质承载面，不强求准确文字。
