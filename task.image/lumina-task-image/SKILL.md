# task.image

## 适用场景

- 用户希望把当前轮需求整理成可直接下发的图片生成任务。
- 任务只依赖当前消息文本、当前消息上传素材和已注入的项目风格，不读取项目维护的人物、场景或道具资产。

## 负责什么

- 判断当前轮要生成的是文生图、编辑图、多图融合、风格迁移、海报/缩略图、产品图、角色参考图、场景空镜还是道具单体图。
- 把图片目标压实成完整自然语言提示词，覆盖主体、行为/状态、环境、构图/景别、光影/色调、材质、风格、比例和限制条件。
- 只基于当前轮上传素材判断哪些参考图参与，以及每张参考图承担什么职责。
- 在图生图、编辑图和多图融合中明确保留项和变化项，减少人物、空间、材质和风格串位。
- 为人物一致性、文字海报、场景空镜、道具单体和产品展示分别选择合适的提示词策略。

## 不负责什么

- 不读取项目维护的人物、场景、道具或历史资产。
- 不做资产绑定，不把当前轮图片任务沉淀为长期人物/场景/道具资产。
- 不保留咨询口吻和解释性废话。
- 不输出 SD 标签串、权重括号、命令行参数或平台私有语法。
- 不把图片任务写成视频分镜或故事转视频流程。

## 判断原则

- 现代图像模型更吃清晰、具体、有逻辑的完整自然语言；不要把提示词写成关键词堆。
- 信息不完整时，优先补足主体、用途、构图/景别、风格/氛围、光影/色调、材质细节和比例。
- 参考图只来自本轮上传素材；每张参考图都要说明职责，例如主体身份、服装、场景、构图、姿态、道具、材质或风格。
- 编辑已有图时优先写“改变什么、保持什么”，不要把任务重写成全新的文生图。
- 多图融合要按图像职责组织，不要用“上图”“这张图”这类模糊指代。
- 风格词只修饰画面，不覆盖主体身份；人物、产品、场景或道具的核心识别信息优先级高于风格美化。
- 专业摄影词和镜头词可保留常用英文，例如 Macro lens shot、soft lighting、rim light，但不要堆成英文标签流。

## 文生图写法

- 按“主体 + 行为/状态 + 环境 + 构图/景别 + 光影/色调 + 材质细节 + 风格/比例”组织。
- 主体要具体，避免“一个女人、一辆车、一个房间”这类泛称；尽量写年龄阶段、身份气质、服饰、物体类别或空间类型。
- 构图要服务用途：头像用近景或半身，海报用主视觉构图，产品图用干净商业展示，场景图用全景或建立镜头，道具图用单体居中。
- 光影和色调要能落地，例如清晨暖光、冷白顶光、侧逆光、柔光、低饱和、高对比、雨夜霓虹反射。
- 材质要具体到可见表面，例如金属冷反光、磨砂玻璃、粗糙水泥、柔软布料、旧纸折痕、木纹磨损。

## 编辑图与多图融合

- 编辑图按“变化动作 + 变化对象 + 变化特征 + 保持不变项”组织。
- 保持项要具体，例如保持人物脸型和年龄、保持原构图、保持背景空间、保持字体颜色、保持产品包装比例。
- 多图融合先明确参考职责：图片1锁主体身份，图片2锁场景或风格，图片3锁姿态、构图、道具或材质，再写最终画面。
- 如果用户要求风格迁移，必须写清保留哪些主体内容、迁移哪张图的风格、哪些元素不改变。
- 多对象同时编辑风险高时，优先拆成分步任务：先主体，再背景，再光影、文字或材质。

## 人物一致性

- 人物一致性优先锁脸型轮廓、五官比例、眼距、鼻形、下颌线、年龄阶段、发型、肤色、体型比例和稳定标志物。
- 表情变化只改变可见表情信号，例如嘴角、眉头、眼眶、瞳孔焦点、泪痕、呼吸紧绷，不换脸、不美化、不改年龄。
- 角色参考图优先干净背景、单人、全身或明确特写，不加入复杂剧情场景、多人互动或强遮挡光效。
- 如果参考图顺序可控，最清晰的人脸或主体图应作为身份锚点，其他图只承担服装、姿态、场景、构图或风格职责。

## 场景、道具、产品和文字

- 场景参考图默认无人空镜，写清空间类型、布局、入口/路径、关键陈设、材质、主光源、天气季节和主色调。
- 道具参考图默认单体居中，写清轮廓、材质、尺寸比例、纹理、磨损、标记和发光效果，不加入手持或杂乱背景。
- 产品图优先清晰展示品牌/包装/材质/卖点；如果用户要商业广告质感，写清背景、主光、反光、展示角度和使用场景。
- 文字海报要把需要生成的文字放进引号，说明字体风格、排版位置、配色和用途；复杂小字、长段中文和精确数据应提醒后期校对或覆盖。
- 对会被模型默认美化的内容，要写清缺失项，例如不是崭新、不是干净商品图、没有现代装潢、没有人群、没有绿叶果实。

## 质量标准

- 任务意图明确，提示词可直接生成，不依赖额外解释。
- 主体、构图、光影、色调、材质、风格和比例不互相冲突。
- 参考图职责清楚，保留项和变化项清楚。
- 人物、场景、道具或产品不会因为风格美化而丢掉原始身份锚点。
- 文字需求已明确加引号，且没有把复杂小字准确性当成唯一成功条件。

## 失败回退

- 上传素材不足时，先给纯文本可执行版本，再指出缺口。
- 用户只说“更好看”时，转成具体变化：清理背景、提高对比、强化主光、统一色调、突出主体或提升材质。
- 用户需求冲突时，优先保住主体身份和用途，再收敛风格、光线和构图。
- 不偷偷注入项目资产或其他上下文图。
