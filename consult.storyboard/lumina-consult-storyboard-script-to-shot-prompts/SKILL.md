# consult.storyboard / script-to-shot-prompts

把中文剧本/小说/对白/散文转成**按场景分组**的紧凑分镜:每个场景 12–15 秒,内部由若干一行式 shot 组成,可被视频/漫画生成模型直接执行。

---

## ⚠ Output Contract (overrides upstream prompt)

**本 skill 的输出格式优先级高于 system prompt 中任何"Markdown 分镜文本""项目元数据""字段化分镜小节"等表述。**任何冲突,一律以本 skill 为准。

### Forbidden — 一旦出现立即整体重写

- 任何项目级元数据块,例如:
  `## 项目`、`**片名：**`、`**片段范围：**`、`**总时长：**`、`**风格基调：**`、`**镜头节奏：**`、`**表演原则：**`、`**分镜策略：**`、`## 空间锚点`
- 任何字段化小节,包括但不限于:
  `**叙事目标：** / **情绪强度：** / **场景/氛围：** / **暗线/铺垫：** / **画面叙事：** / **动作与调度：** / **景别/角度/运镜：** / **视线与空间：** / **台词/旁白：** / **声音与沉默：** / **剪辑承接：**`
- `### 镜头 01：xxx（约 X 秒）` 这种镜头小标题
- `m1: ... / m2: ...` 多动作分块
- 用 bullet list (`- xxx`) 或多个段落描写同一个镜头
- 任何元信息注释段落(声音设计、光线说明、剪辑承接、暗线铺垫、统一标识、风格基调等)
- `---` 水平分割线
- 末尾总结 / 下集预告 / 制作说明
- 镜头里用 `说:` 作叙述前导写台词 — Seedance 把它当噪声而不是 lip-sync 触发器,引号才是触发器(详见对白规则)

### Required — 必须遵守

合法输出**只允许**这六种行存在:

1. (可选)顶部一行总标题 `# 分镜文本`,默认不加,用户要求才加。

2. (可选)顶部「角色锚点」块:**只写一次**,放在所有场景之前,每行一个角色,格式:
   `角色锚点：小美=粉色泡泡袖针织衫/黑色短裙/黑色短直发齐下颌/右眉骨小痣;姜夜=黑色保安制服/短寸/左耳骨钉/右手腕黑色细绳手链`
   后续镜头里出现该角色时,**直接复用锚点里已有的外观 token**,不要在镜头行里再展开新外观描述。下游 AI 视频模型靠 token-level 重复实现跨镜一致性,重复就是协议。

3. (可选)顶部「视觉参考池」单行块:放在「角色锚点」之后,声明 Seedance Universal Reference 的 12 槽位:
   `参考池：@char_main_xiaomei=小美三视图;@char_main_jiangye=姜夜三视图;@scene_baoanshi=保安室主图;@lighting_baoanshi=冷白+红频闪光场;@prop_keycard=门禁卡特写;@ambient_rain=室内雨声 5s loop;@bgm_tension=压力 BGM 8s`
   - 槽位命名:`@<purpose>_<name>`,purpose 取 `char_main` / `char_outfit` / `char_face` / `scene` / `lighting` / `prop` / `style_palette` / `motion` / `transition` / `ambient` / `bgm` / `voice`
   - 上限:9 image + 3 video + 3 audio,共 12 个;video/audio 总长各自 ≤15s
   - **≥4 shot 时建议建参考池;10+ shot 长叙事必建**;每个出主线角色/场景的镜头行需要在描述里 `(@char_main_xxx)` 显式引用
   - 项目里没有对应资产时槽位行省略,下游 flow 会 flag

4. **场景标题行**(每个场景一行,前面空一行):
   `场景X 场景名称 X秒`
   - `X秒` 是该场景所有镜头时长之和,必须落在 **12–15 秒**(用户另行指定时除外)
   - 场景名称简短,例如 `森林公园幼儿园-户外活动`、`保安室门口`、`卧室-手机亮起`
   - 场景标题后**可选追加**与上一场景的衔接信息:`场景X 名称 X秒 衔接:类型(token1+token2)`,例如 `场景2 保安室-身份反转 13秒 衔接:sound_bridge(雨声+@ambient_rain)+match_on_action(姜夜手仍掐住小美咽喉)`
   - 衔接类型枚举:`hard_cut / match_on_action / eyeline_match / sound_bridge / j_cut / l_cut / graphic_match / concept_cut`
   - 第一个场景没有上一场景,不写衔接;第二个场景起**强烈建议**写,10+ shot 长叙事必写

5. **镜头行**(场景标题下,一行一镜头,行间空一行):
   `**镜头X X.Xs 时间: ... [景别/运镜]** 描述`
   - 同一行写完:可见动作 + 情绪 + 道具/环境
   - 描述里出现主线角色/主场景时**直接内联引用参考池槽位**:`小美(@char_main_xiaomei)推开保安室(@scene_baoanshi)的门`,下游 flow 会提取这些 `@xxx` 落到 reference_refs[]
   - 有台词时直接接 `角色（情绪、动作）"台词"`(2026 原生音频模型识别引号为 lip-sync 触发器,见下方对白规则)
   - **镜头编号在整篇内连续递增**(不在每个场景里从 1 重新开始)
   - 单镜**只允许一个主运镜**,**`fast` 关键词只在 `[景别/运镜]` 字段和运镜描述里禁用**(Seedance 2.0 官方文档点名会触发 artifact);需要快节奏改用 `quick whip pan / sharp turn / sudden cut`,把"快"挪到剪辑层
   - 描述剧情/生理 `心跳加快` / `他比上次跑得更快` 这种 fast 用法是允许的(下游 flag 范围只在运镜字段)
   - **不要写焦距/光圈/景深/85mm/f/1.4 这种技术 jargon**(Seedance 对这类参数无响应),要质感请用"medium close-up / practical 钨丝顶光"这种自然描述
   - 行尾可选追加负向约束,格式:`｜避免:无字幕条/无后景人/无品牌 logo`(无边界风险时省略,**全角 `｜` 与半角 `|` 都允许**,下游统一识别)

6. (可选)**镜头行下一行的衔接行**(只在需要 image_to_video / first_last_frame 模式时写):
   `衔接:类型 模式:seedance_input_mode 首帧:@slot 重复:token1+token2`
   - 例如:`衔接:match_on_action 模式:image_to_video_first_frame 首帧:@last_frame_of_shot07 重复:@char_main_xiaomei+@scene_baoanshi`
   - 不写默认 `hard_cut + text_to_video`(下游按这个解析);**Tier 2/3 长叙事强烈建议每镜都写**

其它一概不许出现 —— 不分段、不加额外 heading、不加 bullet、不加结尾总结、不加 `---`。

## Scene Splitting Rules / 场景拆分规则

### 何时切新场景

满足任一条件就开新场景:

- **空间变化**:操场 → 教室 → 大树后 → 飞船内
- **时间跳变**:白天 → 夜晚、当下 → 闪回
- **强叙事节拍切换**:日常 → 异象出现、平静 → 危机、揭示反转、英雄登场

### 单个场景时长

- **默认 12–15 秒**(每场景),用 4–7 个镜头承载
- 用户明确指定时长时,以用户为准
- 时长超过 15 秒就拆成两个场景,例如 `场景3a 树后藏身 7秒` + `场景3b 闪电变身 8秒`,而不是塞进同一场景

### 场景命名

`地点-动作` 或 `地点-情绪锚点`:

- ✅ `森林公园幼儿园-户外活动`
- ✅ `保安室-危机爆发`
- ✅ `卧室-手机亮起`
- ❌ `开场`、`场景一`、`第一段`(过虚)
- ❌ `团团和露露豆豆在阳光明媚的幼儿园操场上玩耍`(过长)

## Shot Line Rules / 镜头行规则

每行必须包含,且**只用一行**:

1. `**镜头X` (整篇连续递增,不跨场景重置)
2. ` X.Xs ` (时长)
3. `时间: XX。` (白天/夜晚/黄昏/凌晨等)
4. ` [景别/运镜]**` (粗体闭合在景别标签后)
5. 空格 + 可见动作描述 / 情绪 / 道具
6. (可选)台词:`角色（情绪、动作）说:「台词」`

### 时长建议(单镜)

- 环境镜头: 1.5–2.5s
- 表情特写: 1.0–1.5s
- 简单插入镜头: 0.8–1.2s
- 对白行: 2.0–3.5s
- 突发动作: 0.5–1.2s
- 反转/收口: 1.5–2.5s

### 用户指定时长覆盖

如果用户说"每镜 5 秒""每场景 30 秒"等,**直接采用用户值,忽略默认**,并据此重算镜头数量与场景拆分;行格式不变,长镜头通过把更多动作/反应/台词压在同一行内承载。

## Conversion Formula

*(internal heuristic — guides reasoning; never appears in output)*

源文本 → 输出:

> 场景 + 人物设定 + 外貌 + 动作 + 表情 + 语气 + 台词 + 反转
> ↓
> 场景标题(空间+情绪锚点,12–15秒)
>   └ 镜头编号 + 时长 + 时间 + 景别/运镜 + 可见动作 + 情绪 + 台词 + 反应/异常细节

实操流程:

1. **粗切场景**:按空间/时间/节拍把原文切成若干 12–15 秒段落
2. **抽场景信息**:时间、地点、室内外、光线/天气
3. **抽人物**:姓名、外貌、情绪
4. **抽可见动作**:体态、手部、走位、转身、抓握
5. **抽台词**:长台词必要时拆成多个对白镜头
6. **加插入镜头**:手、门锁、监控、手机、阴影、倒影、道具
7. **加反应镜头**:每条关键台词或动作之后给对方面部/身体反应
8. **检查总时长**:每场景所有镜头时长之和必须 12–15 秒(或用户值)

## Shot Ordering Pattern (within a scene)

*(internal heuristic)*

短悬疑/反转节奏:
建立 → 角色登场 → 表情特写 → 对白 → 反应 → 插入 → 动作 → 反转特写 → 判决台词 → 收尾

## Camera Vocabulary

景别词典(选其一):`全景 / 远景 / 中景 / 中近景 / 近景 / 特写 / 极特写 / 反应镜头 / 插入镜头 / 主观镜头`

主运镜词典(**只选一个**,不要混):`固定 / 推镜(push-in) / 拉镜(pull-out) / 摇摄(pan) / 升降(tilt) / 跟拍(tracking) / 环绕(orbit) / 手持(handheld) / 低机位疾冲 / 缓慢推镜`

节奏副词(可选,加在主运镜后):`slow / smooth / stable / gradual / gentle`(Seedance 2.0 官方推荐) — **严禁 `fast`**(官方点名会 degrade)。

常用组合速记:

- `[全景/缓慢摇摄]` — 建立场景
- `[远景/固定]` — 大环境隔离角色
- `[中景/推镜 slow]` — 引出角色或加压
- `[中近景/固定]` — 对白配身体语言
- `[特写/固定]` — 脸/眼/嘴/手/道具
- `[特写/推镜 gradual]` — 强化关键台词或线索
- `[反应镜头/固定]` — 听者即时情绪反应
- `[插入镜头/固定]` — 按钮、门锁、手机、ID 卡、阴影、倒影
- `[快切/手持晃动]` — 突发动作、攻击、混乱(用 `quick whip` 不用 `fast`)
- `[静音镜头/固定]` — 情绪冻结或恐怖停顿
- `[特效镜头/特效呈现]` — 超自然眼睛、阴影扭曲、光晕、glitch
- `[跟拍 smooth]` / `[反打中近景/固定]` / `[主观镜头/固定]` / `[环绕 slow]` / `[抬拍/固定]` / `[低机位疾冲]`

### Multi-shot timestamp syntax (Seedance 2.0 一次成片多镜)

Seedance 2.0 一次生成可输出多镜剪辑序列(总长 ≤15s),把 `[00:00-00:05]` 识别为硬剪辑指令。当一个"镜头行"实际承载的是一个对话块或动作序列(模型自动出多个 sub-shot),**单条行内嵌时间戳硬剪辑**:

```
**镜头7 12.0s 时间: 夜晚。 [对话块/medium two-shot 起]** A 站门口,B 坐桌边,雨打玻璃。[00:00-00:02] medium two-shot;[00:02-00:05] 切 A 中近,A 眨眼喉结轻动,A（试探、压低）"你那天去过吗";[00:05-00:08] 切 B 中近反打,B 半笑 1.2s 沉默,B（警觉、半笑）"去哪";[00:08-00:12] 回 wide,A 转身离开,B 视线追到画面右出框。
```

时间戳总时长 ≤15s(Seedance 一次生成上限),无空隙、无重叠,每块只能有一个主运镜。简单单镜不需要时间戳,直接按 `**镜头X X.Xs 时间: ... [景别/运镜]**` 一行写完就行。

## Dialogue Rules

每条台词所在的镜头里必须有同行的可见动作或情绪铺垫,并按 **lip-sync 三件套** 写,Seedance 2.0 才能在同一次前向就吐出音素级对齐口型(支持 8+ 语种):

1. **引号**:用 `"..."` 直接包裹台词(不要写"说："这种叙述前导;Seedance 把引号识别为 lip-sync 触发器)。
2. **情绪/语气前缀**:放在引号前的括号里,例如 `姜夜（玩味、轻声）"奖励"` / `B（颤抖、压低）"别过来"`。Atlabs 实测:有情绪前缀的台词 lip-sync 准确度明显高于裸台词。
3. **短句**:每条台词 ≤ 12 个汉字 / 8 个英文单词。长台词拆成多个对白镜头,音节越简单口型越准。

弱(犯了所有错):
```text
**镜头4 3s 时间: 夜晚。 [中近景/固定]** 姜夜说:「奖励？」
```
问题:用 `说:` 作叙述前导(Seedance 把它当噪声而不是 lip-sync 触发器)、缺情绪前缀、镜头长度过长。

强(三件套齐):
```text
**镜头4 1.5s 时间: 夜晚。 [特写/固定]** 玻璃反光遮住姜夜半边脸,他嘴角慢慢上扬,露出意味不明的微笑。姜夜（玩味、轻声）"奖励"
```
要点:无 `说:` 前导、有情绪前缀 `（玩味、轻声）`、台词 ≤12 汉字、引号包裹是 lip-sync 触发器。

关键台词后,加一个反应镜头:
```text
**镜头5 1.5s 时间: 夜晚。 [反应镜头/固定]** 小美听见这两个字,笑容更甜,眼神带着得逞的意味,像以为姜夜已经上钩。
```

旁白 / VO / 内心 OS / 画外音**不**用引号、不写 lip-sync 三件套,改用书名号或叙述句,避免被 Seedance 误识别为口型台词触发错位嘴型:

```text
**镜头2 2.0s 时间: 夜晚。 [中近景/固定]** 陈眠盯着手机屏幕,半张脸被冷光照亮,呼吸停了一拍。旁白(陈眠 VO,压低、像被回忆拽出):七年前,她最后一次给我打电话,也是这句。
```

## Visual Specificity Rules

*(internal heuristic)*

- 抽象 `他很害怕` → 视觉 `他喉结滚动,手指扣紧桌沿,目光不自觉避开对方`
- 抽象 `她很得意` → 视觉 `她唇角压不住地上扬,指尖慢慢绕着发梢,眼神像在等猎物靠近`
- 抽象 `气氛很诡异` → 视觉 `冷白灯管闪烁两下,玻璃窗上映出的影子比本人慢了半拍`

### 表情禁形容词,用三件套替代

表情**禁止形容词化**(委屈 / 冷冽 / 慌乱 / 决绝 / 复杂 / 意味深长),写成:

1. **微表情链**:2-4 个微动作串成一条 — `先眨一下眼 → 喉结轻动 → 嘴角下压 0.5 秒 → 视线下落到桌面`
2. **眼神锚**:看哪、看多久 — `视线锁定 B 的左眼 1.6s,B 第二句出口时移到 B 嘴唇`
3. **推近触发**(可选,用景别帮模型把表情拉到中心):`从 medium two-shot 缓慢 push-in 到 close-up,在第 2.4s 处停在脸部 60% 占比`

Seedance 2.0 在 `locked medium shot + 微动作链` 这个组合上特别强(Atlabs / Apiyi 实测的甜区),锁定 medium shot 让模型自动出眨眼/呼吸/头部小幅转向/嘴角细微变化,通常比显式 push-in 更稳。表情戏单镜时长 ≥1.5s,否则微动作还没建立就被切走。

### 灯光禁通用词,用命名光术语

灯光**禁止**只写"冷光""暖光""氛围灯";改用电影/打光库术语:`Rembrandt key / butterfly key / 单盏 practical 钨丝 2700K / rim 蓝 5600K / 实景钠灯橙 2200K / 月光替光 HMI 5600K + 1/4 CTB / 监控屏冷蓝 6500K / 应急灯 1Hz 频闪`。把 `冷蓝色调,气氛压抑` 改成 `5600K 冷白荧光顶光 + 玻璃外门头灯红 3200K 频闪 1.5Hz,明暗交界线沿对角线分割面孔`,Seedance 2.0 的 prompt adherence 收益是数量级的。

## Suspense / Supernatural Texture

*(internal heuristic)*

魅魔/伪人/规则怪谈:加少量异常视觉线索,不过度解释。
影子过长 / 倒影延迟 / 瞳孔短暂转红 / 门锁 glitch / 监控雪花 / 荧光灯闪烁 / 笑容僵硬多停一拍 / 脚步停了影子继续动 / 室温低到呼气可见。
每场景挑一两个,不堆。

---

## Sensitive / Sexual Source Handling

*(internal heuristic)*

源文本若含露骨性化,保留叙事功能但改写为安全视觉:
- `布料极少,胸部呼之欲出` → `造型大胆,气质妩媚,紫色吊带短裙`
- `想做什么都可以` → `我会给你一个特别的奖励`
- 露骨肉体 → 眼神诱导、声音放轻、身体微微前倾、红唇微扬

气氛保留「魅惑、危险、诱导、悬疑」,不写露骨情色。

---

## Violence Handling

*(internal heuristic)*

突发暴力优先电影感、非血腥。
更好:`姜夜突然起身,一只手猛地伸出,瞬间扣住小美的咽喉,将她压制在墙边。`
避免血肉细节,聚焦动作、震惊、压制、反应、权力反转。

## Full Example

源文本:

```text
画面给到保安室内；姜夜隔着玻璃,露出一个微笑：奖励？
姜夜嘿嘿一笑：那还等什么？快进来！
小美连忙走进保安室。下一秒,姜夜的手,一把掐在了对方的咽喉上。小美面带惊恐,挣扎：你,你要做什么！姜夜：奖励,不一定要回家,才能领取。在这里,也一样。
```

合法输出:

```text
场景1 保安室-玻璃后的微笑 12秒

**镜头1 2.0s 时间: 夜晚。** [中景/固定] 保安室内冷白色灯管微微闪烁,姜夜坐在值班桌后,玻璃窗外隐约映出小美站在门口的身影。

**镜头8 2.0s 时间: 夜晚。 [特写/固定]** 小美面部特写,呼吸急促,瞳孔微微收缩,红唇颤抖。小美（惊恐、挣扎）"你要做什么"

**镜头11 2.5s 时间: 夜晚。** [反应镜头/固定] 小美瞳孔骤然放大,挣扎动作慢半拍才反应过来,眼神从惊恐变成意识到不对的呆滞。
```

## Long-Duration Example (用户要求每镜 12-15 秒)

如果用户给的单镜时长本身已经 ≥ 12 秒,**单个镜头本身就构成一个场景**,场景标题 = 该镜核心动作:

```text
场景1 卧室-短信亮起 13秒

**镜头1 13.0s 时间: 夜晚。** [极近特写→中近景/固定轻推] 黑暗卧室里陈眠手中手机屏幕忽然亮起,占满画面的短信弹窗只有一句「别看天」,屏幕轻轻下滑停在发信号码尾号 0719;镜头慢慢拉到陈眠半张脸被冷光切亮的中近景,她拇指悬在屏幕上方,既不点开也不回复,呼吸停了一拍,桌面右前景的旧笔记虚虚露出一角。陈眠（VO,压低、像被回忆拽出）说:「七年前,她最后一次给我打电话,也是这句。」

场景2 卧室-母亲笔记与闪回 14秒

**镜头2 14.0s 时间: 夜晚。** [近景→特写/匹配剪辑闪回] 陈眠拿手机的手微微下垂,镜头顺势带到桌面右侧那页发黄旧笔记,母亲字迹「今晚别看天」被看清的瞬间,匹配剪辑切入七年前夜空——三个月亮悬在城市上方、一座倒悬城市仅一眼掠过、街头人群中突然空出大片站位、白色防寒服背影走进强光下一帧消失,所有声音被短暂真空化半拍。陈眠（VO,克制到发冷）说:「那一年,月亮从一个变成三个。十一秒后,三百万人失踪。我妈也在那一年,死在南极。」
```

## Format Drift Self-Check

提交前扫一遍草稿,只要命中**任意一条**就整体重写:

- 出现任何 `## 项目` / `**片名：**` / `**总时长：**` / `**风格基调：**` / `## 空间锚点` 等元数据块
- 出现任何 `**叙事目标：**` / `**情绪强度：**` / `**画面叙事：**` / `**剪辑承接：**` 等字段化小节
- 出现任何 `### 镜头 X` / `### 场景 X：xxx（约 X 秒）` 风格的 heading
- 出现 `m1:` / `m2:` 动作分块
- 出现 bullet list (`- xxx`) 描写镜头内部
- 出现 `---` 水平分割线分隔场景
- 镜头编号在场景间被重置(应整篇连续递增)
- 任意场景总时长 < 12 秒或 > 15 秒(用户另行指定时除外)
- 镜头行没有以 `**镜头X X.Xs 时间: ... [景别/运镜]**` 开头/闭合
- 出现末尾总结 / 制作说明 / 风格基调段
- 单镜出现两个或以上主运镜(`先推近再环绕` 等);需要切换运镜应改为时间戳 `[00:00-00:02] push;[00:02-00:05] orbit` 或拆成两镜
- `[景别/运镜]` 字段或镜头描述里的运镜段落出现 `fast` 关键词(Seedance 官方 degrade 词;改用 `quick whip pan / sharp turn / sudden cut`);剧情/生理 `心跳加快` 类用法不算
- 出现 `85mm / f/1.4 / shallow DOF / focal plane` 这种技术 jargon(Seedance 不响应)
- 台词没用引号包裹、没有情绪前缀、单条 > 12 个汉字(lip-sync 三件套未齐)
- 用 `说:` / `说道:` 作叙述前导写台词(Seedance 把它当噪声);引号才是 lip-sync 触发器
- 表情用了形容词 `委屈 / 冷冽 / 慌乱 / 决绝 / 复杂` 这种空词,没拆成微表情链
- 灯光只写"冷光""暖光""氛围灯"这种通用词,没用命名光术语
- 时间戳总长 > 15 秒(Seedance 一次生成上限)
- ≥4 shot 但顶部没有 `参考池：` 行;或镜头里引用了 `@xxx` 但顶部参考池没声明该槽位
- 第二个场景起场景标题没带 `衔接:` 后缀,且全片 >2 处出现 0 token 共享(等于跳剧)
- 镜头行下一行的 `衔接:` 行写了非枚举值的衔接类型(只允许 `hard_cut / match_on_action / eyeline_match / sound_bridge / j_cut / l_cut / graphic_match / concept_cut`)
- 衔接行里 `模式:` 写了非枚举值(只允许 `text_to_video / image_to_video_first_frame / first_last_frame / ref_pack / nine_panel_grid_i2v / video_extend`)
- 衔接行模式不是 `text_to_video` 但 `首帧:` 留空

合法的行只有六种:
1. (可选)顶部 `# 分镜文本`
2. (可选)顶部 `角色锚点：A=...;B=...` 单行
3. (可选)顶部 `参考池：@slot=描述;@slot=描述...` 单行
4. 场景标题行 `场景X 名称 X秒` 或 `场景X 名称 X秒 衔接:类型(token)`
5. `**镜头X X.Xs 时间: ... [景别/运镜]** ...` 的镜头行
6. (可选)镜头行下一行 `衔接:类型 模式:xxx 首帧:@xxx 重复:xxx`

其它一概不允许。

## Final Checklist Before Delivering

- [ ] 是否按场景分组?每个场景以 `场景X 名称 X秒` 开头?
- [ ] 每个场景总时长是否落在 12–15 秒(或用户指定值)?
- [ ] 多角色出现时,顶部是否有 `角色锚点：...` 块,且镜头里复用锚点 token 而不是重新展开外观?
- [ ] ≥4 shot 时是否有 `参考池：@xxx=...;@xxx=...` 单行 manifest?10+ shot 长叙事是否绑了 `@char_main_*` + `@scene_*` + `@lighting_*` 至少 3 类槽位?
- [ ] 出主线角色/主场景的镜头行描述里是否内联 `(@char_main_xxx)` / `(@scene_xxx)` 引用?引用的槽位是否都在顶部参考池声明过?
- [ ] 第二个场景起,场景标题行是否带 `衔接:类型(token)` 后缀?跨场景必给 sound_bridge / J-cut / L-cut / match_on_action 任一?
- [ ] Tier 2/3 长叙事(≥4 shot)是否在镜头行下一行写了 `衔接:类型 模式:xxx 首帧:@xxx 重复:xxx`?
- [ ] 镜头编号是否整篇连续递增,不在每个场景里重置?
- [ ] 每个镜头行是否以 `**镜头X X.Xs 时间: ... [景别/运镜]**` 开头并闭合?
- [ ] 镜头是否一镜一行,没有被拆成多段?
- [ ] 动作是否可见、具体,不是抽象心理描述?
- [ ] 每条对白是否有同行的动作或情绪铺垫?
- [ ] 对白是否符合 lip-sync 三件套(引号 + 情绪前缀 + ≤12 个汉字)?
- [ ] 表情是否用三件套(微表情链 + 眼神锚 + 可选推近触发),没用形容词?

- [ ] 关键对白后是否有反应镜头?
- [ ] 道具/环境是否被用来加压?
- [ ] 边界条件镜头(易出字幕条/品牌 logo/穿模)是否在行尾追加 `｜避免:...`?
- [ ] 需要一次成片的对话块是否用了 `[00:00-00:05]` 时间戳硬切,且总长 ≤15s?
- [ ] 露骨性描写是否被改写为安全视觉语言但保留叙事功能?
- [ ] 暴力是否电影化而非血腥堆叠?
- [ ] 是否完全没有项目级元数据块、字段化小节、bullet、副标题、`---` 分割线、末尾总结?
