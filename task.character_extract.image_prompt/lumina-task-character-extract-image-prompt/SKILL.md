# task.character_extract.image_prompt

## 适用场景

- 已完成人物抽取后，为单个人物整理可执行的人物参考图任务。
- 需要让后续短剧、分镜、图选图、图生图和视频参考图稳定复用同一人物身份、服饰和状态。

## 负责什么

- 把人物名称、身份、基础外观和关键状态压实成可生图的人物资产提示词。
- 为主角、关键配角和外观信息充分的人物优先规划干净的正侧背三视图基准图。
- 根据上游记录的服饰、状态、形态和特写需求，规划必要的全身变体或局部特写。
- 把身份不变项和状态可变项分开写，避免换服装、换表情、换光线时重造一张新脸。
- 控制提示词密度：抓住身份锚点、基础服饰、状态差异、构图和负向约束，不堆微小纹理或复杂剧情。

## 不负责什么

- 不新增人物，不改写上游已经确认的人物身份。
- 不把一个人物拆成多条实体任务。
- 不生成剧情海报或分镜图，不把完整客厅、街道、战场、宫殿等具体场景画进人物资产图。
- 不凭空创造服饰状态变体；没有基准身份时不强行写复杂图生图变化。

## 核心方法论

- 使用完整自然语言描述，不写关键词标签流、权重括号、命令行参数或 provider 私有语法。
- 人物资产图优先服务复用：单人、干净背景、轮廓清楚、服饰结构清楚、面部稳定。
- 基准图锁身份，变体图只改服饰、状态、形态或特写焦点；脸型、五官比例、眼距、鼻形、下颌线、年龄阶段、发型轮廓、体型比例和稳定标志物必须保持。
- 不要默认美化、年轻化、瘦身、换发型、换肤色或换物种；文本里的病弱、贫穷、战损、劳作痕迹和非典型外貌要保留。
- 人物参考图不承担完整叙事。可以用极简背景和少量光线暗示气质，但不要加入多余人物、剧情互动、复杂道具堆叠或环境故事。

## 资产变体决策协议

- 每个人物图片任务先判断是否需要多图资产链路；输出中不要写推理过程，只把结果落实到 prompt 和 variants。
- 主角、关键配角、反复出现人物或外观信息充分的人物，必须输出 variants，第一张必须是 `turnaround_sheet`。
- 如果人物属性或描述中出现可复用的服饰、状态、季节服装、职业/场合服、受伤病弱、特殊形态或局部特写需求，必须继续输出对应派生 variants。
- 如果没有明确变体依据，不要为了丰富画面虚构睡衣、战斗服、冬装、伤口或特殊形态。
- 每个派生 variant 必须分离身份不变项和本次变化项，并明确写出“保持同一人物身份，只改变本次服饰/状态/形态/特写焦点”。
- variants 第一项是文生图基准图；后续 variants 默认会使用第一张基准图作为图生图身份参考。
- 每个 variant 必须填写英文小写 snake_case 的 `variantKey` 和清晰 `title`，不要使用 `variant_1`、`style_a`、`image_b` 这类无语义命名。

## 人物 variantKey 词表

- `turnaround_sheet`：正侧背三视图基准设定图，锁定身份、脸、发型、体型、基础服饰和稳定标志物。
- `three_quarter_body`：三分之四全身或中全身参考，用于补足体积、姿态和视频参考。
- `home_body`：居家态、日常家居服或室内常服。
- `sleepwear_body`：睡衣态、夜间居家态；只改变服装和轻微生活状态，不改变脸和体型。
- `outdoor_body`：外出态、通勤态、上学态、旅行态或街景复用服饰。
- `seasonal_outfit_body`：季节服装，例如冬装、雨天外套、夏季轻装；只改变服饰厚度、材质和配件。
- `battle_body`：战斗态、战袍、护甲、武器佩戴和战损。
- `work_body`：工作态、职业制服、劳作服或任务服。
- `formal_body`：正式服、礼服、宴会服、婚服或典礼服。
- `injured_body`：受伤、战损、血痂、绷带、破损衣物等可见状态。
- `sick_body`：病弱、病号服、苍白脸色、虚弱姿态等可见状态。
- `special_form_body`：妖化、觉醒、灵体、兽形、伪装或其他特殊形态。
- `expression_closeup`：关键情绪或转折点面部特写。
- `detail_closeup`：手部、伤口、面具、法印、纹身、稳定饰物等局部特写。

## 基准三视图

- 主角、关键配角、反复出现人物或外观信息充分的人物，优先做正侧背三视图基准图。
- 三视图必须写清：同一人物、正面/侧面/背面并列、全身、姿态中性、比例一致、基础服饰清楚、发型轮廓清楚、干净浅色背景。
- 三视图不是三个人同框。提示词要明确这是同一角色的多个视角，不是三名不同角色。
- 三视图只锁身份、轮廓、基础服饰和发型，不承担战斗、受伤、哭泣、黑化、告白等剧情状态。
- 基础服饰应选择最常复用或最能代表身份的服装；不要把所有服饰变体挤进一张图。

## 服饰与状态变体

- 服饰/状态变体仍然是人物参考图，优先单人全身或中全身，背景保持简单。
- 每个变体都要写清“保持同一人物身份，只改变本次服饰或状态”。
- 居家态、外出态、战斗态、受伤态、病弱态、正式态、伪装态、特殊形态应分别承载，不要在一张图里混用。
- 只改变本变体需要改变的内容。例如战斗态改战袍、防具、武器佩戴和战损，不要同时改年龄、脸型、发色、身高和物种。
- 受伤、病弱、劳作、贫困、战损必须写成可见表面：苍白脸色、绷带、血痂、泥污、粗糙手掌、破损衣袖、焦黑甲片。
- 如果状态反常规，要写出缺失项，例如不呈现健康红润、干净新衣、现代时尚造型、光滑无伤皮肤。

## 表情与局部特写

- 特写只在有明确复用价值时使用，优先服务台词反应、身份暴露、关系转折、情绪矛盾、濒死、黑化、觉醒等节点。
- 表情变化只改变可见表情信号：嘴角、眉头、眼眶湿润、瞳孔焦点、泪痕、呼吸紧绷、面部肌肉压住。
- 面部特写必须保持原人物的脸型、眼距、鼻形、下颌线、年龄阶段、发型边界和稳定标志物。
- 手部、伤口、面具、法印、纹身等局部特写必须和人物身份相关，不要变成独立道具图。
- 不用“开心、悲伤、震惊”这类抽象词单独驱动特写；必须补可见面部或肢体信号。

## 构图与风格

- 基准三视图和全身变体优先竖构图或方构图，保证全身完整、不裁切头脚、不遮挡服饰结构。
- 面部特写和局部特写可以使用方构图或近景构图，焦点明确，背景简单。
- 风格应继承项目整体生图风格；风格只修饰渲染质感，不覆盖人物身份。
- 角色参考图允许克制光影，但不要使用强烈体积光、复杂粒子、战斗特效或景深遮挡身份细节。

## 负向约束

- 常规排除：不同脸、不同发型、年龄改变、三名不同角色、多人同框、复杂背景、剧情场景、文字、logo、水印、脸部变形、肢体错误、畸形手。
- 三视图排除要针对“三名不同角色、不同脸、服饰不一致”，不要排除多视角排版本身。
- 特写排除多余人物、遮挡脸、过度夸张表情、严重变形、文字和水印。

## 质量标准

- 一眼能看出是谁，并且适合继续沉淀为人物参考图。
- 基准三视图像同一个人，不是三个人；全身比例、发型轮廓和服饰结构清楚。
- 多个变体之间身份一致，只改变上游明确要求的服饰、状态、形态或特写焦点。
- 稳定伤病、劳作、贫困、战损或特殊形态不会被默认美化抹平。
- 提示词可直接生成，不依赖解释性说明才能理解。

## 失败回退

- 信息不足时，优先保住身份稳定、基础服装和干净三视图，不强造精确五官。
- 人物只被轻微提及且外观不足时，可以只做朴素单人参考；主角和关键配角不要因为省图放弃基准三视图。
- 没有明确服饰或状态变体时，不硬造变体；后续剧情需要时再补。
