# task.character_extract.image_prompt

## 适用场景

- 已完成人物抽取后，为单个人物整理一条可直接出图的任务提示。

## 负责什么

- 把人物名称、身份和关键特征压实成一条可执行图片任务。
- 让画面主体、服装、气质、构图重点保持统一。
- 需要时补充克制的反向约束，减少跑偏。
- 为同一人物建立可长期复用的参考图；默认一张基准图即可，只有确实需要时才补充动作角度或表情锚点。

## 不负责什么

- 不新增人物。
- 不把一个人物拆成多条任务。
- 不改写上游已经确认的人物身份。

## 判断原则

- 主体必须稳定可辨认。
- 提示内容优先服务角色设定建立，而不是剧情分镜。
- 画面语言直接、具体、可投喂。
- 不把剧情瞬间、极端表演、一次性伤病或临时环境写成全部参考图的共同设定，除非它是人物长期稳定状态。
- 如果输出多个 variants，服装、身形、发型、年龄阶段、脸部特征必须保持一致；每个 variant 只改变视角/景别/表情目的。

## 工作方法

- 先吃透当前人物是谁、最该被看见的特征是什么。
- 默认输出 1 张基准设定图；只有解析事实显示同一人物需要额外动作角度或表情锚点时，才写入 variants。
- 如果写入 variants，第一张必须是 `turnaround_sheet`，作为系统后续图生图的基准图：同一人物 front / side / back 三视图并列，姿态中性，服装结构清楚，背景简单，不出现剧情动作，也不是三个人同框剧情。
- 可按事实需要补充 `three_quarter_body`：后续图生图变体，单人三分之四全身或中全景，适合视频动作参考，必须保留同一物种/身份、服装和身形比例。
- 可按事实需要补充 `expression_closeup`：后续图生图变体，单人面部特写，保留同一发型、年龄阶段和脸部特征，表情可体现角色气质但不过度戏剧化。
- 再决定适合的画面景别、服装状态和气质表达。
- 最终整理成一条人物资产任务；没有必要多图时不要写 variants。只有输出多个 variants 时，系统才会编排成同一身份的“文生图基准图 → 图生图派生图”流程。

## 质量标准

- 一眼能看出是谁。
- 画面目标明确，没有空泛修辞。
- 结果适合继续沉淀为人物参考图。
- 多个 variants 之间必须像同一个人，而不是不同造型或不同年龄版本。

## 失败回退

- 信息不够时，优先保住身份稳定和核心外观。
- 不额外捏造原文没有支持的设定。
- 外观信息不足时，用身份、年龄阶段、服装年代、气质和文本风格建立克制参考，不强造精确五官。
