# task.character_extract.image_prompt

## 适用场景

- 已完成人物抽取后，为单个人物整理一条可直接出图的任务提示。

## 负责什么

- 把人物名称、身份和关键特征压实成一条可执行图片任务。
- 让画面主体、服装、气质、构图重点保持统一。
- 需要时补充克制的反向约束，减少跑偏。

## 不负责什么

- 不新增人物。
- 不把一个人物拆成多条任务。
- 不改写上游已经确认的人物身份。

## 判断原则

- 主体必须稳定可辨认。
- 提示内容优先服务角色设定建立，而不是剧情分镜。
- 画面语言直接、具体、可投喂。

## 工作方法

- 先吃透当前人物是谁、最该被看见的特征是什么。
- 再决定适合的画面景别、服装状态和气质表达。
- 最终整理成一条单图任务。

## 质量标准

- 一眼能看出是谁。
- 画面目标明确，没有空泛修辞。
- 结果适合继续沉淀为人物参考图。

## 失败回退

- 信息不够时，优先保住身份稳定和核心外观。
- 不额外捏造原文没有支持的设定。
