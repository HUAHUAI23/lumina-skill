# task.character_extract.image_prompt

## 适用场景

- 已完成人物抽取后，为单个人物整理可执行的人物参考图任务。
- 需要让后续短剧、分镜、图选图、图生图和视频参考图稳定复用同一人物身份、服饰和状态。

## 负责什么

- 把人物名称、身份、基础外观和关键状态压实成可生图的人物资产提示词。
- 为主角、关键配角和外观信息充分的人物优先规划干净的正侧背三视图基准图。
- 根据上游记录的服饰、年龄阶段、稳定状态和特殊形态，规划必要的状态三视图或主视图资产；表情、眼部、嘴唇、手部和伤口局部特写不在资产库长期维护。
- 把身份不变项和状态可变项分开写，避免换服装、换表情、换光线时重造一张新脸。
- 控制提示词密度：抓住身份锚点、基础服饰、状态差异、构图和负向约束，不堆微小纹理或复杂剧情。

## 不负责什么

- 不新增人物，不改写上游已经确认的人物身份。
- 不把一个人物拆成多条实体任务。
- 不生成剧情海报或分镜图，不把完整客厅、街道、战场、宫殿等具体场景画进人物资产图。
- 不凭空创造服饰状态变体；没有基准身份时不强行写复杂图生图变化。

## 核心方法论

- 使用完整自然语言描述，不写关键词标签流、权重括号、命令行参数或 provider 私有语法。
- 人物资产图优先服务复用：单人、干净背景、轮廓清楚、服饰结构清楚、面部稳定。
- 基准图锁身份，状态资产只改服饰、状态或形态；年龄阶段资产只改年龄成熟度和阶段体态。脸型家族相似性、五官比例关系、眼距、鼻形、下颌趋势、发型线索和稳定标志物必须保持。
- 不要默认美化、年轻化、瘦身、换发型、换肤色或换物种；文本里的病弱、贫穷、战损、劳作痕迹和非典型外貌要保留。
- 人物参考图不承担完整叙事。可以用极简背景和少量光线暗示气质，但不要加入多余人物、剧情互动、复杂道具堆叠或环境故事。

## 资产变体决策协议

- 每个人物图片任务先判断是否需要多图资产链路；输出中不要写推理过程，只把结果落实到 prompt 和 variants。
- 主角、关键配角、反复出现人物或外观信息充分的人物，输出 variants，第一张是身份三视图或主视图资产；没有稳定状态依据时只输出这一张。
- 如果人物属性或描述中出现可复用的服饰、状态、季节服装、职业/场合服、受伤病弱、特殊形态或年龄阶段，才继续输出对应状态三视图/主视图 variants。
- 如果没有明确变体依据，不要为了丰富画面虚构睡衣、战斗服、冬装、伤口或特殊形态。
- 每个派生 variant 必须分离身份不变项和本次变化项，并明确写出“保持同一人物身份，只改变本次服饰/状态/形态/年龄阶段”。
- variants 第一项是文生图基准图；后续 variants 默认会使用第一张基准图作为图生图身份参考。
- 增量抽取已有基准图时，新增服饰、状态、特殊形态或年龄阶段 variants 默认使用现有人物基准图/最接近阶段图作为图生图身份参考；不要重新输出新的基础身份图，除非原基准图质量或身份锚点明显不可用。
- 每个 variant 必须填写 `assetDescription`，这是结构化 schema 的必填字段：一句话说明唯一身份是谁、这张资产是基础身份还是哪个状态三视图、它供 story_to_video 在什么镜头决策中使用。
- `variantKey` 只是技术去重标识，不使用固定词表；可以用简短 snake_case，但决策信息必须写在 `assetDescription`。

## 基准三视图

- 主角、关键配角、反复出现人物或外观信息充分的人物，优先做正侧背三视图基准图。
- 三视图必须写清：同一人物、正面/侧面/背面并列、全身、姿态中性、比例一致、基础服饰清楚、发型轮廓清楚、干净浅色背景。
- 三视图不是三个人同框。提示词要明确这是同一角色的多个视角，不是三名不同角色。
- 三视图只锁身份、轮廓、基础服饰和发型，不承担战斗、受伤、哭泣、黑化、告白等剧情状态。
- 基础服饰应选择最常复用或最能代表身份的服装；不要把所有服饰变体挤进一张图。

## 服饰与状态变体

- 服饰/状态变体仍然是人物参考图，优先单人全身或中全身，背景保持简单。
- 每个变体都要写清“保持同一人物身份，只改变本次服饰、状态或年龄阶段”。
- 居家态、外出态、战斗态、受伤态、病弱态、正式态、伪装态、特殊形态应分别承载，不要在一张图里混用。
- 只改变本变体需要改变的内容。例如战斗态改战袍、防具、武器佩戴和战损，不要同时改年龄、脸型、发色、身高和物种；年龄阶段图则只改变年龄成熟度、身高比例、脸部稚嫩/成熟程度、阶段服饰和体态。
- 受伤、病弱、劳作、贫困、战损必须写成可见表面：苍白脸色、绷带、血痂、泥污、粗糙手掌、破损衣袖、焦黑甲片。
- 如果状态反常规，要写出缺失项，例如不呈现健康红润、干净新衣、现代时尚造型、光滑无伤皮肤。

## 年龄阶段变体

- 童年、少年、青年、成年、老年等属于同一人物的年龄阶段 variants，不是新人物任务。
- 例如“童年的沈深”和“青年的沈深”应在同一个沈深图片任务中输出不同年龄阶段图；assetDescription 要写清“沈深童年阶段”“沈深青年阶段”，并说明供 story_to_video 按时间线选图。
- 年龄阶段图必须保持身份延续：核心脸型家族相似性、眼距和眉眼关系、鼻形、下颌趋势、发色/发际线线索、稳定胎记/疤痕/饰物等不变或有合理演化。
- 童年阶段可以改变身高比例、脸部稚嫩度、发型整理方式、儿童服饰和体态；青年/成年阶段可以改变成熟度、服饰、身形和气质，但不能换成另一张脸。
- 如果文本只出现一个年龄阶段，不为了完整人生默认补其他阶段；只有上游明确记录多个年龄阶段或后续会复用时才输出年龄 variants。

## 表情与局部特写

- 特写只在 story_to_video 具体镜头里派生，不作为本阶段 variants 输出。
- 表情变化只作为镜头级文本表演信号：嘴角、眉头、眼眶湿润、呼吸紧绷、视线停住、面部肌肉压住；不要输出眼部、嘴唇、表情或手指局部 variants。
- 右眼血纱布、固定疤痕、面具裂缝、常亮法印、长期战损这类会跨镜头复用的状态，可以进入人物状态三视图或主视图；一次性伤口、血迹、握持、掐诀、嘴角抽动和瞳孔变化不进入资产库。
- 面部或局部特写如果后续确实需要，由 story_to_video 基于人物三视图/状态图做镜头级 img2img；本阶段只保证身份锚点和稳定状态干净可用。
- 不用“开心、悲伤、震惊”这类抽象词单独驱动资产图；需要状态资产时必须是稳定可见状态，而不是瞬时表情。

## 构图与风格

- 基准三视图和全身变体优先竖构图或方构图，保证全身完整、不裁切头脚、不遮挡服饰结构。
- 镜头级面部特写和局部特写由 story_to_video 基于三视图/状态图做 img2img，不在本阶段选择近景构图。
- 风格应继承项目整体生图风格；风格只修饰渲染质感，不覆盖人物身份。
- 角色参考图允许克制光影，但不要使用强烈体积光、复杂粒子、战斗特效或景深遮挡身份细节。

## 负向约束

- 常规排除：不同脸、非本变体要求的年龄改变、三名不同角色、多人同框、复杂背景、剧情场景、文字、logo、水印、脸部变形、肢体错误、畸形手。非年龄阶段图还要排除不同发型和年龄漂移；年龄阶段图只允许年龄、体态和阶段服饰按上游要求变化。
- 三视图排除要针对“三名不同角色、不同脸、服饰不一致”，不要排除多视角排版本身。
- 镜头级特写派生排除多余人物、遮挡脸、过度夸张表情、严重变形、文字和水印；这些不作为本阶段 variants。

## 质量标准

- 一眼能看出是谁，并且适合继续沉淀为人物参考图。
- 基准三视图像同一个人，不是三个人；全身比例、发型轮廓和服饰结构清楚。
- 多个变体之间身份一致，只改变上游明确要求的服饰、状态、形态或年龄阶段。
- 稳定伤病、劳作、贫困、战损或特殊形态不会被默认美化抹平。
- 提示词可直接生成，不依赖解释性说明才能理解。

## 失败回退

- 信息不足时，优先保住身份稳定、基础服装和干净三视图，不强造精确五官。
- 人物只被轻微提及且外观不足时，可以只做朴素单人参考；主角和关键配角不要因为省图放弃基准三视图。
- 没有明确服饰或状态变体时，不硬造变体；后续剧情需要时再补。
