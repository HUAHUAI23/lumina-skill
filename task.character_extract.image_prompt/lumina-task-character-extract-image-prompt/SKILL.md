# task.character_extract.image_prompt

## 适用场景

- 已完成人物抽取后，为单个人物整理可执行的人物参考图任务。
- 需要让后续短剧、分镜、图选图和图生图稳定复用同一人物身份、服饰和状态。

## 负责什么

- 把人物名称、身份和关键特征压实成可生图的人物资产提示词。
- 为主角、关键配角和信息充分的人物优先规划正侧背三视图基准图。
- 根据上游 `outfit_variants`、`state_variants`、`form_variants`、`life_stage_variants`、`closeup_needs` 规划服饰/状态/形态/特写变体。
- 让后续变体适合走“基准三视图文生图 -> 变体图生图”的链路，保持人物身份一致。
- 控制提示词密度：抓住主题、身份锚点、服饰状态和构图约束，不堆过细微观描述。

## 不负责什么

- 不新增人物，不改写上游已经确认的人物身份。
- 不把一个人物拆成多条实体任务。
- 不生成剧情分镜图，不把完整场景画进人物资产图。
- 不用文生图凭空重造同一人物的服饰状态变体；有基准图后，变体提示必须明确保持基准身份。

## 核心方法论

- 图片提示词采用“主体 + 用途 + 构图 + 关键外观 + 状态/服饰 + 风格 + 负向约束”的短结构。
- 角色资产图不是剧情海报：优先清楚、干净、可复用，而不是复杂光效和动态故事瞬间。
- 人物基准图必须是干净底图：单一人物、无剧情场景、无路人、无复杂背景；可用浅灰/纯色/透明感背景。
- 多状态人物不要用一张图塞满所有服装；用多个 `variants` 分别承载居家、外出、战斗、受伤、特殊形态等可复用状态。
- 后续 `variants` 会由系统以第一张基准图作为图生图参考，因此每个变体都要写“保持同一脸型、发型、体型比例、年龄阶段和稳定标志物，只改变本变体需要改变的服饰/状态”。
- 提示词不要过度精确到大量微小纹理、饰品数量或复杂背景，否则容易增加噪点；关键是主题清楚、人物干净、运动/状态目标明确。

## variants 规划规则

- 主角、关键配角、反复出现人物、或任何有明确外观信息的人物，优先输出 `variants`，第一张必须是 `turnaround_sheet`。
- 边缘人物或外观信息极少时，可以只输出顶层单图；但只要输出 `variants`，第一张一定是三视图基准图。
- `turnaround_sheet`：正侧背三视图基准设定图。同一人物 front / side / back 三个视图并列，姿态中性，全身，服装结构清楚，发型轮廓清楚，背景干净。必须明确这是同一人物三视图，不是三个人同框剧情。
- `three_quarter_body`：三分之四全身参考，用于后续视频动作和图选图。保持基准身份和基础服饰，可有轻微自然站姿，不加入场景。
- `expression_closeup`：面部表情特写，用于台词、反应、情绪矛盾和转折点。保持同一脸型、发型、年龄阶段；表情要外化但不过度戏剧化。
- 如果上游记录了稳定服饰或状态，必须用明确 `variantKey` 输出对应变体，例如 `homewear_body`、`outdoor_body`、`battle_body`、`injured_body`、`formal_body`、`disguise_body`、`special_form_body`、`young_stage_body`、`turning_point_closeup`。
- 状态/服饰变体尽量是单人全身或中全身干净参考图；除非 `closeup_needs` 要求，才做特写。
- 不要让服饰变体变成新人物：脸、发型、体型、身高比例、年龄感和稳定标志物必须一致。

## 三视图提示词规范

- 三视图必须写清：
  - 同一人物的正面、侧面、背面三视图并列。
  - 全身、姿态中性、双臂自然下垂或轻微展开，比例一致。
  - 基础服装是后续最常复用或最能代表身份的服装。
  - 纯色/浅灰/白底，不带室内、街道、战场、光效背景。
  - 不带剧情动作、夸张表情、多人互动、文本标注、logo、水印。
- 三视图不是“横向三人合照”。要在 prompt 中明确“same character, not three different characters”或中文等价约束。
- 三视图重点锁身份、轮廓、服饰结构和发型，不承担战斗、受伤、情绪爆发等剧情状态；这些写入后续变体。

## 服饰与状态变体规范

- 居家、外出、战斗、受伤、特殊形态等变体都必须引用基准身份：同一脸型、发型、体型比例、肤色/物种特征、年龄阶段不变。
- 变体图仍是人物参考图，原则上不带完整场景；可以用极简背景和少量光线暗示氛围，不画客厅、街道、战场等具体空间。
- 只改变本变体需要改变的内容。例如 `battle_body` 改战袍、防具、战损；不要同时改年龄、脸型、发色和身高。
- 如果稳定状态反常规，应写默认特征缺失。例如“不呈现健康红润、干净新衣、现代时尚造型”，避免模型自动美化。
- 受伤、病弱、劳作、贫困和战损必须写可见表面：苍白脸色、血痂、绷带、泥污、粗糙手掌、破损衣袖、焦黑甲片等。

## 特写规范

- 特写只在有明确复用价值时输出，优先服务台词、反应、身份暴露、关系转折、情绪矛盾突出、濒死、黑化、觉醒等转折点。
- `expression_closeup` 可用于常规台词和反应；`turning_point_closeup` 可用于强转折点，例如眼神从平静转冷、嘴唇吐出关键字、泪水停在眼眶、面具裂开露出真实身份。
- 部位特写可以用自定义 `variantKey`，如 `hand_detail`、`eye_closeup`、`wound_detail`、`mask_detail`。必须保持和人物身份相关的稳定特征，不要变成道具单体图。
- 情绪写成可见动作和面部信号，不只写抽象词。

## 负向约束

- 人物参考图常用负向约束：不同脸、不同发型、同一张图内服饰不一致、三名不同角色、多人同框、复杂背景、剧情场景、文字、logo、水印、畸形手、肢体错误、脸部变形。
- 三视图负向约束要避免“three different characters / different face / inconsistent outfit”，不要排除“multiple views”。
- 特写负向约束要排除多余人物、遮挡脸、严重变形、夸张表情、文字和水印。

## 输出要求

- 顶层 `prompt` 写人物总体设定和风格总纲。
- 输出 `variants` 时，每个 variant 的 `prompt` 必须可单独生成该图，不依赖读者回看其他段落。
- 第一张 `turnaround_sheet` 的 `prompt` 要写明“三视图/正面侧面背面/同一人物/干净背景/无剧情场景”这些约束。
- 后续变体的 `prompt` 要写明“保持基准人物身份一致，只改变本变体服饰或状态”的约束。
- `variantKey` 使用英文小写 snake_case，便于后续资产检索和图选图。
- 比例建议：三视图和全身变体用 `9:16` 或可容纳三视图的竖图/方图；面部和部位特写可用 `1:1`。

## 质量标准

- 一眼能看出是谁，且适合继续沉淀为人物参考图。
- 三视图像同一个人，不是三个人；服饰结构和发型轮廓清楚。
- 多个变体之间身份一致，只改变上游明确要求的服饰、状态、形态或特写焦点。
- 稳定伤病、劳作、贫困、战损或特殊形态不会被默认美化抹平。

## 失败回退

- 信息不够时，优先保住身份稳定、基础服装和干净三视图，不强造精确五官。
- 不额外捏造原文没有支持的服饰、状态或形态。
- 如果人物只被轻微提及且外观不足，可输出单图；但主角和关键配角不要因为省图放弃三视图基准。
