# task.character_extract

## 适用场景

- 从小说、剧本、短剧分场、故事梗概中抽取稳定人物资产。
- 为后续人物三视图/主视图、状态三视图、分镜图选图和视频参考图建立人物库。

## 负责什么

- 优先把文本中的人物提及映射到已存在人物资产，合并同一人物的姓名、外号、代词和关系称谓。
- 人物抽取必须严格覆盖：凡是有姓名、稳定称谓、说话、被旁白指代、直接行动、被镜头观看或后续需要保持脸/服饰连续的人物，都必须进入 `newEntities`、`existingEntityUpdates` 或 `unresolvedMentions`，不能因为外观信息少而漏掉。
- 找出值得长期维护的人物：主角、关键配角、反复出现的人物、推动剧情的人物，以及后续画面必须稳定复用的人物。
- 为人物保留可生图的一致性锚点：年龄阶段、物种/身份、脸部轮廓、五官比例、发型、身形比例、基础服饰、稳定标志物、常见服饰状态和可见历史痕迹。
- 把同一人物的多种稳定状态记录为同一个身份下的可复用资产说明，例如居家、外出、睡觉、战斗、受伤、病弱、伪装、特殊形态和不同年龄阶段。
- 识别后续需要长期维护的身份资产，例如基础三视图、外出状态三视图、居家状态三视图、睡眠状态三视图、战斗/受伤/特殊形态三视图。

## 不负责什么

- 不抽取场景和独立道具；只记录人物身上的稳定服饰、随身标志物或身份道具。
- 不直接写最终图片提示词，不规划图片任务节点。
- 不把一次性动作、单镜头表情、临时站位、路人群像沉淀成人物资产。
- 不把同一人物的每套衣服拆成新人；身份相同就优先记录为同一人物的服饰或状态变体。
- 不为凑齐设定而发明文本没有支持的脸型、服装、职业背景或关系。

## 判断原则

- 身份名优先级：明确姓名 > 稳定称呼/外号 > 稳定关系称谓加身份补充。不要把“我、他、她、男人、女人”直接作为长期资产名。
- “父亲、爹、他”“母亲、娘、她”等明显同一人的称呼必须合并；无法确认是否同一人的提及保留为待解析。
- “童年的沈深、少年的沈深、青年的沈深、成年沈深、老年沈深”这类同名或可确认同一身份的年龄阶段，必须合并为同一个人物资产；不同年龄段记录为同一人物下的年龄阶段/状态资产，不新建多个沈深。
- 人物不只限于人类；有名字、称呼、行动意志或叙事连续性的妖怪、怪兽、动物伙伴、灵兽、拟人角色，都按人物资产处理。
- 同一人物跨服饰、状态、年龄或时间出现时，默认保持同一身份；年龄跨度、物种形态或身份伪装只影响同一人物下的阶段资产命名，并说明与原身份的关系，不拆成多个独立人物。
- 旁观群体、路人、村民、医生、同学等如果没有稳定叙事作用或连续画面需求，通常不新增人物资产。
- 气质不能替代外观。不要只写“漂亮、清冷、强大、阴郁”；要尽量转成可见锚点，例如站姿克制、发型整齐、衣着冷色、眼神少躲闪、肩背挺直。
- 不要默认美化、年轻化或健康化。病弱、贫穷、战损、劳作痕迹、非典型外貌如果是叙事事实，应作为稳定状态保留。

## 视觉锚点分层

- 身份不变项优先级最高：年龄阶段、物种/身份、脸型轮廓、眼距与眉眼比例、鼻形、嘴唇特征、下颌线、发型轮廓、肤色/皮肤质感、身高比例、体态和稳定标志物。
- 基础服饰是最常复用或最能代表身份的服装，优先服务后续三视图和常规视频镜头。
- 服饰变体只记录后续会复用的稳定状态，例如居家服、通勤装、校服、战袍、铠甲、正式服、伪装服、病号服。
- 状态变体写可见表面，不写抽象经历。例如“右手指节粗糙有裂口和血痂”优于“很辛苦”；“战袍边缘焦黑破损”优于“经历大战”。
- 特殊形态要和普通形态挂在同一人物下，例如妖化、灵体、兽形、修仙法相、觉醒形态；只有物种和轮廓完全改变时才强调阶段差异。
- 随身物只记录对人物身份或情感连续有稳定价值的东西；如果物件需要单独特写、交接或跨场景复用，应交给道具抽取。

## 资产线索沉淀

- 抽取阶段必须把后续图生图状态资产需要的线索沉淀到 summary 或 attributeEntries，不直接写图片提示词。
- attributeEntries 不要求固定 key；优先用少量自然语言说明资产需求，例如“需要基础身份三视图”“出现外出状态三视图”“第二章新增受伤战损状态三视图”。
- 每条资产线索都要能回答：唯一身份是谁、这张资产用于什么状态、保持哪些身份锚点不变、只改变什么可见内容、什么时候复用。
- 服饰/状态资产记录居家、睡衣、外出、通勤、校服、战斗、工作、正式、伪装、受伤、病弱、觉醒等可复用状态；没有文本依据时不要硬造。
- 年龄阶段资产记录童年、少年、青年、成年、老年等可复用阶段；同一人物的年龄变化要写清保持的身份血缘/五官延续锚点，以及本阶段只改变年龄、身高比例、脸部成熟度、发型/服饰阶段和体态。
- 面部表情、手部、伤口、面具、法印、纹身、饰物局部等只作为镜头级需求记录在 summary 里，不作为资产库图片长期维护。

## 年龄阶段合并规则

- 同名人物或上下文明确同一人的不同年龄段，永远优先合并到同一个 entity；不要因为年龄变化拆成多个角色。
- 年龄阶段可以生成不同人物状态图，但必须在同一流程任务中归属于同一人物资产，例如“沈深基础青年三视图”“沈深童年阶段三视图”。
- 如果当前文本只出现一个年龄阶段，就只记录当前阶段；不要为完整人生默认补童年/青年/老年。
- 如果后续增量文本新增同一人物的年龄阶段，用 existingEntityUpdates 补充年龄阶段 asset 需求；不要新建人物，也不要重新生成无关基准图。
- 跨年龄图生图要强调身份延续：保持核心脸型家族相似性、眼距/眉眼关系、鼻形、下颌趋势、发色/发际线线索和稳定标志物；只改变年龄成熟度、体型比例、服饰和阶段状态。

## 特写与转折点

- 关键人物可以记录后续 story_to_video 可能要用三视图派生的特写需求，尤其是台词反应、关系变化、身份暴露、立场转折、法力觉醒、濒死、黑化、强压情绪等节点。
- 特写不是资产库图片；它只提示 story_to_video 在具体镜头里用人物三视图或状态三视图做 img2img 派生。
- 情绪必须外化为画面信号：眼眶发红、嘴角抽动、瞳孔焦点变化、呼吸压住、肩膀压低、手指收紧、目光避开或直视。
- 表情变化要和身份锚点分开：可变的是嘴角、眉头、眼眶、瞳孔焦点、泪痕和呼吸；不变的是脸型、眼距、鼻形、下颌线、年龄阶段和稳定发型。
- 常见特写锚点包括眼睛、嘴唇、手指掐诀、手背伤口、喉间血迹、面具裂缝、发簪/戒指/武器握持、纹身、法印发光。

## 工作方法

- 先扫描稳定称呼和叙事作用，再判断是否可复用到已有身份。
- 全文扫描后必须做覆盖审计：逐一核对姓名、称谓、台词说话人、旁白主体、动作主体、被观看对象和关系称谓，确认每个应稳定的人物都被抽取或标为待解析。
- 决定新增前先问：这个人物后续是否需要稳定复用？是否有足够身份依据？是否只是一次性群像或功能性路人？
- 对主角和关键配角，先锁身份不变项，再整理基础服饰、状态资产、特殊形态和镜头级特写线索。
- 对出现多年龄阶段的关键人物，先确认这些阶段是否同一身份，再在同一个人物下整理年龄阶段资产；不要把“童年沈深”和“青年沈深”拆成两个可独立绑定的人物。
- 对外观信息不足但叙事明显重要的人物，保留可靠的年龄阶段、身份关系、气质和服饰层级，不硬造具体五官。
- 对非人稳定身份，明确其物种/形态、轮廓、体型、可动部位、材质质感和叙事身份。
- 不把场景写进人物属性；“客厅里的她”只抽人物状态或服饰，“客厅”交给场景抽取。
- 最终人物说明要短而准：说明是谁、叙事作用、基础外观、基础服饰，以及是否存在重要服饰/状态/特写变体。

## 质量标准

- 人物列表干净、不重复，称呼稳定。
- 不漏人物：稳定出场、说话、被旁白指代或承担镜头主体的人物都能在结果中找到归属；不能只抽主角而漏关键配角或新增状态。
- 每个主角或关键配角至少能回答：是谁、和故事有什么关系、基础外观是什么、最适合做参考图的基础服饰是什么。
- 如果文本出现多套稳定服饰或状态，人物资产能回答：有哪些变体、什么时候使用、哪些差异需要后续图生图保持身份一致。
- 关键人物能支撑“基准人物三视图/主视图 + 状态三视图 + 镜头级特写派生”的图片链路，且身份不因表情、服装或场景切换而漂移。
- 抽取结果不把剧情解释、心理评语或作者评价当作可生图外观。

## 失败回退

- 边界模糊时，宁可少收也不误收一次性人物。
- 信息不足时保留待解析，不用代词硬建长期资产。
- 没有外观依据时，用保守的身份和状态描述，不臆造脸型、发色、服饰和背景。
