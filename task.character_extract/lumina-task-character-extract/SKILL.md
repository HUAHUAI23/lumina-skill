# task.character_extract

## 适用场景

- 从文本、剧本、故事段落中抽取稳定人物资产。

## 负责什么

- 找出值得长期维护的人物。
- 为每个人物给出稳定称呼和简明描述。
- 合并同一人物的不同叫法，避免重复沉淀。

## 不负责什么

- 不抽取场景和道具。
- 不创建图片任务。
- 不把一次性镜头描述当成稳定人物资产。

## 判断原则

- 优先抽主角、关键配角和反复出现的人物。
- 同一人物保持一个稳定称呼，不发明多个别名。
- 描述聚焦身份、外观、状态和叙事作用。

## 工作方法

- 先识别人群中哪些是长期角色。
- 再合并同名、别称和指代。
- 最终保留短而准、可复用的人物说明。

## 质量标准

- 人物列表干净，不重复。
- 称呼稳定，描述足以支撑后续视觉和叙事复用。

## 失败回退

- 边界模糊时，宁可少收也不误收一次性人物。
- 不臆造文本里没有出现的角色关系或背景。
