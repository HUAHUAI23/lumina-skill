# consult.screenplay

## 阶段定位

这是“叙事到剧本”阶段，位于分镜咨询之前。

完整链路是：

```text
叙事/梗概/故事想法
-> consult.screenplay 反复咨询并确认剧本文本（本阶段）
-> 插入单个剧本文本节点
-> 文本节点发送到聊天
-> consult.storyboard 或 consult.storyboard-script-to-shot-prompts 把剧本转为分镜文本并反复确认
-> 插入单个分镜文本节点
-> 文本节点发送到聊天
-> flow.story_to_video.shot_split 或 …-scene-as-shot 把分镜文本结构化为 shot list
-> 下游分批 Seedance 2.0 调用（multi-shot timestamp / I2V / first_last_frame / ref_pack / video_extend / nine_panel_grid_i2v）
-> 后期 consistency review + 调色
```

这个阶段不是把小说句子改成分行对白，也不是按时间顺序复述剧情。你的任务是把文学叙事
转译为可表演、可拍摄、可继续分镜、**且对下游 AI 视频产线友好**的视听剧本。

## 产线感知（写剧本时必须知道下游怎么生）

剧本写法直接影响下游 Seedance 2.0 出片的成本和一致性。三个 Tier 决定剧本结构密度：

| Tier | 总时长 | 场景数 | 主线人物 | 主场景 | 剧本结构提示 |
|---|---|---|---|---|---|
| Tier 1 | ≤ 45s | 1-3 场 | ≤2 | 1 个 | 单线推进，少跳场，每场 ≤15s |
| Tier 2 | 45s-2min | 3-8 场 | ≤4 | ≤3 个 | 主线 + 1 副线，跨场必有声音/动作/视线承接 |
| Tier 3 | 2min+ | 8+ 场 / 多集 | ≤6 主角 + 配角池 | ≤5 个主场景 | 强主角池、强主场景池、章节式分集、跨集回收 |

判定 Tier 的优先级：用户明确指定时长 > 故事自然分量 > 类型默认（短剧 60s、剧情 3-5min、连载式按集）。

剧本写完后，在剧本顶部"开篇说明"里**必须告知用户判定的 Tier 和理由**，因为 Tier 决定下游分镜咨询会推荐 Tier 1/2/3 哪条链路策略，进而决定 Seedance 调用次数和成本预算。

**关键约束**（来自下游产线实测）：

- Seedance 单次输出 ≤ 15 秒：**任何超过 15s 的连续动作必须能拆成 2+ 段**，每段语义自洽。
- 30 秒一致性墙：**总时长超过 30s 的故事，主角必须有跨场识别锚**（同一外貌描述跨场复用，下游建 turnaround sheet）。
- 50-70% usable rate：**关键场不能依赖小概率发挥**（例如复杂群像、多人同框激烈动作、高细节物理交互），剧本里要给出可拆备选方案。
- batch by similarity：**剧本里同景别、同场景、同光线条件的场次集中编排**比"完全按时间线散开"更利于产线（下游能批生成）。剧情要求不允许集中时不要硬凑，但给得自然就给。

## 本轮产物边界

- 给用户看的说明要短，只说明本轮做了什么、主要调整了哪里、还能继续怎么改。
- **必须在用户说明里告知判定的 Tier**（Tier 1 / 2 / 3）和理由，这决定下游分镜咨询的链路策略与产线预算。
- 剧本文本必须是完整正文，可以直接审阅、继续修订或发送到分镜咨询。
- 剧本顶部的"产线提示"meta block（详见 §产线提示 meta block 一节）必须随剧本一起输出，给下游 storyboard 消费。
- 用户要求修改时，合并上下文并输出当前最新版完整剧本，不只输出差异片段。

## 工作顺序

- 先判定用户要的是新写、改写、扩写、压缩、分集、调台词，还是把上一版继续打磨。
- 再抽取故事骨架：主角欲望、阻力、代价、误解、信息差、关键物件、价值翻转和结尾落点。
- 决定叙事顺序：保守顺叙、倒叙钩子、插叙记忆、平行推进或交叉剪辑；选择必须服务悬念、情绪或因果。
- 把心理和说明改成可见行为、空间关系、道具状态、声音变化和对方反应。
- 写场次时先保证每场都有状态变化，再写对白。台词只是表演的一部分，不是剧情说明书。
- 完稿前做一次核销：删掉任何不改变关系、信息、局势或情绪的段落。

## 上游故事设计职责

- 剧本阶段先解决故事，不把问题留给分镜补救。分镜可以强化表达，但不能替剧本发明主线、动机和关键转折。
- 每场戏都要有价值翻转：希望变失望、信任变怀疑、亲密变疏离、安全变危险、弱势变反击。没有翻转的场，只保留能承载必要铺垫或反差的部分。
- 先决定信息结构：观众和角色谁知道真相。优先使用“观众知道、角色不知道”的悬念结构；需要反转时，提前埋可回看的反常细节。
- 矛盾升级至少命中一条：代价变重、障碍变复杂、时间更紧。中段不能多场重复同一种受挫。
- 重要压力尽量做三次推进：第一次暗示，第二次逼迫，第三次爆发或反转。
- 根据题材气质控制密度：爽剧型前置强冲突和高频回报；剧情型保持悬念与人物困境；文学型用物象、空间和沉默推进主题，不硬套爽点。
- 长故事要拆成可确认的场和集，每集都有小目标、小阻力、小翻转和尾钩，不能平均切段。

## 负责什么

- 把叙事、梗概、小说段落、短剧想法、人物关系或用户口述转成完整剧本。
- 保留用户给定的题材气质、人物关系、关键事件、关键台词、结局方向和篇幅要求。
- 先整理故事线，再写场次：主角欲望、阻力、误解、信息差、价值翻转、情绪落点、伏笔回收。
- 补足必要的场次、动作、对白、旁白、停顿、潜台词、场面调度、情节铺垫和情绪转折。
- 允许为戏剧性重排叙事顺序，例如倒叙钩子、插叙记忆、平行推进和交叉剪辑，但必须服务悬念、情绪或因果。
- 按用户指定的集数、每集时长或总时长组织剧本；没有指定时按自然剧情节点组织。
- 让剧本适合下一阶段转分镜：场景、人物、动作、对白、旁白、关键道具、情绪节点、声音和转场动机都要清楚。

## 不负责什么

- 不创建任务。
- 不创建或修改项目资产。
- 不写分镜图提示词、视频提示词、镜头输入模式、模型参数或平台专属语法。
- 不输出逐镜生成参数，但可以写剧本级的画面重点、视线、遮挡、声音桥和转场意图。
- 不把剧本拆成多个画布节点；最终只进入一个文本说明节点。
- 不在用户已经确认的剧情基础上擅自改变核心事件、人物关系、结局和题材气质。

## 剧本组织建议

建议使用稳定、易改、易转分镜的 Markdown：

```text
# 剧本文本

<!-- 产线提示 meta block：写在剧本最顶部，下游 storyboard 消费 -->

## 产线提示

Tier：Tier 2（4-10 段，约 90 秒，3 个主场景）
主线人物：林照月（女主）/ 母亲（病危状态）/ 林照月之父（仅出现于回忆）
主场景：病房 / 走廊 / 旧居客厅（回忆）
核心物象：玉佩（3 阶段：完整 → 被攥皱 → 被剪碎合照覆盖）
跨段承接计划：
  - 场 1→2：sound_bridge（监护仪声延伸）
  - 场 2→3：graphic_match（玉佩特写 → 旧合照特写，圆形构图呼应）
  - 场 3→4：l_cut（旧居钢琴音延续 1.5s 进病房）
灯光基线：病房 5600K 冷白荧光顶光 + 监护仪冷蓝 6500K；旧居 2700K 钨丝暖光 + 黄昏窗光
建议下游产线：批生成（病房 close-up 一批 / 旧居 wide 一批）+ 每镜 2 变体 + 后期统一冷调

## 第 1 集：标题（约 60 秒）

### 场 1：地点 / 时间

人物：（写出场角色全名，命名一旦定稿不要换说法——下游靠这些名字建 @char_main_<名> 槽位）
场景目标：
信息变化：
情绪曲线：
场景氛围：（明暗交界线位置 + 冷暖色温分布，便于下游锁 HDVP 光影深度墙）
暗线/铺垫：
铺垫/回收：
对上一场承接：（衔接类型 + 共享 token / 道具 / 声音；第 1 场可省）

【画面叙事】
画面先让观众看见什么、隐藏什么、误判什么；关键道具、空间关系、视觉焦点。

【动作与调度】
角色动作、站位、视线、距离、遮挡、可见证据、关键停顿。
高风险位移写明屏幕方向（向画面右、向深处、靠近镜头）便于下游锁 HDVP 矢量。

【对白】
角色A（语气 / 潜台词）：台词。
角色B（语气 / 潜台词）：台词。

【旁白】
只写画面无法承担、或与画面形成反差的信息。

【声音与转场】
环境音、音效、沉默、J-Cut/L-Cut、动作承接、匹配剪辑或悬念尾钩。
跨场承接节点（sound_bridge / match_on_action / eyeline_match / graphic_match / l_cut / j_cut）

```

不是每场都必须填满所有栏目，但关键场必须有场景目标、信息变化、画面动作、对白/旁白、
情绪承接和转场。**剧本顶部的"产线提示"meta block 是 Tier 2/3 必填**——它是下游 storyboard
建参考池、续写跨镜手册、连续性账本的种子文本。删掉一场不会影响故事和情绪时，应合并或删除。

## 叙事到剧本的方法

- 先找全段主线：谁想要什么、谁阻止、代价是什么、观众先知道什么、角色暂时不知道什么。
- 每场都是“变化单元”，开始和结束必须有状态差：关系变近或变远，秘密暴露或被遮蔽，信任建立或崩塌，局势升级或反转。
- 不按原文顺序机械搬运。可以把最有冲击力的视觉结果前置为钩子，再回到原因；可以用插叙让当下动作刺穿回忆；可以平行推进两条线制造对照。
- 铺垫必须落到可见证据：道具、伤口、习惯动作、环境异常、一句被误解的台词、一个被回避的眼神。
- 关键物件要有状态变化：初次出现、压力中被使用或握紧、最终碎裂/归还/消失/暴露。不要只在台词里说它重要。
- 把解释性叙事转成可表演动作：看见、触碰、迟疑、沉默、躲开、逼近、递出、收回、攥紧、松手、回头。
- 把抽象情绪转成物理表现：呼吸停住、手指发抖、视线避开、笑意消失、话没说完、衣角被攥皱、脚步停在门槛外。
- 把背景信息藏进冲突、道具、反应和误判里，不用大段说明直接灌给观众。
- 文学型叙事可以保留旁白，但旁白必须和画面动作互补或反讽，不能重复解释画面已经表达的信息。

## 铺垫、回收与核心物象

- 铺垫必须可见、可听或可被误解：一道伤口、一句口误、一个空位、一张旧照、一次回避、一个被反复触碰的物件。
- 核心信物或主题物象优先三次出现：第一次建立日常状态，第二次承受压力或状态变化，第三次回收、毁坏、归还、消失或转化。
- 回收要改变观众对前文的理解；只是重复出现不是回收。
- 结尾尽量呼应前文的动作、物件、台词或空间，让主题落到画面，而不是落到解释性旁白。

## 场景、氛围与暗线

- 场景要参与叙事，不只是放人物说话。每个关键场景都应承担关系压力、身份差异、信息隐藏、主题意象或行动阻碍。
- 氛围必须物理化：冷白灯、潮湿墙、雨声、门外脚步、空座位、长桌距离、机器低鸣、窗外红光、背景人群停顿等。
- 不写“气氛很压抑”这种空话；要写谁被什么压住、观众从哪里看出压迫。
- 暗线要提前埋入可见证据：反复出现的道具、被回避的称呼、异样的伤口、背景里的同一个人、总被打断的话、异常声音。
- 暗线不要一次讲明。第一次像普通细节，第二次与压力关联，第三次改变观众对前文的理解。
- 穿插叙事必须有触发点。插叙、闪回、平行线或倒叙不是装饰，它们要来自当下动作、道具、声音、空间相似或情绪撞击。
- 平行叙事要形成对照：一条线在误判，另一条线给观众真相；一条线安静，另一条线危险；一条线逃避，另一条线逼近。

## 微型转译示例

不要这样写：

```text
母亲已经不爱她了。林照月很绝望，她道歉失败后离开。
```

应该转成：

```text
病房里，母亲始终背对床边。林照月把玉佩放到病危通知书上，手指停了很久。
枕头边露出一角被剪碎的合照，林照月没有看见。
她低声说“我以后不烦你了”，母亲没有回答，只把被角攥出一道深褶。
林照月走到门口又停住，最终没有回头。走廊灯闪了一下，病房里只剩监护仪声。
```

这个示例的重点不是固定情节，而是转译方式：抽象关系变成背对、遮挡、道具、反应和声音。

## 画面叙事

- 画面优先于解释。不要写“他很痛苦”“她很害怕”，要写观众能看见的行为、空间压迫、道具状态和反应。
- 设计信息差：让观众在前景、中景或背景里看到角色没看到的证据、危险或关系裂缝。
- 关键冲突不要只靠台词宣告。用距离、遮挡、背对、空位、门缝、病床边缘、桌面裂痕、灯光变化等可见细节承担情绪。
- 重大信息之后必须有反应时间：动作本体、接收反应、环境或关系后果。不要让角色立刻赶下一句台词。
- 秘密揭露、死亡、背叛、耳光、目击、认亲、告白、分手、处刑、反转等高强度事件，至少拆成“发生、反应、后果”三层，不要一句话带过。
- 空镜可以存在，但必须服务转场、情绪、伏笔、环境压力或后果，不能用来凑篇幅。
- 强物理交互要用可生成、可拍的片段替代：搏斗可拆成脚步、泥水、剪影、破碎眼镜；拥抱亲吻可拆成攥紧衣料、伞落水洼、脚步靠近；暴力穿刺可拆成刀光、血滴、静音反应。

## 镜头语言意识

- 这是剧本，不是分镜，但写作时必须有镜头意识：什么该给特写，什么该给远景，什么该留在画外，什么该用声音提前引入。
- 情绪优先，其次故事、节奏、视线、构图和空间连续。不要为了华丽描写牺牲因果和人物动机。
- 关系戏要写清站位、距离和视线：谁靠近，谁后退，谁看着对方，谁看向画外或避开。
- 证据、信物、伤口、遗书、药瓶、裂纹、手部动作等关键视觉锚点不要交给远景；要在剧本里提示近看或被角色/观众注意到。
- 悬念场可以使用前中后景的信息不对称：角色在前景做日常动作，真相或威胁在背景边缘逐渐显露。
- 场景切换要有承接：声音桥、动作匹配、形状匹配、视线切换、物件接续或一句台词压到下一场。
- 高风险位移（开门进入/穿过走廊/从一区到另一区）在【动作与调度】里写明屏幕方向和深度变化（向画面右、向深处、靠近镜头），让下游分镜知道需要锁 HDVP 矢量。
- 跨光影区切换（暗→亮、冷→暖、室外→室内）在【场景氛围】里显式写出冷暖区和明暗交界线位置，便于下游锁 HDVP 光影深度墙。

## 下游可拆性（写给 storyboard / story_to_video / Seedance 友好）

剧本不写镜头号，但要让下游能稳定切出 micro-shot 级原子分镜，并且能直接消费产线提示生成参考池。每场戏写完后回看：

### 原子化尺子

- 关键动作链是否能自然切成 2-5 段？例如"进门 → 抛媚眼 → 抱胸前倾说话 → 食指点唇"是 4 段，不是 1 段。
- 跨动作转折（玩笑变恐惧、温柔变冷酷、放松变警觉）是否给出可见信号（瞳孔、嘴角、呼吸停顿、手指停住），让 storyboard 能把它放进单独的揭示节拍？
- 每条对白是否标明该段是说话人为主体（适合中近景或特写），还是听者反应优先（适合切反打）？
- 缺席角色（门后伪人/楼上声音/电话那头）是否标明声源方向和被听者反应？让下游用 sfx 暗示而非塞进画内动作。
- 高风险物理动作（碰撞/扑倒/压墙）是否拆成"预备 → 爆发 → 余震"三段，并写出可见反馈（发丝/纸张/光影摆动），让下游能映射到 beatRole / physicalHint？
- **任何"持续动作"段落是否能压进 ≤15s 的连续电影镜头**？超过 15s 的连续动作必须能拆成 2 段，每段语义自洽（Seedance 单次生成硬上限）。
- **角色入场方式可见**：剧本里描述一个角色进入场景时是否写明"从哪里来 / 起始位置 / 速度"（"林照月推门走进，左手仍握门把" / "母亲从画面深处的病床慢慢转头"）？Seedance 2.0 对"主体延迟出现"非常敏感，下游 storyboard 需要把角色锚定在镜头首句；剧本如果只写"林照月开始道歉"而不写她在画面哪一侧，下游就只能猜或让模型自己决定，常导致"人物突然闪现 / 拍间融合"。**避免**"突然出现 X / 忽然闪现 X / 画面里多了 X" 这种被动揭示语。

### 参考池槽位 seed（Tier 2/3 必看）

剧本里的人物 / 场景 / 道具命名直接决定下游能不能稳定建参考池。约定：

- **主线人物**全程使用同一个全名指代（"林照月"不要中途换"小月"或纯"她"）。下游建 `@char_main_林照月` / `@char_main_lin_zhaoyue` 槽位，命名一致才能跨场景 token 重复。
- **主场景**有稳定名字（"病房"全文不要换"医院 305 室"或"她躺的那间屋"）。下游建 `@scene_病房` / `@scene_bingfang`。
- **核心物象**有稳定名字 + 状态机（"玉佩"全文不要换"那块绿色挂坠"）。下游建 `@prop_玉佩` + 写到连续性账本"道具状态机"。
- 同一个角色 / 场景 / 物象一旦改名，下游产生 2 个槽位、token 不重复、跨镜身份失锁。**剧本层是最廉价的命名一致性防线**。

### batch by similarity 友好的场次组织

下游产线按"相似度分批"生成（同景别一批、同场景一批、同光线一批）效果远好于"按时间线散开生成"。剧本写法可以不影响剧情顺序的前提下：

- **场次相邻聚类时，优先把同场景的相邻场放在一起**（除非剧情要求平行剪辑）。
- **回忆 / 闪回段集中在 1-2 个区块**而不是散落全片，让下游能集中批生成（同光线、同场景、同主体）。
- **平行剪辑两条线**：每条线连续给 2-3 场再切，比每场都跳更利于下游产线（且观众更易跟得上）。
- 如果剧情要求严格交替（每场必须切线，例如悬疑追逐），不强求集中，但要在产线提示里 flag 给下游：本片不友好 batch，下游需要走标准 ref pack + 多变体策略。

### 跨场承接节点（写在剧本里，下游直接消费）

剧本【声音与转场】栏要显式写出跨场衔接类型和共享 token，不要留给下游 storyboard 现想。允许的衔接类型：

- `sound_bridge`：上一场末段声音延伸进下一场画面
- `j_cut`：下一场声音提前进入上一场末段
- `l_cut`：上一场声音覆盖下一场首段
- `match_on_action`：上一场动作未完成，下一场从另一角度续上
- `eyeline_match`：上一场角色看 X，下一场直接给 X
- `graphic_match`：形状 / 构图 / 物件呼应
- `concept_cut`：主题 / 反讽 / 对比（克制使用，全片 ≤2 处）
- `hard_cut`：直接切（默认，不写也是 hard_cut）

写法示例（在【声音与转场】栏）：
```
跨场承接：sound_bridge（监护仪 beep 延伸 1.2s 进下一场走廊）
共享 token：监护仪声 / 林照月
```

不强制改写剧本格式，但写作时心里有这把"原子化尺子 + 命名一致性 + batch 友好 + 跨场承接"四把工具，下游 storyboard 和 Seedance 才不会把整段戏砸成一段散文 prompt 或一片漂移连环画。

## 台词与潜台词

- 对白要能表演，不要替作者解释剧情。
- 每句台词都要有功能：推进剧情、暴露关系、揭示信息、制造误会、施压、撒谎、掩饰、试探、转场或完成情绪翻转。
- 角色说出口的内容可以和真实意图相反。用动作和反应暴露潜台词，例如微笑时攥紧手帕、道歉时不敢抬头、答应时后退半步。
- 重要对白后给反应：沉默、眼神、动作、呼吸、答非所问、打断、转身或环境声压过台词。
- 原文中的反问、誓言、告白、判决、辱骂、承诺、主题句优先保留，但要放在有压力和反应的位置。
- 可以补桥接台词，但要让角色语气、身份、关系和此刻利益成立。

## 声源类型

- `对白` 是角色在画面内或场内开口说出的话，必须能被表演动作、口型、视线或听者反应承接。
- `旁白 / VO` 用于跨时间、记忆组织、主题反差或画面无法直接表达的信息；旁白应配空镜、背影、物件证据、回忆画面或慢推凝视，不要覆盖无关动作。
- `画外音` 是场内或邻近空间传来的声音，例如门外脚步后的呼喊、电话里的人声、走廊传来的广播；要写清声源方向或空间关系。
- `内心 OS` 是角色未开口的心理独白，不能写成角色嘴上说出来；画面要用凝视、停顿、手部动作、呼吸或反应支撑内心信息。
- 同一段不要让对白、旁白、画外音和内心 OS 互相抢信息；优先让画面承担情绪，声音只补画面无法承担或需要反差的信息。

## 台词容量

- 普通中文对白/旁白：约 4-5 字/秒。
- 重情绪、哭腔、病弱、迟疑：约 3-4 字/秒。
- 争吵、抢白、信息流快读：最多 5-6 字/秒。
- 极快 6-7 字/秒只用于特殊风格，慎用。
- 人物台词负责当下目标、关系博弈和潜台词，不负责替作者解释设定。
- 旁白负责跨时间、记忆、主题反差和画面无法直接表达的信息；旁白不能重复解释画面已经表达的情绪。
- 粗算可说字数时，要先扣掉动作、停顿、反应和余韵。4 秒普通对白通常只适合 12-18 个汉字；6 秒普通对白约 20-28 个汉字；重情绪更少。
- 一场里如果台词太密，要加动作段、沉默、反应或拆场，不要让角色连续解释。

## 节奏与结构

- 平稳段可以给完整动作和少量对白，让观众理解关系和处境。
- 冲突爆发段要压缩句子、加快动作、插入短反应和后果，不要用长段解释稀释爆点。
- 顿悟、创伤、告别、失败和重大决定要留停顿，让沉默、手部动作、环境音或空白承担余震。
- 长叙事不要平均切段；按事件、情绪翻转、信息揭示、伏笔回收和尾钩切场或切集。
- 每集要有开场钩子、基本冲突、情绪爆点或信息揭示、结尾回报或尾钩。
- 短剧可以提高误会、反转和情绪回报密度，但不要硬套爽文公式改坏题材气质。

## 分集与时长

- 用户给“1 分钟一集”时，每集约 60 秒，并保持独立小结构。
- 用户给总时长时，先判断能承载多少场和多少台词，不够时主动压缩支线或合并场次。
- 用户没有给总时长时，优先按自然剧情段落组织，并在用户说明里交代建议集数和依据。
- 时长有限时保留主线、关键反转、核心台词和视觉伏笔，删掉只解释设定、不改变局势的段落。

## 产线提示 meta block（Tier 2/3 必填）

剧本顶部的产线提示是写给下游 storyboard 看的"产线种子"，让 storyboard 不必从剧本正文反推产线决策。包含七项：

1. **Tier**：写明 Tier 1 / 2 / 3 + 一句话理由（总时长 / 场数 / 主角数）。下游据此选 Tier 1/2/3 链路策略。
2. **主线人物**：列出全名 + 一句话身份提示（角色定位 / 关键状态）。命名一旦定稿不要换说法（"林照月"全程不能再叫"小月"或"她"作为主指代）；下游 storyboard 直接复用这些名字建 `@char_main_<名>` 参考池槽位。
3. **主场景**：列出全部主场景名 + 一句话空间/光线提示。下游建 `@scene_<名>` 槽位。
4. **核心物象**：列出贯穿物象（玉佩 / 戒指 / 旧合照 / 红衣 / 火柴等），写明 3 阶段状态机（出现 → 受压 → 毁坏/归还/消失/转化）。下游写到连续性账本的"道具状态机"。
5. **跨段承接计划**：列出关键场之间的衔接节点（sound_bridge / match_on_action / eyeline_match / graphic_match / l_cut / j_cut / concept_cut）。下游 storyboard 不必现想衔接，直接消费这套预案。
6. **灯光基线**：列出每个主场景的命名光场（色温 + 灯型 + 位置），例如"病房 5600K 冷白荧光顶光 + 监护仪冷蓝 6500K"。下游写到连续性账本的"光线连续矩阵"。
7. **建议下游产线**：一句话告知 batch by similarity 分组建议（按景别 / 场景 / 光线分批）+ 每镜变体数（推荐 2-3）+ 后期调色基线（统一冷 / 暖 / 中性）。

格式示例（写在剧本最顶部 `# 剧本文本` 之后）：

```text
## 产线提示

Tier：Tier 3（10+ 段，约 4 分钟，5 个主场景，多集结构）
主线人物：
  - 林照月（女主，留学回国 / 母亲病危）
  - 母亲（病危状态，全程卧床）
  - 林父（仅在回忆出现，西装 / 戴眼镜）
主场景：
  - 病房（5600K 冷白荧光 + 监护仪冷蓝 6500K，明暗交界线沿病床纵轴）
  - 走廊（节能灯惨白 + 急救灯红橙 1Hz 频闪）
  - 旧居客厅（2700K 钨丝暖光 + 黄昏窗光，夕阳侧逆光）
  - 雨夜街道（钠灯橙 2200K + 霓虹粉，雨水反光）
  - 电话亭内部（应急灯绿，狭窄）
核心物象：
  - 玉佩（完整→被攥皱→被剪碎合照覆盖→最终归还）
  - 旧合照（完整→被剪→被烧角→被焚毁）
  - 病危通知书（递出→被攥皱→被压在玉佩下）
跨段承接计划：
  - 第 1 集 场 1→2：sound_bridge（监护仪声延伸进走廊）
  - 第 1 集 场 2→3：graphic_match（玉佩特写圆形 → 旧合照特写圆形）
  - 第 1 集 场 3→4：l_cut（旧居钢琴音延续 1.5s 进病房）
  - 第 2 集 场 1→2：match_on_action（林照月推门动作未完）
  - 第 2 集 场 4→5：concept_cut（病房白光 → 雨夜街道霓虹，主题对比）
灯光基线：详见各主场景注释；全片冷调主基线，回忆段切暖。
建议下游产线：
  - 批生成：病房 close-up 一批 / 走廊 wide 一批 / 旧居所有镜头一批
  - 每镜变体数：2-3（关键反转镜 4-5）
  - 后期调色：统一冷调（5600K 基线），回忆段单独走 2700K 暖调
```

**Tier 1 短片段（≤45s）允许省略产线提示**——下游能从剧本正文推断。
**Tier 2/3 必须有产线提示**——剧本越长，下游越需要这套种子来组织产线。

## 质量标准

### 故事质量
- 剧本文本能直接给用户审阅和修改。
- 故事线清楚，有铺垫、有升级、有翻转、有回收，不是流水账。
- 每场都有冲突、变化、信息价值或情绪价值。
- 画面能承担叙事，台词能承担关系和潜台词，旁白不替画面重复解释。
- 对白不超载，留有动作、沉默和反应。
- 角色语气、关系、动机和行为一致。
- 保留用户原始叙事的核心情绪和题材气质。

### 下游可消费性
- 剧本能自然进入分镜阶段，不需要再猜谁在哪里、说了什么、发生了什么、为什么发生。
- 主线人物 / 主场景 / 核心物象命名全文一致，下游可以直接复用建 `@char_main_*` / `@scene_*` / `@prop_*` 参考池槽位。
- Tier 2/3 剧本顶部有产线提示 meta block，下游 storyboard 可以直接消费而不是从剧本正文反推。
- 跨场承接节点在【声音与转场】栏显式标注（衔接类型 + 共享 token），下游不必现想。
- 任何持续动作 ≤15s（Seedance 单次生成上限），超过的能自然拆成 2+ 段。
- 高风险位移和跨光影区切换在【动作与调度】/【场景氛围】栏写明矢量方向 + 明暗边界 + 命名光场。

### 产线友好性（Tier 2/3）
- 用户说明里告知判定 Tier 和理由。
- 总时长 >30s 时主角有跨场识别锚（提示用户准备 turnaround sheet）。
- 场次组织在不影响剧情顺序的前提下尽量利于 batch by similarity（同场景集中、同光线集中）；不利于 batch 时已 flag。
- concept_cut 全片 ≤2 处。

## 最低核销清单

完成前逐项检查：

### 故事层
- 删掉任意一场，是否会损失关系变化、信息揭示、冲突升级、伏笔回收或情绪余震？如果不会，合并或删除。
- 观众不听旁白，只看动作、空间、道具和反应，是否仍能理解关键关系和冲突？
- 每个高强度事件是否都有发生、反应、后果，而不是直接切到下一件事？
- 关键物件是否至少有植入、加压或回收中的两个阶段；若是核心信物，优先完成三阶段变化。
- 台词是否由当下压力触发；删掉解释性台词后，画面是否还能承担信息？

### 镜头层
- 场景之间是否有声音、动作、视线、物件或形状承接，避免拼贴感？跨场承接类型是否在【声音与转场】栏显式标注？
- 高风险位移和跨光影区切换是否在【动作与调度】/【场景氛围】里写明矢量方向和明暗边界，便于下游锁 HDVP？
- 关键动作链是否能自然切成 2-5 段 micro-shot；情绪转折是否给出可见信号让下游能放进揭示节拍？
- **任何持续动作 ≤15s**？超过的能否自然拆成 2+ 段（Seedance 单次生成硬上限）？

### 产线层（Tier 2/3 必查）
- 是否在用户说明里**告知判定的 Tier 和理由**？
- 剧本顶部是否有**产线提示 meta block**（Tier / 主线人物 / 主场景 / 核心物象 / 跨段承接 / 灯光基线 / 建议产线）？
- **主线人物全名是否全文一致**（不混用昵称、代词作主指代）？下游能否直接复用建 `@char_main_<名>` 槽位？
- **主场景名字是否全文一致**？下游能否直接建 `@scene_<名>` 槽位？
- **核心物象是否有稳定名字 + 3 阶段状态机**？下游能否直接写到连续性账本"道具状态机"？
- 总时长 >30s 时，是否提示用户"主角必须有跨场识别锚（下游建 turnaround sheet）"？
- 总时长 >2min 时，是否在产线提示里给出 batch by similarity 分组建议？
- **concept_cut** 全片 ≤2 处？

## 失败回退

- 信息不足时，先给可执行的粗剧本，并在用户说明里明确缺少哪些关键设定。
- 用户只给一句想法时，先生成短剧本雏形，不假装有完整世界观；这种情况通常是 Tier 1，先给单段试拍剧本而不是强行拉成 Tier 2。
- 因果不清时，优先补人物目标、信息差、视觉证据和反应，不用解释性对白硬圆。
- 时长明显不够时，标注预算紧张，给压缩版结构，不硬塞全部剧情。
- **判定 Tier 困难时**（例如用户给的故事自然分量与时长要求严重不匹配），优先按用户指定时长走，把故事压缩 / 拆集而不是超时长；并在用户说明里 flag「故事自然分量为 Tier 3 但用户限定 60s（Tier 2），已压缩 X 场支线，下游产线按 Tier 2 走」。
- **跨场承接想不出来时**（剧情切到平行宇宙、梦境、抽象主题），允许标 `concept_cut` 但全片 ≤2 处；超过 2 处说明叙事散架，应回到故事层重审。
- **核心物象只有 1 阶段**（出现完就再没出现）通常是剧本的"挂在墙上的枪没开火"，回到铺垫层补加压或回收阶段；实在补不出就降级为普通道具，不写进核心物象。
- **总时长 >2 分钟但用户没提供主角参考资产意向**时，在用户说明里明确告知「Tier 3 长叙事的硬底线是 turnaround sheet（角色三视图同光同服），需要先到外部图像模型出图后回来，否则下游跨镜一致性会显著下降」。
