# consult.video_prompt

## 适用场景

- 用户想写、改、优化视频提示词，但还不准备直接发任务。

## 负责什么

- 把视频想法整理成可直接使用的提示词建议。
- 补足主体动作、机位、镜头运动、节奏和场景变化。
- 在必要时给出简洁的镜头拆解建议。
- 对用户提供的多个提示词做优劣评估，指出哪些适合直接投喂，哪些更像导演案、shot list、素材说明或内部分析。
- 说明优化方向时要落到结构、时长、动作密度、参考素材、声音和稳定性约束，不只说“更精简”。

## 不负责什么

- 不创建视频任务。
- 不读取项目维护资产。
- 不把咨询结果冒充正式流程规划。

## 判断原则

- 视频提示词优先保证动作和镜头语言清楚。
- 多镜头需求要保持先后顺序和段落目标。
- 少用空泛形容词，多给真实可执行的信息。
- Seedance 2.0 类提示词优先按"固定参数 + 锚点目录 + 自然节拍分镜（SHOT N）+ 声音设计 + 稳定性约束"判断；评估时警惕单段散文导致的信息过载。
- 好提示词不是越详细越好；直接给模型的版本要删除叙事目的、关键确认、焦段光圈、观众理解、黄金三秒等内部导演表格字段。
- 参考图说明分两层：图片生成提示词用于生成素材，视频提示词只在锚点目录列一次；每个 SHOT 只引用本拍需要的子集，不全镜头复用全图集合。
- 高风险镜头（位移/碰撞/穿行/跨光影）按 HDVP 三层判断：空间锚点是否固定、运动矢量是否清楚、物理反馈是否可见。
- **主体前置（Subject-First）**：每个 SHOT 的画面描述必须以"镜头 + 主要人物/主体"开头，再写场景/光线/陈设；不允许"先环境后人物"的写法，否则模型会把人物当作画面延迟揭示的元素，出现"人物突然闪现"或与场景割裂的硬切感。详见 §主体前置规则。
- **景别精度（Shot-Size Precision）**：每个 SHOT 的 `镜头` 行必须用 景别词典（大全景/远景/全景/中景/中近景/近景/特写/大特写/反应镜头/插入镜头/主观镜头）+ 单一主运镜 + 节奏副词写完；禁止焦距、光圈、景深、85mm/f/1.4 这种技术 jargon（Seedance 不响应甚至会 degrade）。需要"压缩感/空间感"用自然语言写：广角空间感 / 自然视角 / 人像压缩感 / 微距质感；需要画面占比就直接写"主体占画面 60%""脸占 40%""前景 1/3 + 主体中景"。详见 §景别与镜头质感规则。

## 主体前置规则（Subject-First）

Seedance 2.0 的注意力分配高度依赖"先出现 = 先建立"。如果一个画面里的关键人物在描述中被放到环境之后，模型常会把人物当作"画面后段才出现的元素"，导致两类典型崩坏：

1. **人物突然闪现**：环境先就位后，人物在中间几帧被"塞进"画面，呈现拼贴感或硬切。
2. **画面融合错位**：上一拍残留的视觉记忆和当前拍的环境先建立，主体最后才出现并和上一拍主体融合。

经验对比（用户实测）：

```text
# 反例 A（环境前置，人物易闪现）
明亮的森林小木屋客厅已经完全恢复整洁，阳光铺在木地板上，收纳篮安稳放好，
镜头平视缓慢推近到站在中央的嘟嘟与围绕家人。

# 正例 B（主体前置，人物稳定建立）
镜头平视缓慢推近到站在中央的嘟嘟与围绕家人。明亮的森林小木屋客厅已经完全
恢复整洁，阳光铺在木地板上，收纳篮装满了安稳放好的玩具。
```

可执行写法：

- **每个 SHOT 的 visualBeat / 画面描述第一句必须先写"镜头 + 主体所在位置 + 主体动作"**，再补环境/光线/陈设/道具。
- **carry-over 角色（上一拍已经在场的角色）**：本拍开头仍要显式提到 ta（位置 + 状态），不要假设模型记得，例：「朱熹仍立于院心暖光区，一名学子从画面左侧入镜跑近」，不要写「一名学子从画面左侧跑近，前方朱熹的目光看住他」（前方朱熹是"延迟揭示"，易闪现）。
- **新进入角色**：必须写明入画方向 + 起始位置 + 速度，例：「X 从画面左侧阴影区入画走近 / 入镜」；不要用「突然出现」「忽然闪现」「画面里多了 X」这类被动揭示语。
- **同框多角色**：一句话内先把所有角色按"主→次"顺序列完，再统一接环境，例：「嘟嘟立画面中央，妈妈嘟米在左、爸爸嘟杉在右、栗栗与岩土爷爷分列后景，明亮客厅木地板被阳光铺亮」。
- **画外角色 / 缺席声音**：不要写进 visualBeat 的主体动作；用 audioDirectives.sfx + 当前主体反应承载。
- **镜头自己"发现"角色** 是反模式：禁止「镜头推近后揭示出 X 站在中央」「镜头摇过最终落在 X 身上」这类把主体当作"惊喜揭示"的写法；改成「X 站在中央，镜头平视缓慢推近」。

评估提示词时按这个清单查：

- 第一句是不是「环境 + 阳光 + 陈设」？是 → flag「主体前置缺失，易出现人物闪现」。
- carry-over 角色是不是从第二句才出现？是 → flag「carry-over 角色被延迟揭示」。
- 新进角色有没有写入画方向？没有 → flag「角色入画方向缺失，可能位置漂移或闪现」。
- 是否使用「揭示出/最终出现/原来站着」这类被动揭示词？是 → 改写为主体前置 + 客观状态描述。

## 景别与镜头质感规则（Shot-Size Precision）

Seedance 2.0 官方文档点名焦距/光圈/景深/85mm/f/1.4 等技术 jargon 会让模型 degrade。需要镜头质感请改写自然短语：

| 你想表达 | 不要写 | 改写为 |
|---|---|---|
| 大场面、广角 | 24mm wide | 大全景 / 广角空间感 / 视野开阔 / 主体占画面 1/4 |
| 叙事中景 | 35mm cinematic | 中景 / 自然视角 / 主体占画面 1/2 |
| 自然视角 | 50mm normal | 中近景 / 自然视角 / 不变形 |
| 亲密人像 | 85mm portrait | 近景 / 人像压缩感 / 背景柔化 / 脸占画面 40% |
| 物证细节 | 100mm macro / f/2.8 浅景深 | 特写 / 微距质感 / 主体占画面 60% / 背景虚化 |

景别词典（择一）：`大全景 / 全景 / 远景 / 中景 / 中近景 / 近景 / 特写 / 大特写 / 反应镜头 / 插入镜头 / 主观镜头`

每个 SHOT 的 `镜头` 行写法：`<景别> + <一个主运镜> + <节奏副词>`，例：

- ✅ `中近景，缓慢推镜 slow，主参考 @图片2`
- ✅ `特写固定，主体占画面 60%，背景虚化`
- ✅ `大全景轻横移 smooth，主体占画面 1/4，前景留草坪 1/3`
- ❌ `medium close-up 85mm 浅景深 f/1.4`（焦距 jargon，Seedance degrade）
- ❌ `先推近再环绕`（单镜两运镜，模糊运镜）
- ❌ `fast push-in`（fast 关键词官方点名 degrade，改 `quick whip pan / sudden push-in`）

景别变化要分到不同 SHOT 或不同时间戳，不要一个时间段内写"从全景推到特写"。需要内部景别变化时写成：`[00:00-00:02] 中景固定` → `[00:02-00:05] 推镜 slow 到近景，主体占画面 50%`。

## 工作方法

- 先锁定主体、场景和动作主线。
- 再补机位、镜头运动、节奏和结尾状态。
- 最终给出一版可直接使用的提示词，必要时附简短镜头建议。
- 如果用户问“哪些好哪些不好”，按以下维度评估：时间轴是否清楚、每段是否一个镜头重心、主体和参考素材是否明确、动作是否可连续生成、声音是否去重并对应画面、约束是否短而有效。
- 如果用户问“怎么优化”，先给原则，再给可套用模板或局部改写；不要只输出抽象建议。
- 对 4-8 秒视频，建议 2-3 个时间段；对 9-12 秒视频，建议 2-3 个时间段；对 13-15 秒视频，建议 3-4 个时间段。超出时优先删减动作或拆镜。
- 动作戏允许奔跑、追逐和命中，但要减少同一时间段里的并行动作；文戏优先用停顿、视线、呼吸、手部动作承接台词。
- 遇到成品 prompt 评估时，按顺序检查：参考素材是否重复、是否写了“按图片顺序读取参考”、长概述是否重复时间轴、画外音说明是否被放进引号台词、声音是否重复、素材数量是否职责重叠、动作密度是否超出时长。

## 优劣判断速查

- 值得学习：SHOT 分块自然、每拍镜头重心明确、**首句主体前置**、动作之间有过渡、台词有口型或反应锚点、声音有事件来源、每个 SHOT 子集化引用参考图、景别用 大全景/中景/近景/特写 等标准词。
- 需要压缩：单段散文 + 时间段堆叠、**首句环境前置**、栏目太多、解释太多、每段重复素材职责、声音重复、同一秒内动作过多、情绪只写抽象词、特写拍引用全部参考图。
- 不建议直接投喂：完整导演案、专业 shot list、摄影参数表（焦距/光圈/景深）、图片生成素材清单、剧情梗概或台词原文堆叠、长篇"画面与动作"自然段。
- 优化时保留：SHOT 编号、时长、自然镜头提示、主体、自然镜头句、可见情绪信号、本拍激活的参考图子集、关键台词位置、声音事件、简短稳定性约束。
- 优化时删除：叙事目的、关键确认、焦段光圈、镜头型号、观众心理解释、重复 BGM/音效、和画面无关的形容词、自然段连接词（"随后/紧接着/最后"）、被动揭示语（"揭示出/最终落在/原来是"）。

## 自然节拍分镜（推荐输出形态）

把单段散文 prompt 升级为 `SHOT N / 起–止s` 的自然节拍格式：

- 头部：`画风 | 时长 | 比例 | 画面大小` 一行；`锚点：- @图片N = 类型:身份` 列表。
- `总览：≤60 字` 写"从哪动到哪、情绪落到哪"。
- 每个 SHOT 内先写一句自然镜头句，再按需补 `镜头`、`参考`、可见信号、空间约束或声音事件；景别和运镜只是 `镜头` 行里的自然提示，不是枚举表头。
- 时间切片服从镜头节奏：≤6s 通常 1-3 段；7-10s 通常 3-4 段；11-15s 通常 4-5 段。
- 高风险位移补 SDV 句式：`空间：<起点锚点> → <X+/X-/Y+/Y-/Z+/Z-> → <终点锚点>`；也可以直接写中文方向。
- 约束行收尾：`约束：人物身份/服装一致 / 禁止穿过不透明物体 / 明暗交界线不漂移 / 无字幕水印`。

## 常见问题改法

- 参考图顺序误读：删除"按 `@图片1` 至 `@图片N` 顺序读取参考"；改为"图片编号只代表参考职责，每个 SHOT 子集化引用"。
- 参考职责重复：合并到顶部"锚点目录"，每个 SHOT 仅写 `锚点：@图片N + @图片M（主：@图片N）`。
- 单段散文 + 时间段堆叠：拆成 `SHOT 1/2/3...` 多镜头格式，每段先写一句自然镜头句。
- **环境前置导致人物闪现**：把首句的"明亮客厅 + 阳光 + 收纳篮 + 镜头推近到 X"调头改为"镜头推近 X 站在中央 + 后接客厅 + 阳光 + 收纳篮"。详见 §主体前置规则。
- **人物被"揭示"**：「镜头推近后揭示出 X」「最终落在 X 身上」这种被动揭示句改成「X 立画面中央/X 从画面左侧入镜，镜头缓慢推近」。
- **carry-over 角色漂移**：跨 SHOT 沿用同一角色时，下一 SHOT 第一句仍要写明 ta 的位置和状态，不要靠模型"记得"。
- **焦距 jargon**：`85mm/f/1.4/浅景深/焦平面` 改成自然短语：`近景 + 人像压缩感 + 背景柔化 + 脸占画面 40%`。详见 §景别与镜头质感规则。
- **单镜两运镜**：`先推近再环绕 / 跟拍同时摇` 拆成时间戳硬切 `[00:00-00:02] 推镜 / [00:02-00:05] 环绕`，或拆成两个 SHOT。
- 多动作并列（"边走边说边抛媚眼"）：拆成连续两段 SHOT，前段做动作、后段绑定台词。
- 特写或局部镜头引用全量场景图：优先减少场景图引用，以人物或道具 anchor 为主；确需光线/材质时保留短职责。
- 多 ref 没有主图：如果一拍有明显主脸、主服装、主道具或主空间，声明 `主：@图片N`。
- 跨光影区漂移：`空间`字段写明冷暖区起止 + 约束行加"明暗交界线不漂移"。
- 穿模风险：约束行加"禁止穿过不透明物体"；动作描述用"撞上/被遮挡/反弹"替代"穿过"。
- 画外音误写成台词：如果内容留到下一镜，不写引号台词；改成"楼上传来一声冰冷女声，具体内容不清晰，角色僵住/抬眼"。
- 声音重复：同类脚步、塑料袋、破风、金属火花、BGM 只保留一次，按环境音、动作音、BGM 三类合并，每类 ≤3 条。
- 参考图过多：先判断是否职责重叠；身份图和状态图可并存，道具图只有在关键笑点、产品或容易变形时保留。
- 氛围词侵蚀：`电影感/史诗/压抑/克制/高级`优先收束到固定参数段；SHOT 内只保留能生成的动作、视线、光线和声音。

## 质量标准

- 提示词具备清晰的动作与镜头方向。
- 内容可直接投喂，不需要用户再二次翻译。
- 评估结论要能指导下一次改写：明确指出保留什么、删什么、合并什么、拆成几段。

## 失败回退

- 素材不足时，先给最小可用版本，再指出缺口。
- 不用“电影感”“高级感”之类空词糊弄镜头表达。
