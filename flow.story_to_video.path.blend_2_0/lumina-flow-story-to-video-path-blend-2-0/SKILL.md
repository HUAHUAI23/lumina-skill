# story_to_video path blend 2.0

## 路径定位

- 本路径面向 2.0 视频模型。
- 公共上下文只提供当前支持的视频场景和建议时长；本路径自行决定模式选择。
- 整个流程以首帧和参考图生视频为主。
- 重点是根据分镜完成可控分镜图，不是只靠轻语言控制。

## 视频模式规则

- 首帧为主。
- 参考图生视频也支持，适合需要持续约束多个角色、场景、道具、构图或状态参考的镜头。
- 首尾帧谨慎使用，只在结尾硬性结果不能拆镜、也不能由首帧自然演化时使用。
- 纯文字足够时可以使用文生视频。
- 不因为公共上下文支持某模式就机械使用，按当前镜头风险选择。

## 分镜图规则

- 根据分镜决定需要哪些分镜图。
- 可以使用文生图、图生图、溶图完成分镜图。
- 需要同框新画面时，可以把角色图、场景图、道具图溶成一张当前镜头主分镜。
- 可以补特写、动作构图或状态参考图，前提是它解决身份、空间、道具交互、动作构图、状态异变或证据识别风险。
- 不把每个动作阶段都写成独立图片，不为丰富画面添加无关参考图。

## 提示词规则

- 图生图和溶图提示必须说明每张参考图承担什么职责。
- 溶图最终只生成一张当前镜头画面，不做拼贴或多图并列。
- 状态异变要写清保留什么、覆写什么，避免模型恢复成默认漂亮状态。
- 视频提示围绕已生成分镜图，描述动作方向、情绪落点、镜头和节奏。
- 视频提示优先使用 Seedance 时间戳正文：固定参数单独放前面，正文按时间段写景别、运镜、主体动作、场景光线和关键对白。
- 时间段只控制主题和运动，不把姿态逐帧写死。

## 质量标准

- 分镜图服务当前镜头执行，不做海报、立绘或无关氛围图。
- 首帧主分镜可以独立支撑视频起始画面。
- 额外分镜图有明确职责，不重复、不凑数。
- 视频提示词和首帧或参考图一致，不凭空切换空间或人物。
- 首尾帧使用克制，只服务硬性结果。
